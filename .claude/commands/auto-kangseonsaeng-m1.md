# 강선생 지시서: M1 감성 + UX (Week 1)

/auto --mode feature M1 감성 설계 + UX 개선 일괄 실행

## 작업 (순서대로)

### 1. 온보딩 자동 실행
- src/app/page.tsx에서 OnboardingGate 컴포넌트 연결
- 사용자 온보딩 완료 여부 체크 → 미완료 시 OnboardingGate 표시
- 코드 변경량 10줄 이내

### 2. 기출분석 메인 탭
- src/components/layout/ 하단 네비게이션에 '기출분석' 탭 추가
- 링크: /kice/analytics
- 기존 탭 구조 확인 후 5개→5개 유지 (하나를 "더보기"로 접거나 재배치)

### 3. 에러 메시지 인간화
- src/app 내 error.tsx 파일 전체 탐색
- 모든 "오류가 발생했습니다" → 구체적이고 따뜻한 메시지로 교체
- 예: "잠시 문제가 생겼어요. 학습 기록은 안전합니다. 새로고침하면 대부분 해결돼요."
- 각 페이지 맥락에 맞는 메시지 (퀴즈, 기출, 개념학습 등)

### 4. 점수별 감성 분기
- 퀴즈/진단 결과 화면에서 점수 구간별 메시지:
  - 0~30%: 격려 + 구체적 다음 액션
  - 31~60%: 성장 인정 + 취약 영역 제안
  - 61~90%: 칭찬 + 마무리 학습 제안
  - 91~100%: 축하 + 다음 도전 제안

### 5. 용어 순화
- 네비게이션, 버튼, 메뉴 전체 스캔
- 변환 목록:
  - '진단평가' → 유지 (수험생에게 익숙)
  - 'SRS' → '간격 반복'
  - 'BDS' → '상황 시뮬레이션'
  - 'Leitner' → '플래시카드'
  - 'MDX' → 노출 안 되게 처리
  - 기타 영어 전문용어 전수 검사

### 6. 빈 상태 메시지
- 데이터 없는 화면 전체 탐색 (오답노트 비어있을 때, 북마크 없을 때 등)
- 각 빈 상태에 안내 메시지 + CTA 버튼 추가
- 예: "아직 오답 노트가 없어요. 퀴즈를 풀면 자동으로 쌓여요! → 퀴즈 시작하기"

### 7. 기출↔개념 직링크
- /kice/analytics 결과에서 취약 키워드 클릭 시 해당 MDX 개념 페이지로 이동
- src/lib/concept-urls.ts 활용 (이미 존재할 수 있음, 확인 후 사용)

### 8. 마이크로카피 P0 + P1 (14건)
- X 종합 보고서의 마이크로카피 감사 보고서 참고

### 9. 로딩 상태
- 모든 페이지의 loading.tsx 확인
- 스켈레톤 또는 의미 있는 로딩 텍스트 추가
