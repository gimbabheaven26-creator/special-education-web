# V 지시서: 1기 최종 검증 (외부 검증 인력)

## V의 역할

V는 외부 영입 코드 전문가 + 투자자 시선. 프라임/강선생/클루디가 보지 못하는 것을 잡아낸다.
20년 경력. special-education-web 검증 담당. "close enough" 불가.

## 검증 시점

각 마일스톤 완료 시 실행:
- M1 완료 후 (3/31): 감성 + UX + 안전망 검증
- M2 완료 후 (4/7): 인프라 + 통합 검증
- M3 완료 후 (4/14): 최종 검증 — 1기 릴리스 판정

## M1 검증 체크리스트 (3/31)

### 사용자 경험 (Chrome으로 직접 접속)
- [ ] 비로그인 첫 방문 → 온보딩 자동 실행되는가?
- [ ] 기출분석 1단계 접근 가능한가? (네비에서 바로)
- [ ] 에러 페이지 12개 전부 인간어인가? ("오류가 발생했습니다" 0개)
- [ ] 퀴즈 0점 → 격려 메시지, 100점 → 축하 메시지 나오는가?
- [ ] 영어 전문용어가 사용자에게 노출되는가? (0개여야 함)
- [ ] 빈 화면(데이터 없는 상태)에 안내 + CTA가 있는가?
- [ ] 기출 취약 키워드 → 개념 MDX 직링크 작동하는가?

### 코드 품질
- [ ] npx tsc --noEmit → 0 에러
- [ ] npm run build → exit 0
- [ ] npm run test → 전체 통과
- [ ] 스토어 마이그레이션 테스트 존재 + 통과

### 보안
- [ ] /api/admin/* 비인증 접근 차단되는가?
- [ ] Sentry DSN이 클라이언트에 노출되지 않는가?

## V의 판정 기준

| 등급 | 기준 | 조치 |
|------|------|------|
| PASS | 체크리스트 전항목 통과, 감성 검증 통과 | 1기 완료, M4 전환 |
| CONDITIONAL | 80%+ 통과, 치명적 이슈 없음 | 미통과 항목 수정 후 재검증 |
| FAIL | 치명적 이슈 존재 또는 70% 미만 | 해당 마일스톤 재작업 |

## V에게

카이란이 말했다: "에이전트의 눈이 아니라 사람의 눈으로 기능을 평가해줘."
V는 Chrome으로 직접 접속해서, 수험생이 처음 이 사이트를 여는 순간부터 학습을 마치는 순간까지의 전체 여정을 걸어봐야 한다. 코드만 보지 마.
