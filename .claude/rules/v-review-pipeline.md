# V 리뷰 파이프라인

## 구조

V(외부 검증 인력)가 `docs/v-reviews/`에 리뷰 파일을 저장한다.
X는 세션 시작 시 이 디렉토리를 확인한다.

## 세션 시작 시 필수 확인

```
1. docs/v-reviews/ 디렉토리에 파일이 있는가?
2. 미해결 항목(OPEN 상태)이 있는가?
3. 내 담당 항목이 있으면 → 현재 작업보다 우선 처리
```

## V 리뷰 파일 형식

파일명: `v-review-YYYYMMDD-{대상}.md`

```markdown
# V 검증: {대상}

## 발견 사항

| # | 항목 | 심각도 | 담당 | 상태 |
|---|------|--------|------|------|
| 1 | rate limit 없음 | MEDIUM | 강선생1 | OPEN |
| 2 | Discord 인젝션 | MEDIUM | 강선생1 | OPEN |

## 상세
(각 항목의 위치, 문제, 제안)
```

## 상태 관리

- **OPEN**: V가 발견, 아직 수정 안 됨
- **FIXED**: 담당 에이전트가 수정 후 커밋 해시 기록
- **WONTFIX**: 카이란 승인 하에 수정 안 함 (사유 기록)
- **DEFERRED**: M4 이후로 미룸 (사유 기록)

담당 에이전트가 수정하면 상태를 FIXED로 바꾸고 커밋 해시를 기록한다.
V가 다음 검증 시 FIXED 항목을 확인하고 닫는다.

## 우선순위

| 심각도 | 처리 기한 |
|--------|----------|
| CRITICAL | 즉시 (현재 작업 중단) |
| HIGH | 당일 |
| MEDIUM | 해당 마일스톤 내 |
| LOW | 다음 마일스톤 또는 DEFERRED |
