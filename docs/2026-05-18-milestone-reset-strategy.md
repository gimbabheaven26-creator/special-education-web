# SEW 100-Year Strategy: Milestone Reset

> 작성: 2026-05-18  
> 작성자: 루멘(Codex)  
> 목적: 기존 M4 중심 계획을 넘어, 특수교육 임용고시 서비스의 최종 도착점과 다음 마일스톤 체계를 재설정한다.

## 0. 결론

SEW의 장기 도착점은 “무료 퀴즈 앱”이 아니다.

**SEW는 특수교육 임용 수험생을 위한 `검증 가능한 학습 운영체제`가 되어야 한다.**

그 운영체제는 세 가지 엔진으로 움직인다.

1. **전문성 엔진**: 기출, 법령, NISE 용어, 교육과정, 특수교육 HLP를 근거로 콘텐츠를 만들고 계속 검증한다.
2. **성적 향상 엔진**: 진단, 인출 연습, 망각 곡선, 간격 반복, 오답 분류, 실전 모의고사를 하나의 루프로 묶는다.
3. **AI 학습 엔진**: AI를 “채팅 장난감”으로 쓰지 않고, 문항 생성, 해설 검수, 개인화 계획, 서술형 피드백, 전문성 감사에 녹인다.

기존 M4는 폐기하지 않는다. 다만 **M4는 더 큰 로드맵의 “콘텐츠 엔진 초기화” 단계**로 재배치한다.

## 1. 현재까지의 성과

현재 SEW는 이미 시장의 빈틈을 정확히 찌르고 있다.

### 이미 확보한 자산

- 2016~2026 KICE 특수교육 기출 JSON/PDF 데이터
- KICE 키워드/가중치 분석 자료
- NISE 용어사전 기반 데이터
- 101개 MDX 개념학습 파일
- 약 3,100개 이상 퀴즈 문항
- OX, 객관식, 기입형, 서술형, 복합 시나리오형 문항 타입
- 오답노트, 플래시카드, Leitner SRS, 기록, 마스터리, 진단, 오늘학습
- AI-Human 문항 생성/검수 파이프라인
- 기출 ↔ 개념 양방향 연결
- Sentry, rate limit, E2E, V/루멘 검증 체계

### 현재의 핵심 약점

- 콘텐츠 품질을 “상시 검증하는 운영 체계”가 아직 약하다.
- 사용자는 기능을 많이 보지만, “내 성적이 오른다”는 한 문장 루프가 아직 약하다.
- AI가 일부 생성 파이프라인에는 들어왔지만, 학습 경험 전체를 바꾸는 수준은 아니다.
- 내부 slug, 숫자 오류, 라벨 불일치처럼 신뢰를 떨어뜨리는 UX 잔흠이 있었다.
- 기존 M4는 콘텐츠 비율 정규화 중심이라, 장기 제품 비전을 담기엔 좁다.

## 2. 외부 근거 요약

### 특수교육 전문성 기준

- CEC와 CEEDAR Center는 특수교사를 위한 High-Leverage Practices(HLPs)를 정리했다. 이는 협력, 평가, 사회·정서·행동 지원, 수업 네 영역의 핵심 실천으로 볼 수 있다.  
  출처: [Council for Exceptional Children HLP](https://ceds.exceptionalchildren.org/topics/high-leverage-practices)
- IRIS Center는 특수교육 모듈, 사례, 평가, 용어 설명을 교육자 연수형 구조로 제공한다. SEW의 개념학습은 단순 글보다 “사례 → 생각 → 설명 → 평가” 구조를 참고할 수 있다.  
  출처: [IRIS Center](https://iris.peabody.vanderbilt.edu/)
- 한국에서는 KICE 기출, NISE 자료, 교육부 특수교육 교육과정, 국가법령정보센터가 1차 근거가 되어야 한다.  
  출처: [KICE](https://www.kice.re.kr/), [국립특수교육원](https://www.nise.go.kr/), [교육부 2022 특수교육 교육과정 고시](https://www.moe.go.kr/boardCnts/viewRenew.do?boardID=141&boardSeq=90411&lev=0&searchType=)

### 학습 과학 기준

- 인출 연습은 학교·교실 맥락에서도 일관되게 학습 성과를 높이는 것으로 정리되어 있다.  
  출처: [Retrieval Practice Consistently Benefits Student Learning](https://link.springer.com/article/10.1007/s10648-021-09595-9)
- Bjork의 desirable difficulties 관점은 spacing, interleaving, retrieval, varied practice를 “당장은 어렵지만 장기 파지에 유리한 학습 조건”으로 본다.  
  출처: [Bjork Learning and Forgetting Lab](https://bjorklab.psych.ucla.edu/)
- Quizlet, Khanmigo, NotebookLM류 최신 학습 앱은 “AI가 자료를 학습 세트·퀴즈·튜터링 대화로 바꿔 주는 방향”으로 이동하고 있다.  
  출처: [Quizlet AI Study Tools](https://quizlet.com/features/ai-study-tools), [Khanmigo](https://www.khanacademy.org/khan-labs)

### AI 학습 앱 트렌드

최신 학습 앱의 방향은 대체로 네 가지다.

1. **자료 변환**: 노트, PDF, 강의자료를 플래시카드·퀴즈·요약으로 자동 변환한다.
2. **개인화 연습**: 사용자의 약점과 기억 상태를 반영해 다음 문제를 고른다.
3. **AI 튜터**: 정답을 바로 주기보다 질문을 통해 사고를 유도한다.
4. **학습 습관화**: 스트릭, 일일 목표, 짧은 세션, 알림, 리마인드로 재방문을 만든다.

SEW가 따라야 할 방향은 “AI 챗봇 추가”가 아니라, **특수교육 임용 데이터 위에서 이 네 기능을 정확히 구현하는 것**이다.

## 3. 새 북극성

### North Star

> 수험생이 오늘 무엇을 공부해야 할지 고민하지 않아도, SEW가 기출 근거와 개인 약점에 기반해 다음 20분을 제시하고, 그 20분이 실제 점수 향상으로 연결되는 서비스.

### 제품 약속

1. **시험을 안다**: KICE 기출, 교육과정, 법령, NISE 용어를 근거로 한다.
2. **나를 안다**: 사용자의 약점, 기억 상태, 풀이 습관, 실전 준비도를 반영한다.
3. **오늘을 준다**: 막연한 대시보드가 아니라 지금 해야 할 한 세션을 준다.
4. **성장을 증명한다**: 정답률, 숙련도, 약점 감소, 실전 점수 예측으로 보여준다.
5. **전문성을 지킨다**: AI가 만든 것은 사람이 검수하고, 근거 없는 해설은 공개하지 않는다.

## 4. 전략 축 1: 특수교육 전문성 운영체계

### 목표

콘텐츠가 많아지는 것보다 더 중요한 것은 **틀리지 않는 것, 근거가 있는 것, 시험과 연결되는 것**이다.

### 전문성 소스 계층

| Tier | 근거 | 사용 |
|------|------|------|
| 1 | KICE 기출, 국가법령정보센터, 교육부 고시, NISE 공식 자료 | 정답·법령·교육과정·핵심 정의 |
| 2 | CEC HLP, IRIS, PBIS/MTSS/UDL 등 국제 특수교육 기준 | 교수전략·사례형 해설 보강 |
| 3 | KCI/RISS/학술논문, 교재, 강사 자료 | 심화 설명과 최신 쟁점 |
| 4 | AI 생성 초안, 사용자 제보 | 검수 전 자료. 공개 콘텐츠로 직접 승격 금지 |

### 콘텐츠 객체 표준

모든 개념, 용어, 문항은 아래 메타데이터를 가져야 한다.

```ts
type KnowledgeUnit = {
  id: string;
  title: string;
  subject: string;
  chapter: string;
  sourceTier: 1 | 2 | 3 | 4;
  sourceRefs: string[];
  kiceRefs: string[];
  niseTerms: string[];
  legalRefs?: string[];
  curriculumRefs?: string[];
  hlpRefs?: string[];
  lastReviewedAt: string;
  reviewedBy: 'human' | 'V' | 'lumen' | 'X';
  confidence: 'verified' | 'needs-review' | 'draft';
};
```

### 전문성 점검 루프

1. **생성**: AI 또는 사람이 문항/개념 초안 작성
2. **근거 연결**: sourceRefs, kiceRefs, niseTerms 필수
3. **기계 검증**: contract, Zod, 중복, slug, 정답 형식, 해설 길이, 출처 누락 검사
4. **전문 검수**: 카이란 또는 지정 리뷰어가 승인
5. **V 검증**: contract 위반, 데이터 정합성, 보안, 공개 가능성 확인
6. **사용자 제보 반영**: 오류 제보가 들어오면 해당 문항 confidence를 needs-review로 강등
7. **정기 감사**: 월 1회 KICE/법령/교육과정 변경 감지

### 핵심 기능

- **전문성 대시보드**: 전체 문항 중 verified/draft/needs-review 비율
- **기출 커버리지 지도**: KICE 키워드 대비 개념/문항/동형문제 커버리지
- **용어 정확성 검사**: NISE 용어사전 기준으로 표현 통일
- **법령 신선도 검사**: 법령 개정일/문항 작성일 비교
- **오류 제보 워크플로**: 사용자 제보 → 숨김/검토/반영/기각 상태 관리

## 5. 전략 축 2: 임용고시 성적 향상 엔진

### 목표

사용자가 원하는 것은 “기능”이 아니라 **합격 가능성이 올라간다는 확신**이다.

SEW의 학습 루프는 아래처럼 단순해야 한다.

```text
진단 → 오늘의 20분 → 인출 → 피드백 → 복습 예약 → 실전 점검 → 약점 갱신
```

### 핵심 학습 원리

| 원리 | 제품 구현 |
|------|-----------|
| 인출 연습 | 모든 개념은 읽기 후 즉시 문제로 회상 |
| 망각 곡선 | FSRS 또는 개선된 SRS로 복습일 자동 예약 |
| 간격 반복 | 오답/용어/기출 키워드별 복습 큐 |
| 인터리빙 | 같은 과목 몰아풀기와 섞어풀기 모드 모두 제공 |
| 바람직한 어려움 | 너무 쉬운 문제보다 70~85% 성공 구간 유지 |
| 메타인지 | 확신도, 찍음 여부, 실수 유형 기록 |
| 성장 피드백 | “잘했어요”가 아니라 “어떤 약점이 줄었는가” 표시 |
| 실전 전이 | OX보다 기입형·서술형·시나리오형 비중 강화 |

### 사용자가 매일 보는 구조

1. **오늘의 20분**
   - 5분: 어제 틀린 문항 인출
   - 7분: 오늘의 핵심 개념 1개
   - 5분: 관련 기출/동형 문항
   - 3분: 서술형 답안 뼈대 또는 용어 플래시카드

2. **오늘의 근거**
   - “왜 이걸 지금 해야 하는가”를 보여준다.
   - 예: “최근 7일간 법령 IEP 문항 정답률 42%, KICE 빈출 가중치 상위 15%”

3. **오늘의 성장**
   - 문제 수보다 약점 변화 중심.
   - 예: “IEP 법정 절차 오답률이 67% → 40%로 내려갔어요.”

### 점수 향상 기능

- **FSRS 전환**: Leitner UI는 유지하되 스케줄러를 FSRS 계열로 업그레이드
- **확신도 기반 채점**: 맞았지만 찍은 문제는 복습 대상으로 남김
- **오답 유형 분류**
  - 개념 공백
  - 용어 혼동
  - 법령 조항 착오
  - 문제 요구 미독해
  - 서술 구조 부족
- **실전 모의고사**
  - 기입형/서술형 중심
  - 제한 시간
  - 답안 작성 루브릭
  - 회차별 점수 추이
- **합격 준비도 지수**
  - KICE 커버리지
  - 최근 실전 점수
  - 약점 안정성
  - 복습 이행률
  - 서술형 루브릭 점수

## 6. 전략 축 3: AI 학습 엔진

### AI를 넣을 곳

AI는 화면 중앙의 챗봇이 아니라, 학습 루프의 각 지점에 들어가야 한다.

| 위치 | AI 역할 | 공개 전 조건 |
|------|---------|-------------|
| 콘텐츠 생성 | KICE 기반 동형 문항 초안 생성 | human/V 승인 |
| 용어 변환 | NISE 용어 → 플래시카드/기입형 생성 | 용어 출처 필수 |
| 서술형 피드백 | 루브릭 기반 답안 피드백 | 정답 단정 금지, 근거 표시 |
| 약점 분석 | 오답 이력 요약과 다음 학습 추천 | 사용자 데이터 기반 |
| 튜터링 | 정답을 바로 주지 않고 질문으로 유도 | 안전 프롬프트, 출처 제한 |
| 품질 감사 | 해설-정답 불일치, 출처 누락 탐지 | 자동 강등만, 자동 승인 금지 |

### AI 원칙

1. **AI는 초안 작성자다. 최종 권위자가 아니다.**
2. **AI 답변은 출처 없는 정답을 말하지 않는다.**
3. **법령·교육과정·기출 정답은 RAG 또는 고정 데이터에서만 가져온다.**
4. **서술형 피드백은 채점이 아니라 루브릭 코칭으로 표현한다.**
5. **AI가 사용자의 불안을 키우면 실패다. 다음 행동을 줄여야 한다.**

### 우리 서비스형 AI 기능

- **AI 기출 변형소**
  - 원본 기출 선택 → 같은 개념/난이도/문항 구조의 동형문제 생성
- **AI 답안 코치**
  - 사용자의 서술 답안을 4점 루브릭으로 피드백
  - “빠진 핵심 용어”, “논리 순서”, “사례 적용” 표시
- **AI 약점 브리핑**
  - 일주일에 한 번: “이번 주 실제로 줄어든 약점 / 남은 약점 / 다음 3세션”
- **AI 전문성 감사관**
  - draft 문항의 출처 누락, 법령 표현 오류, 정답-해설 불일치 표시
- **AI 스터디 파트너**
  - 스터디그룹 없이도 구두 인출처럼 질문을 던짐

## 7. 새 마일스톤 체계

기존 M4 이후를 아래처럼 재편한다.

### M4R: 신뢰 회복과 전문성 기준선

**목표**: 사용자가 “이 앱은 시험과 특수교육을 정확히 안다”고 느끼게 한다.

- 내부 slug 노출 제거
- 숫자/정답률 신뢰성 보정
- subject/chapter display formatter 전역화
- KICE/NISE/법령/교육과정 sourceRef 표준 도입
- 콘텐츠 confidence 상태 도입
- 오류 제보 → 검토 큐
- 전문성 대시보드 1차

**완료 기준**

- 사용자 화면에서 내부 slug 노출 0건
- 주요 기록/진단/결과 숫자 0~100 범위 보장
- 문항/개념의 sourceRef 누락률 측정 가능
- V 검증 PASS

### M5: 성적 향상 루프

**목표**: 매일 앱을 켜면 “오늘 이걸 하면 점수가 오른다”가 보인다.

- 오늘의 20분 세션
- FSRS 또는 FSRS-compatible 복습 큐
- 확신도/찍음 여부
- 오답 유형 분류
- 약점 기반 추천의 이유 표시
- 성장 피드백 고도화
- 진단 → 오늘학습 → 기록 연결 강화

**완료 기준**

- 사용자가 홈에서 1클릭으로 개인화 세션 시작
- 세션 종료 후 다음 복습일 자동 생성
- 오답 유형별 약점 통계 표시
- 7일 재방문 루프 검증

### M6: 실전 임용 모드

**목표**: SEW가 “기출 PDF 대체재”를 넘어 실전 준비 도구가 된다.

- KICE 실전 모의고사 모드
- 기입형/서술형 답안 입력
- 4점 서술형 루브릭
- 회차별 점수 추이
- 기출 원문 → 개념 → 동형 → 오답 복습 루프
- 시험 D-day 기반 커리큘럼

**완료 기준**

- 2016~2026 회차별 실전 모드 실행
- 사용자 답안 저장
- 루브릭 기반 자기채점/AI 코칭
- 회차별 점수/약점 추이 표시

### M7: AI-Human 콘텐츠 공장

**목표**: 콘텐츠를 빠르게 늘리되, 품질은 떨어뜨리지 않는다.

- KICE 빈출 미출제 주제 자동 탐지
- 동형 문제 생성 UI 연결
- AI draft → 전문성 감사 → human approve
- NISE 용어 기반 플래시카드 자동 생성
- 법령 개정 감지와 영향 문항 표시
- 콘텐츠 갭 자동 리포트

**완료 기준**

- draft 생성부터 approve까지 감사 로그 남김
- sourceRef 없는 AI 문항 공개 불가
- KICE 주요 키워드 80% 이상에 개념/문항/기출 연결

### M8: 커뮤니티와 검증 네트워크

**목표**: 사용자 집단이 콘텐츠 품질과 동기 유지에 기여하게 한다.

- 오류 제보 신뢰도 시스템
- “나도 틀렸어요”를 약점 공감/난이도 지표로 확장
- 스터디 세션 공유
- 합격자/상위 사용자 풀이 전략 카드
- 공개 랭킹은 opt-in 유지

**완료 기준**

- 사용자 제보 처리 SLA 정의
- 제보가 콘텐츠 confidence에 반영
- 커뮤니티 기능이 학습 루프를 방해하지 않음

### M9: 특수교육 지식 그래프

**목표**: SEW만의 방어 불가능한 해자를 만든다.

- 용어 ↔ 개념 ↔ 기출 ↔ 법령 ↔ 교육과정 ↔ 문항 그래프
- “이 개념이 왜 중요한가”를 기출 가중치로 설명
- 개인 약점이 지식 그래프 위에 시각화
- 출제 예측은 확률 놀이가 아니라 커버리지 안내로 표현

**완료 기준**

- 핵심 KICE 키워드 TOP 100 그래프화
- 사용자별 약점 그래프 표시
- 그래프 노드마다 학습/문항/기출 진입 가능

## 11. 최종 방향

SEW가 100년 살아남으려면, 유행을 따라가는 앱이 아니라 **특수교육 임용 학습의 신뢰 인프라**가 되어야 한다.

단기적으로는 UX 신뢰와 오늘의 학습 루프를 정리한다.  
중기적으로는 실전 모의고사와 FSRS 기반 복습으로 성적 향상 엔진을 만든다.  
장기적으로는 KICE/NISE/법령/교육과정/문항/사용자 데이터를 연결한 지식 그래프와 AI-Human 콘텐츠 공장을 만든다.

**다음 마일스톤의 이름은 M4 Phase 2가 아니라 `M4R: 신뢰 회복과 전문성 기준선`이어야 한다.**
