## 1. 프로젝트 개요

**기존 앱** `special-education-web`을 **포크**하여 완전히 새로운 UX 철학의 앱을 만든다.

| | 기존 앱 | 새 앱 |
|---|---|---|
| 철학 | 읽기 → 풀기 | **풀기 → 필요하면 읽기** |
| 메인 화면 | 과목 목록 | **"오늘의 학습" 원버튼** |
| 학습 콘텐츠 | MDX 챕터 (전면 배치) | **백엔드** (문제에서 막힐 때만 불러옴) |
| 진행 시각화 | 정답률 % | **숙달도 4단계** + 로드맵 |
| 동선 | 사용자가 직접 선택 | **알고리즘이 결정** |

## 2. 만들어야 할 핵심 화면 3개

### 화면 1: 홈 (대시보드)
- **"오늘의 학습 시작" 원버튼** — 알고리즘이 복습/신규 문제 혼합하여 세션 생성
- 숙달도 요약 바 (과목별: 새로움 / 서툼 / 숙달 / 완벽)
- 일일 퀘스트 3개 (브론즈/실버/골드)
- 스트릭 카운터 + XP
- "오늘 복습 예정: 23문제" 같은 복습 알림

### 화면 2: 로드맵
임용고사는 범위가 정해진 시험 → 전체 범위를 한눈에 보여주는 것이 핵심.

**3가지 뷰** (탭 전환):

1. **엑셀시트 뷰** (우선 구현)
   - 행: 과목 > 챕터/토픽
   - 열: 학습 상태, 숙달도, 문제 수, 정답률, 마지막 학습일
   - 셀 클릭 → 학습 내용 + 관련 문제가 하이퍼링크로 연결
   - 정렬/필터 가능

2. **마인드맵 뷰** (이후)
   - 과목 간 관계, 큰 그림 파악
   - 중심 노드에서 과목 → 챕터 → 핵심 개념 분기

3. **지도 뷰** (이후)
   - 워크래프트 럼블의 아제로스 지도 스타일
   - 선형 경로 + 노드 (잠금/가능/완료/마스터)
   - 마일스톤 체크포인트 (과목 완료 시)
   - CSS/SVG로 구현 (3D 그래픽 불필요)

### 화면 3: 문제 풀기 (퀴즈 세션)
- 10문제 한 세션 (그 중 3문제는 과거 오답에서 자동 삽입)
- 문제 유형: 객관식, OX, 기입형, 서술형
- **오답 시**: "이 개념 보기" 버튼 → 관련 학습 내용을 모달/사이드패널로 불러옴
- 세션 완료 시: 등급별 보상 연출 (CSS 애니메이션, 브론즈/실버/골드)
- 결과에서 "해당 챕터 보기" 링크 제공

## 3. 차용할 UX 패턴 (벤치마킹 결과)

### 말해보카에서
- **숙달도 4단계 분류**: 갓배운(새로움) → 서툼 → 숙달 → 완벽 (매우숙달)
- **망각곡선 기반 복습 스케줄**: 문항별 마지막 풀이일 + 정답 횟수로 다음 복습일 계산 (1일, 3일, 7일, 14일, 30일 간격)
- **액티브 리콜 우선**: 패시브 읽기가 아니라 문제 풀기가 기본
- **"오늘 복습할 N개" 표시**: 홈에서 복습 예정 수량을 보여줌

### 듀오링고에서
- **단일 선형 경로**: "다음 뭐하지?"를 사용자가 결정하지 않음. 알고리즘이 결정.
- **일일 퀘스트 3단계**: 세션 1회 완료(브론즈), 연속정답 5개(실버), 복습 10개(골드)
- **스트릭 프리즈**: 보유 다이아로 구매, 하루 놓쳐도 스트릭 유지
- **세션 3-5분**: 짧은 루프 + 빈번한 보상
- **자동 복습 혼합**: 새 문제 70% + 복습 30% 자동 섞기

### 클래시로얄/워크래프트 럼블에서
- **노드맵 경로**: 진행을 시각적으로 표시 (잠금 → 해금 → 완료 → 마스터)
- **3-스타 시스템**: 시도함(1) / 통과(2) / 마스터(3)
- **보상 연출**: 결과 화면에 바운스인 애니메이션, 색상별 등급 (브론즈/실버/골드)
- **골든 체크포인트**: 한번 달성하면 후퇴 없음 (마스터된 토픽은 유지)
- **"다음 보상 미리보기"**: 다음 마일스톤까지 얼마 남았는지 항상 표시

## 5. Supabase 데이터 현황

동일 DB를 공유하므로 새 데이터 입력 없이 기존 데이터 활용.

| 테이블 | 행 수 | 용도 |
|---|---|---|
| subjects | 15 | 과목 메타데이터 |
| chapters | 39 | 챕터 (2026-03-14 laws 정리 후) |
| quiz_questions | ~660 | 퀴즈 문항 (4종: multiple/ox/fill_in/descriptive) |
| worksheet_questions | ~316 | 워크시트 문항 |
| worksheet_topics | 32 | 워크시트 토픽 (anti-discrimination 삭제 후) |
| reviews | - | 사용자 리뷰 |

**주의**: 새 데이터는 당분간 입력하지 않기 (카이란 지시)

### 추가 필요할 수 있는 테이블/필드
- `mastery_records`: 문항별 숙달도 추적 (question_id, mastery_level, last_seen, correct_count, next_review)
- `daily_quests`: 일일 퀘스트 상태
- `user_stats`: XP, 스트릭, 다이아몬드 등

→ 이것들은 **클라이언트 Zustand + localStorage**로 먼저 구현하고, 나중에 Supabase로 이전하는 것이 빠름.

## 7. 디자인 원칙

1. **그래픽 없이 구조로 승부** — CSS/SVG만으로 게임적 느낌. 화려한 일러스트 불필요.
2. **선택 마비 제거** — 사용자가 "뭘 해야 하지?" 고민하지 않게. 앱이 결정.
3. **짧은 루프, 빈번한 보상** — 5-10문제 → 보상 → 반복. 3-5분 단위.
4. **숙달도가 진행의 핵심 지표** — 정답률이 아닌 "몇 개를 마스터했나"
5. **모바일 우선** — 터치 타겟 44px, 바텀 내비, 짧은 세션
