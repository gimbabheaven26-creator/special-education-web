## Issue #1 — [콘텐츠] 매년 출제 주제 퀴즈 부재 (P0)

### 배경

5개년(2022-2026) 전공A 분석 결과, **매년 반복 출제되는 4개 영역의 퀴즈가 0문제**임.
시각장애/청각장애/뇌성마비/의사소통은 5년 연속 출제(전공A 12문항 중 3~4문항 차지)인데 학습 콘텐츠가 전무하여, 사용자가 KICE 최빈출 영역을 연습할 수 없는 상태.

### 할 일

- [ ] **시각장애 퀴즈팩** (최소 5문제)
  - 점자 규정(약자, 수표), 보행 훈련(지팡이법, 안내법, 방향정위), 시력 측정(지수/안전수동/광각), 시기능 훈련(추적/추종), 점자정보단말기
- [ ] **청각장애 퀴즈팩** (최소 5문제)
  - 청력도 해석(6분법), 인공와우 구조(변환기/수신기/전극), 보청기 유형, 지문자(모음 지문자 포함), 어음명료도
- [ ] **지체장애(뇌성마비) 퀴즈팩** (최소 5문제)
  - 뇌성마비 유형(경직/무정위/실조), GMFCS 5단계, 원시반사(ATNR/STNR), 자세보조기기, 보치아
- [ ] **의사소통장애 퀴즈팩** (최소 3문제)
  - 조음음운(U-TAP/변별자질/음운변동), AAC 참여모델, 자발화 분석, EMT(강화된 환경중심 언어중재)

### 우선순위: **P0** (즉시)

> 이 4개 팩이 없으면 KICE 출제 비중의 약 30%를 커버하지 못함.

## Issue #2 — [콘텐츠] introduction 과목 퀴즈 부족 (P0)

### 배경

KICE 출제 비중 **24%로 전 과목 최대**인데, 현재 퀴즈는 **5문제(7.7%)** 뿐.
지적장애(증후군/적응행동), 정서행동장애(품행장애/DSM-5), 건강장애(병원학교) 등이 빠져 있음.
갭 분석 기준 **-16.3%p** 차이로, 7개 과목 중 가장 심각한 불균형.

### 할 일

- [ ] 지적장애 심화: 증후군별 특성(다운/프래더윌리/엔젤만/윌리엄스), 적응행동 개념 — 5문제
- [ ] 정서행동장애: 품행장애/반항성장애 구분, DSM-5 분류, 사회불안장애 — 3문제
- [ ] 건강장애: 병원학교/원격수업 법적 근거, 결석발작(소발작) — 2문제
- [ ] 자폐성장애 보강: 반향어, 공동관심, TEACCH vs PRT 비교 — 3문제
- [ ] 감각장애 개론: 시각장애/청각장애 정의·분류 기초 — 2문제
- 목표: introduction 퀴즈 5 → **최소 20문제**

### 우선순위: **P0** (즉시)

## Issue #3 — [기능] KICE 실전형 퀴즈 포맷 도입 (P1)

### 배경

현재 퀴즈 타입은 `multiple | ox | fill_in | descriptive` 4종.
KICE 4점 서술형은 **시나리오 → 3~4개 하위 질문** 구조인데, 현 포맷으로는 이를 표현할 수 없음.
2026 전공A 기준 12문항 중 8문항(67%)이 복합 시나리오형.

### 할 일

- [ ] `QuizType`에 새 유형 추가 검토:
  - `scenario_composite` — 시나리오 기반 복합문항 (caseContext + 하위 질문 3~4개)
  - `error_finding` — "틀린 것 N가지 찾아 고치시오" 형태
  - `graph_interpretation` — 도표/그래프/청력도 해석 (이미지 첨부)
- [ ] `QuizQuestion` 인터페이스 확장 검토:
  - `subQuestions?: SubQuestion[]` — 하위 질문 배열
  - `imageUrl?: string` — 그래프/도표 이미지
- [ ] 기존 `caseContext` 필드 활용 방안 우선 검토 (구조 변경 최소화)
- [ ] 시나리오형 샘플 퀴즈 2~3문제 제작하여 UI 검증

### 우선순위: **P1** (다음 스프린트)

> 콘텐츠(P0 이슈) 추가와 병행 가능하나, 스키마 변경이 필요하므로 설계 우선.

자유학기제 변경, 범교과 학습주제, 진로연계교육, 학교스포츠클럽, 전공과 등이 출제됨.
현재 퀴즈에 2022 개정 관련 문항 **0문제**.

### 할 일

- [ ] 범교과 학습주제 10가지 퀴즈 — 2문제
- [ ] 진로연계교육 개념·적용 — 2문제

- [ ] 전공과(18세 이후) 교육과정 — 1문제
- 총 최소 **9문제**, curriculum 과목에 배치

### 우선순위: **P1** (다음 스프린트)

현재 이들 도구에 대한 퀴즈 **0문제**.

## Issue #6 — [데이터] 복합영역 태그 시스템 (P2)

### 배경

현재 `QuizQuestion.subject`는 **단일 문자열**(`string`)로, 1문제 = 1과목 분류.
그러나 KICE 4점 문항은 거의 모두 **2~3개 영역 융합**임.

예시(2026 전공A-5): 강선생 분류 `curriculum` → 실제 포함 영역 `curriculum + inclusive(TAI) + assessment(평가조정)`

단일 분류로는 "통합교육+교육과정" 같은 교차 학습이 불가능하고, 사용자에게 실전과 다른 학습 경험을 제공하게 됨.

### 할 일

- [ ] `QuizQuestion` 인터페이스에 복합 태그 필드 추가:
  ```
  subjects?: string[]  // 다중 영역 태그 (기존 subject는 주 영역으로 유지)
  ```
- [ ] 기존 65문제에 대해 복합 태그 매핑 (최소 난이도 3 문제 대상)
- [ ] 과목별 퀴즈 조회 시 `subjects` 배열도 검색 대상에 포함하도록 `getQuizzesBySubject` 수정
- [ ] 향후 "복합 영역 연습" 모드 UI 검토 (Phase 2)

### 우선순위: **P2** (Phase 2)

> 기능적 변경이므로 콘텐츠 추가(P0/P1)가 먼저. 단, 신규 퀴즈 제작 시 `subjects` 필드를 미리 넣어두면 나중에 마이그레이션 비용 절감.
