# 종합 분석 리포트: 수험생 관점의 서비스 평가

> 작성일: 2026-03-15 | 5개 병렬 에이전트 리서치 종합
> 관점: **특수교육 임용시험 수험생**

---

## TL;DR — 핵심 3줄 요약

1. **시장에 디지털 학습 도구가 전무하다** — 기출 분석, SRS, 오답 자동화, 학습 분석 모두 미개척. 우리가 첫 주자.
2. **이미 구현된 기능의 방향은 정확하다** — 하지만 "쓸 만한 수준"까지 가려면 데이터 영속성, 문항 수, 기출 모의고사 모드가 필수.
3. **인강 대체가 아닌 보완으로 포지셔닝** — 개념 이해는 인강, 개념 정착(반복·인출·분석)은 우리 앱.

## 1. 시장 환경 (경쟁사 리서치)

### 1.1 시장 구조

| 플레이어 | 역할 | 디지털 도구 |
|----------|------|------------|
| 해커스/박문각/임용닷컴/올인패스 | 인강+학원 (연 200~300만원) | 동영상 스트리밍만 |
| G스쿨 | 인강 | 장특법 Quiz 앱 (매우 제한적) |
| KICE | 기출 원본 | PDF만 제공 |
| 다음카페 (특수교사를 꿈꾸며 등) | 커뮤니티 | 없음 |
| **우리 서비스** | **무료 웹앱** | **유일한 통합 디지털 도구** |

### 1.2 결정적 시장 공백 (Market Gap)

| 기능 | 글로벌 에듀테크 | 임용시험 분야 | 우리 서비스 |
|------|----------------|--------------|------------|
| 간격반복(SRS) | Anki, Quizlet | 전무 | Leitner 구현 완료 |
| 자동 오답노트 | 다수 | 전무 (수기) | 자동수집+필터+재시험 |
| 학습 분석 | 대시보드 보편화 | 전무 | 히트맵+추이+취약영역 |
| 기출 디지털화 | CBT 보편화 | PDF만 존재 | 21개 JSON 완료 |
| 게이미피케이션 | XP/스트릭 보편화 | 전무 | XP+스트릭+일일목표 |
| 적응형 테스팅 | AI 맞춤 난이도 | 전무 | 오답 자동 믹싱(부분적) |

**결론: 우리가 이 시장의 첫 디지털 도구다. 경쟁자가 없다.**

## 2. 현재 서비스 강점 (UX 분석)

### 2.1 잘 되어 있는 것

| 강점 | 수험생 가치 | 상세 |
|------|------------|------|
| **학습 순환 구조** | 체계적 복습 | 학습→퀴즈→오답→플래시카드→통계 피드백 루프 완성 |
| **오답 자동 믹싱** | 간격반복 효과 | 퀴즈 세션에 이전 오답 3문제 자동 삽입 |
| **다양한 문제 유형** | 실전 대비 | 객관식, OX, 기입형, 서술형, 시나리오형 — 실제 임용시험 유형 반영 |
| **기입형 채점 로직** | 정확한 자동채점 | "또는" 복수정답, 괄호 약어(DRO, PBS) 매칭 |
| **취약영역 딥링크** | 효율적 보강 | 통계→취약과목→오답→연습 한 번에 연결 |
| **모바일 터치 UX** | 이동 중 학습 | 44px 터치타겟, 스와이프 플래시카드, 하단 탭바 |
| **3D 플립 플래시카드** | 몰입감 | Leitner 5박스 + 스와이프 인터랙션 |
| **KICE 기출 디지털화** | 기출 분석 | 2016~2026 전공A+B 21개 시험 1,111 키워드 |

### 2.2 gosari-namu-path에서 참고할 점

| 기능 | 설명 | 적용 가치 |
|------|------|----------|
| **Supabase Auth** | 사용자 인증 → 서버 데이터 동기화 | 치명적 약점 해결 (데이터 영속성) |
| **React cache()** | 서버 컴포넌트 쿼리 중복 제거 | 성능 최적화 |
| **MDX 인터랙티브** | 콘텐츠 내 퀴즈/시각화 삽입 | 학습 콘텐츠 풍부화 |
| **Knowledge Graph** | 개념 간 연결 시각화 | 기출 키워드 연결망 |
| **Study Session Timer** | 학습 시간 자동 측정 | 학습량 정량화 |
| **Playwright E2E** | 핵심 플로우 자동 검증 | 품질 보증 |
| **PWA 지원** | 오프라인+홈 화면 추가 | 모바일 접근성 |

### 2.3 edumind에서 참고할 점

| 기능 | 설명 | 적용 가치 |
|------|------|----------|
| **FSRS (ts-fsrs)** | Leitner보다 정교한 간격반복 | 이미 package.json에 있지만 미사용 |
| **신뢰도 기반 평가** | 찍음/반반/확실 → Hard/Good/Easy | 자기 인식(메타인지) 강화 |
| **오답 유형 분류** | lucky_correct/careless/conceptual_gap/confusion | 취약점 정밀 진단 |
| **6가지 퀴즈 모드** | 순차/랜덤/약점집중/시간제한/마라톤/데일리 | 다양한 학습 상황 대응 |
| **세션 복구** | localStorage 자동저장 → 이어풀기 | 퀴즈 중 이탈 대응 |
| **적응형 난이도** | 정답률에 따라 난이도 자동 조절 | 최적 학습 부하 유지 |
| **코칭 시스템** | 풀이 후 학습 조언 제공 | 자기주도 학습 안내 |

## 3. 현재 서비스 약점 (수험생 관점)

### 3.1 치명적 (서비스 신뢰도 파괴)

#### 데이터 영속성 없음
- 모든 학습 기록(퀴즈 5,000건, 오답 500건, 플래시카드, 스트릭, XP)이 **localStorage만** 사용
- PC→모바일 전환, 브라우저 데이터 삭제 시 **전체 기록 소실**
- 임용 준비는 1~3년 장기전 — 학습 기록 유실은 치명적
- **해결**: Supabase Auth + 서버 동기화 (gosari 참고)

### 3.2 심각 (학습 효율 저하)

#### 챕터/범위별 퀴즈 선택 불가
- 과목 단위로만 퀴즈 시작 가능, 챕터 선택 불가
- "오늘은 IEP 챕터만 50문제" 같은 집중 학습 불가
- 수험생은 취약 챕터를 반복 연습하고 싶어한다

#### 문항 수 부족 (710문항)
- 과목당 평균 65문항 → 7~8세션이면 소진
- 며칠만 쓰면 같은 문제를 반복해서 만남
- 최소 과목당 200+ 필요

#### 오답노트에서 해설 미표시
- WrongNoteCard에서 `question`과 `answer`만 표시
- `explanation` 필드가 렌더링되지 않음
- 왜 틀렸는지 복습하려면 해설이 필수

#### KICE 기출 ≠ 앱 퀴즈 유형 불일치
- 실제 임용시험: **기입형(2점) + 서술형(4점)만** 출제 (객관식/OX 없음)
- 앱 퀴즈: 객관식/OX 비중이 높음
- 실전 대비 관점에서 기입형·서술형 문항 비중 확대 필요

### 3.3 중요 (사용 편의 저하)

| 약점 | 설명 |
|------|------|
| 기출 모의고사 모드 없음 | KICE 데이터는 있지만 시간제한+채점+히스토리 없음 |
| 서술형 자기 채점 한계 | 키워드 매칭/채점 가이드라인 보조 장치 없음 |
| 학습 플래너 없음 | D-day 기반 과목별 분량 배분 기능 없음 |
| 난이도별 필터 없음 | difficulty 1/2/3 필드 있지만 선택 UI 없음 |
| 일일 목표 커스터마이징 | setDailyGoal 함수 존재하지만 호출 UI 없음 |
| 기출 모바일 접근 불가 | /kice가 모바일 하단 탭바에 없음 |
| 검색 범위 제한 | 퀴즈 question/explanation만, MDX 본문 미포함 |
| 플래시카드 타임존 | Leitner: UTC / StudyStore: KST — 불일치 가능 |

## 4. 전략적 포지셔닝

```
┌─────────────────────────────────────────────┐
│         특수교육 임용시험 준비 생태계          │
│                                             │
│  [인강/학원] ←── 개념 이해 (대체 불가)       │
│       ↓                                     │
│  [우리 앱]  ←── 개념 정착 (반복·인출·분석)   │
│       ↓         ★ 이 영역이 비어있다 ★       │
│  [커뮤니티] ←── 정보 공유 + 동기 부여        │
└─────────────────────────────────────────────┘
```

**핵심 메시지**: 인강 수강생이 **추가 비용 없이** 쓰는 보조 도구. 경쟁이 아닌 공존.
