# 콘텐츠 작성 가이드

> 작성 대상: 특수교육 임용고시 준비 웹사이트 콘텐츠
> 작성 형식: Google Docs 초안 → MDX 변환
> 작성자 기준: 중등특수교사 임용고시 수험생 눈높이

---

## 1. 전체 콘텐츠 구조

### 과목 구성 (7과목 × 3단원 = 총 21단원)

| 과목 slug | 과목명 | 단원 수 |
|-----------|--------|---------|
| introduction | 특수교육학 개론 | 3 |
| curriculum | 특수교육 교육과정 | 3 |
| inclusive-education | 통합교육 | 3 |
| assessment | 진단 및 평가 | 3 |
| behavior-support | 행동지원 | 3 |
| transition | 전환교육 | 3 |
| laws | 관련 법령 | 3 |

### 단원당 필수 섹션

각 단원 MDX 파일은 반드시 아래 5개 섹션을 포함해야 합니다.

1. **핵심개념** - 단원의 핵심 정의와 개념 2~5개
2. **주요이론** - 이론적 배경, 모델, 절차 설명
3. **키워드정리** - 시험에 자주 나오는 용어 정리표
4. **기출연계** - 실제 임용 기출 문제 유형 분석
5. **학습체크** - 단원 마무리 자가점검 항목

## 2. Google Docs 작성 방법

### 2-1. 문서 제목 규칙

```
[과목명] 단원N. 단원제목
예시: [행동지원] 단원1. 응용행동분석(ABA) 기초
```

### 2-2. 구조 (Google Docs 스타일 → MDX 변환 기준)

Google Docs에서 아래 헤딩 스타일을 사용하세요.

| Google Docs 스타일 | MDX 변환 결과 | 용도 |
|-------------------|---------------|------|
| 제목 (Title) | frontmatter `title` | 단원 제목 |
| 제목 1 (Heading 1) | `## 섹션명` | 핵심개념, 주요이론 등 5대 섹션 |
| 제목 2 (Heading 2) | `### 소제목` | 섹션 내 세부 항목 |
| 제목 3 (Heading 3) | `#### 세부항목` | 소제목 내 세부 내용 |
| 본문 (Normal) | 일반 문단 | 설명 텍스트 |
| **굵게** | `**키워드**` | 핵심 용어, 반드시 시험에 나오는 개념 |

### 2-3. 필수 frontmatter 정보 (문서 맨 위에 메모 형식으로 작성)

Google Docs 문서 맨 위에 다음 메모를 남겨두세요. MDX 변환 시 frontmatter로 옮겨집니다.

```
[MDX FRONTMATTER]
title: 단원 제목 (한국어)
description: 단원 설명 1~2줄
subject: 과목 slug (영문)
chapter: 단원 slug (영문)
order: 단원 순서 (숫자)
keywords: 키워드1, 키워드2, 키워드3
```

## 3. 각 섹션 작성 기준

### 섹션 1. 핵심개념

- 해당 단원에서 반드시 알아야 할 정의 2~5개
- 각 개념은 **용어**: 정의 형식으로 작성
- 출처(학자명, 연도) 반드시 기재
- 분량: 300~500자

**작성 예시:**
```
## 핵심개념

**응용행동분석(ABA, Applied Behavior Analysis)**: Baer, Wolf, Risley(1968)가 정의한 것으로,
행동의 원리와 기법을 사회적으로 중요한 행동의 개선에 체계적으로 적용하는 과학이다.
```

### 섹션 2. 주요이론

- 이론의 발전 과정, 핵심 모델, 세부 원리 설명
- 번호 목록 또는 소제목 구조로 작성
- 비교가 필요한 경우 반드시 **표(table)** 사용
- 분량: 800~1,500자

**표 작성 예시 (Google Docs 표 → MDX 변환):**
```
| 강화 유형 | 정의 | 예시 |
|----------|------|------|
| 정적 강화 | 행동 후 선호 자극 제공 | 칭찬 스티커 제공 |
| 부적 강화 | 행동 후 혐오 자극 제거 | 숙제 면제 |
```

### 섹션 3. 키워드정리

- 시험에 나오는 핵심 용어를 **용어 | 설명 | 출제 빈도** 형식의 표로 정리
- 출제 빈도: ★★★(자주), ★★(보통), ★(가끔)
- 분량: 10~20개 키워드

### 섹션 4. 기출연계

- 실제 기출 문제 출제 경향 분석
- 기입형/서술형 예상 문제 유형 제시
- 각 유형별 답안 작성 포인트 안내
- 주의: 저작권이 있는 실제 기출 문제 전문 복사 금지. 유형·패턴만 제시.

**기출 유형 분석 예시:**
```
### 기출연계

#### 기입형 출제 패턴
- "( )은/는 행동 후 자극을 제공하여 행동 빈도를 증가시키는 것이다" → 답: 정적 강화
- "ABA의 ( )가지 차원 중 하나인 ( )은/는 행동 변화가 환경에 일반화되어야 함을 의미한다"

#### 서술형 출제 패턴
- 사례 제시 후 ABA 원리 적용하여 분석하는 형식
- 강화/벌의 종류를 구분하고 각각의 예시를 제시하는 형식
```

### 섹션 5. 학습체크

- 단원 이해도를 확인하는 자가점검 질문 5~10개
- 체크리스트 형식 (- [ ] 항목)
- 각 항목은 "~를 설명할 수 있다" 또는 "~의 차이를 구분할 수 있다" 형식

## 4. 서식 규칙

### 4-1. 굵게 (Bold) 사용 기준

- **반드시 시험에 나오는 핵심 용어**: `**용어**`
- 중요한 숫자/연도: `**1968년**`, `**7가지**`
- 학자 이름(처음 등장 시): `**Baer, Wolf & Risley**`

### 4-2. 표(Table) 사용 기준

비교/대조가 필요한 경우 반드시 표를 사용하세요:
- 강화 vs 벌 비교
- 이론가들의 주장 비교
- 평가 방법 비교
- 법령 조항 비교

### 4-3. 인용 블록

출처나 정의를 직접 인용할 때:
```
> Baer, Wolf & Risley(1968): "응용행동분석은 행동을 개선하기 위해 행동의 원리를
> 적용하는 과정이다."
```

### 4-4. 주의/팁 박스

```
> **⚠️ 시험 주의사항**: 정적/부적의 '정적(positive)'은 '좋다'는 의미가 아니라
> '자극을 더한다(+)'는 의미입니다.
```

## 5. 퀴즈 데이터 작성 방법

퀴즈는 MDX 파일과 별도로 TypeScript 파일(`/src/data/quizzes/[subject].ts`)에 작성합니다.

### 5-1. 퀴즈 타입

임용고시 형식에 맞춰 **fill_in(기입형)** 과 **descriptive(서술형)** 을 주로 사용하세요.

| 타입 | 임용 형식 | 사용 기준 |
|------|----------|----------|
| `fill_in` | 기입형 | 용어, 개념, 숫자를 빈칸에 채우는 문항 |
| `descriptive` | 서술형 | 사례 분석, 개념 설명, 비교 서술 문항 |
| `ox` | - | 개념의 참/거짓 판단 (보조용) |
| `multiple` | - | 사지선다 (보조용, 임용에는 없음) |

### 5-2. 문항 ID 규칙

```
[과목 약어]-q[번호]
예시: bs-q6, cu-q1, ie-q3
```

과목 약어:
- bs: behavior-support
- in: introduction
- cu: curriculum
- ie: inclusive-education
- as: assessment
- tr: transition
- lw: laws

### 5-3. 난이도 기준

| 난이도 | 기준 |
|--------|------|
| 1 | 기본 개념 정의 암기 (반드시 알아야 할 것) |
| 2 | 개념 적용, 비교 분석 |
| 3 | 사례 적용, 복합 분석, 고난도 서술 |

### 5-4. tags 활용

```typescript
tags: {
  disability: '정서행동장애',  // 관련 장애 유형
  year: 2023,                  // 기출 연도 (기출 문제인 경우)
  round: 1                     // 기출 차수 (1차=필기, 2차=면접)
}
```

## 6. Google Docs → MDX 변환 절차

### 변환 순서

1. **Google Docs에서 마크다운으로 내보내기**
   - 파일 → 다운로드 → 마크다운(.md)

2. **파일 확장자 변경**
   - `.md` → `.mdx`

3. **frontmatter 추가** (파일 맨 위에)
   ```mdx
   ---
   title: "단원 제목"
   description: "단원 설명"
   subject: "과목-slug"
   chapter: "단원-slug"
   order: 1
   keywords: ["키워드1", "키워드2"]
   ---
   ```

4. **저장 위치**
   ```
   /content/[subject-slug]/[chapter-slug].mdx
   예시: /content/behavior-support/aba-basics.mdx
   ```

5. **변환 후 체크리스트**
   - [ ] 표가 올바르게 변환되었는가
   - [ ] 굵게(Bold) 서식이 유지되었는가
   - [ ] 헤딩 레벨이 올바른가 (H1→##, H2→###, H3→####)
   - [ ] 특수문자 깨짐이 없는가

### 흔한 변환 오류

| 오류 | 원인 | 해결 |
|------|------|------|
| 표가 텍스트로 변환됨 | Google Docs 표 형식 문제 | 수동으로 마크다운 표 재작성 |
| 줄바꿈 2개 필요 | 마크다운 문단 구분 | 빈 줄 추가 |
| 괄호 안 텍스트 굵기 사라짐 | 마크다운 파싱 오류 | `\(` 이스케이프 처리 |

## 8. 작성 우선순위

아래 순서로 작성을 진행하세요:

1. **1순위**: behavior-support (행동지원) - ABA, PBS는 매년 출제
2. **1순위**: assessment (진단 및 평가) - FBA, CBM 자주 출제
3. **2순위**: curriculum (교육과정) - IEP는 필수
4. **2순위**: inclusive-education (통합교육) - 협력교수, UDL
5. **3순위**: introduction, transition, laws

## 9. 정확성 검토 요청

콘텐츠 작성 후 반드시 아래를 확인하세요:

- KICE(한국교육과정평가원) 공식 자료와 대조
- 최신 특수교육법 시행령(연도 확인) 기준 적용
- 학자명·연도 오류 여부 (APA 형식 기준)
- AI가 생성한 내용은 반드시 교사가 직접 검토 후 게시

> **⚠️ 중요**: 이 웹사이트의 콘텐츠는 임용고시 수험생들이 학습하는 자료입니다.
> 오류가 있으면 수험 결과에 직접 영향을 미칩니다. 모든 내용은 공식 자료와 반드시 대조하세요.
