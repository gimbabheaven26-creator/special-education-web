# 이음진 시험 헌법

Date: 2026-06-08
Branch: `codex/ieumjin-greenfield`
Status: Draft 1

## 0. 반성

이음진은 목적지 없이 새 화면을 덧칠하는 프로젝트가 아니다. SEW에는 이미 특수교육 임용시험 구조, KICE 기출 데이터, 용어 데이터, 개념 MDX, 기출-개념 연결, 동형 문제 생성 규칙이 존재한다.

앞으로 이음진은 SEW Classic의 UI와 라우트를 가져오지 않는다. 대신 SEW가 축적한 시험 도메인 구조를 공식 출처와 대조해 새 학습 경험으로 재구성한다.

## 1. 목적지

이음진의 목적지는 다음 한 문장이다.

> 특수학교(중등) 교사 임용시험의 실제 법령, 시행계획, 기출 구조를 기준으로 수험생의 지식을 전공A/B 답안 가능 상태까지 끌어올리는 용어 중심 학습 캠퍼스.

이 문장에 들어맞지 않는 기능은 만들지 않는다.

## 3. SEW에 이미 있던 도메인 구조

이음진은 다음 자산을 "도메인 재료"로 읽는다. 화면, 네비게이션, Classic 라우트, 기존 제품명은 가져오지 않는다.

| 자산 | 위치 | 현재 가치 | 승계 방식 |
| --- | --- | --- | --- |
| 시험 구조 | `src/data/exam-structure.json` | 특수학교(중등) 임용시험, 전공A/B 문항 수·배점·시간, 과목 가중치, 수준별 전략 | 이음진의 시험 상수 후보. 공식 공고와 대조 후 별도 schema로 이관 |

## 4. 시험 구조 상수

현재 SEW 데이터 기준 전공 구조는 다음과 같다. 실제 서비스 노출 전에는 매년 시도교육청 공고와 KICE 원문으로 재검증한다.

| 교시 | 문항 구조 | 배점 | 시간 |
| --- | --- | --- | --- |
| 전공A | 단답형 4문항 x 2점, 서술형 8문항 x 4점 | 40점 | 90분 |
| 전공B | 단답형 2문항 x 2점, 서술형 9문항 x 4점 | 40점 | 90분 |

이음진의 모든 연습은 결국 아래 질문으로 환원되어야 한다.

1. 이 용어를 전공A/B 단답형 답으로 쓸 수 있는가?
2. 이 용어를 사례 지문 속에서 찾아낼 수 있는가?
3. 이 용어와 혼동되는 개념을 구분할 수 있는가?
4. 이 용어를 기출형 서술 답안의 문장으로 확장할 수 있는가?

## 5. 제품 골격

이음진의 중심 객체는 페이지가 아니라 `용어 렌즈`다.

```ts
type TermLens = {
  termId: string;
  label: string;
  aliases: string[];
  officialDefinition: SourceBackedText;
  distinction: DistinctionCard[];
  conceptBridge: ConceptRef[];
  kiceRefs: KiceRef[];
  examDna: ExamDna;
  analogSeeds: AnalogSeed[];
  reviewState: ReviewState;
  sourceHealth: SourceHealth;
};
```

`용어 렌즈`는 항상 다음 순서로 경험된다.

```text
용어 진입
→ 공식 정의
→ 혼동 구분
→ 개념교
→ 기출진
→ 답안 문장화
→ 동형 문제 생성
→ 복습 예약
```

이 구조 때문에 `/terms`, `/concepts`, `/kice`, `/quiz`, `/flashcards`처럼 기능별 섬을 나열하는 방식은 이음진의 1차 구조가 될 수 없다. 필요하면 내부 기능으로만 존재한다.

## 6. KICE DNA 규칙

동형 문제와 기출형 학습은 SEW의 KICE 분석 수치를 따른다.

| 규칙 | 기준 | 제품 적용 |
| --- | --- | --- |
| 지문 길이 | KICE context 평균 105자 | 모바일 기본 지문은 80~140자. 심화는 원문 길이 유지 |
| 자료 구조 | `(가)/(나)` 병치가 절반 이상 | 단순 퀴즈보다 자료 비교형을 우선 |
| 하위 자료 | sub_items 83.8% | 렌즈 안에 자료 카드와 물음 카드를 분리 |
| 과제 구조 | tasks 72.5%, 평균 3개 | 모바일은 1~2개로 압축하되 원문 모드는 전체 유지 |
| 복합 영역 | 47.4% | 행동지원+개론, 교육과정+통합교육 등 교차 렌즈를 필수 제공 |

AI가 동형 문제를 만들 때는 이 규칙을 통과하지 못하면 초안으로도 저장하지 않는다.

## 7. AI 사용 원칙

AI는 답을 꾸며내는 존재가 아니라 시험 구조를 조립하고 검수 대기 초안을 만드는 작업자다.

AI가 해도 되는 일:

- 공식 정의와 기출 문항을 연결할 후보 추천
- 원본 기출의 `exam_dna` 추출
- 같은 개념, 다른 각도, 같은 난이도의 동형 문제 초안 생성
- 사용자 오답 원인을 `개념 공백`, `용어 혼동`, `법령 착오`, `지문 독해 실패`, `서술 구조 부족`으로 분류
- 서술형 답안의 빠진 키워드와 논리 순서 피드백

AI가 하면 안 되는 일:

- 공식 근거 없이 법령·교육과정 문구를 확정
- 검수 없이 정답/해설을 공개
- 원본 기출을 무단 변형해 출처 없는 새 문제처럼 노출
- 사용자가 모르는 사이 과거 Classic 문항 UI로 이동시키기

## 8. 전문성 유지 시스템

이음진의 콘텐츠는 아래 상태 중 하나를 가져야 한다.

| 상태 | 의미 | 공개 범위 |
| --- | --- | --- |
| `verified` | 공식 출처, KICE 원문, 사람 검수 완료 | 전체 공개 |
| `source-linked` | 출처는 연결됐으나 사람 검수 전 | 학습 보조로 제한 |
| `ai-draft` | AI 초안 | 관리자/검수 화면만 |
| `stale` | 법령·교육과정·공고 갱신 필요 | 사용자에게 경고 표시 |
| `blocked` | 근거 충돌 또는 오류 제보 미해결 | 공개 금지 |

매년 해야 할 감사:

1. 국가법령정보센터 법령 시행일 확인
2. 교육부 특수교육 운영계획 갱신 확인
3. KICE 최신 기출 원문 PDF 확보
4. 시도교육청 시행계획 공고와 전공A/B 구조 대조
5. NISE 용어 변경 여부 확인
6. 동형 문제의 원본 대비 난이도·구조 drift 검사

## 9. 학습 과학 적용

이음진은 예쁜 사전이 아니라 기억과 답안 능력을 올리는 시스템이어야 한다.

| 원리 | 이음진 구현 |
| --- | --- |
| 인출 연습 | 정의를 읽은 직후 빈칸/단답/서술 문장으로 회상 |
| 간격 반복 | 용어별 기억 상태와 오답 유형별 복습 예약 |
| 망각 곡선 | 쉬운 용어는 간격 확대, 혼동 용어는 짧은 간격 유지 |
| 인터리빙 | 같은 과목 반복 후 관련 과목 교차 문제를 섞음 |
| 바람직한 어려움 | 정답률 70~85% 구간을 유지하도록 난이도 조정 |
| 메타인지 | `알고 맞음`, `찍어서 맞음`, `헷갈려서 틀림`, `처음 봄` 기록 |
| 전이 | OX보다 단답형·서술형·사례형으로 이동 |

일일 세션은 다음 20분으로 제한한다.

```text
3분: 오늘의 용어 1개 공식 정의 회상
5분: 혼동 용어 2개 구분
5분: 관련 기출 1개 분석
5분: 동형 문제 1개 풀이
2분: 답안 문장과 복습 예약
```

## 10. 금지 목록

이음진에서는 다음을 금지한다.

- `Next`, `SEW Next`라는 제품명 사용
- Classic 라우트를 첫 경험으로 연결
- Classic Header/Footer/BottomTab을 이음진 사용자 화면에 노출
- `/concepts`, `/kice`, `/admin/ai-generate` 같은 기존 페이지를 이음진 핵심 동선으로 재사용
- "퀴즈 앱", "대시보드", "기록 페이지" 중심의 평면적 IA
- 공식 출처 없는 AI 해설 공개
- SEW Classic 운영 배포와 이음진 배포를 같은 Vercel 프로젝트로 섞기

## 11. 첫 vertical slice

첫 구현 단위는 `기능적 행동평가(FBA)` 하나로 한다.

완료 조건:

1. NISE/공식 정의 또는 검수 정의 연결
2. 관련 개념: 행동지원, PBS, ABC 관찰, 기능 가설, 대체행동, 선행사건/후속결과
3. 관련 KICE 문항 3개 이상 또는 없으면 명시적 "기출 직접 연결 없음" 표시
4. KICE DNA 추출: 문항 유형, 지문 구조, 물음, 채점 키워드
5. 동형 문제 1개 생성 초안
6. AI 초안은 `ai-draft`, 사람 검수 전 공개 금지
7. 사용자는 20분 안에 정의 회상, 구분, 기출 분석, 동형 풀이, 복습 예약을 끝낼 수 있어야 함

## 12. 다음 마일스톤

### M0. 헌법 고정

- 이 문서를 기준으로 이음진 설계와 구현을 한다.
- `docs/ieumjin-isolation.md`는 제품 격리 원칙, 이 문서는 시험 목적지 원칙이다.

### M1. 원천 색인

- `data/kice-기출`에 원문 PDF URL, 검수자, 검수일, 공식/동형/예상 구분을 추가할 schema를 설계한다.
- `data/terminology`를 `TermLens` 후보로 변환하는 read-only index를 만든다.

### M2. 기능적 행동평가 렌즈

- 하나의 용어에서 공식 정의, 혼동 구분, 개념, 기출, 동형, 복습이 이어지는 실제 화면을 만든다.
- 기존 Classic 화면으로 이동하지 않는다.

### M3. 검수 공방

- AI가 만든 동형 문제를 `ai-draft`로만 저장하고, source-linked/verified 승격은 별도 검수 절차를 거친다.

### M4. 20분 학습 루프

- 렌즈 하나를 끝내면 기억 상태와 다음 복습 시간이 남는다.
- 목표 지표는 방문 수가 아니라 "전공A/B 답안 가능 용어 수"다.

## 13. 수락 기준

이음진 구현 PR은 다음 질문을 통과해야 한다.

1. 이 변경이 전공A/B 답안 가능성을 높이는가?
2. 공식 출처 또는 SEW의 검증된 도메인 자산과 연결되는가?
3. Classic UI/라우트/제품명을 새 서비스에 섞지 않았는가?
4. AI 생성물이 검수 전 공개되지 않는가?
5. 사용자가 클릭하다가 과거 서비스 화면으로 떨어지지 않는가?
6. SEW Classic 운영 배포에 영향을 주지 않는가?

하나라도 실패하면 구현 완료로 보지 않는다.
