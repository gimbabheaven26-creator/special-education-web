# 이음진 격리 원칙

Date: 2026-06-02

## 결정

이음진은 SEW Classic을 고쳐 만든 화면이 아니라, 별도 배포에서 검증할 새 수험지다. 이전 실험 제품의 구조와 문구를 옮겨 심지 않는다.

## 지우는 것

- 이전 실험 제품 라우트
- 이전 실험 도메인 로직
- 이전 실험 세션 루프와 전공A/B 결과판 프로토타입
- 이전 실험 제품의 배포 스크립트와 사용자 검증 문서

## 남기는 것

- 루트 `/`의 이음진 입구
- 기존 특수교육 시험 데이터, 기출 데이터, 용어 데이터, 개념 색인
- 별도 배포를 위한 코드 경계

## 승계 원칙

- SEW Classic의 UI, 라우트, 전역 chrome, 제품명은 승계하지 않는다.
- SEW가 이미 확보한 시험 구조, KICE 기출, NISE 용어, 개념 MDX, 기출-개념 연결 규칙은 공식 출처와 대조한 뒤 이음진의 새 데이터 모델로 재해석한다.
- 시험 목적지와 수락 기준은 `docs/ieumjin-exam-constitution.md`를 따른다.

## 운영 경계

- SEW Classic 운영 프로젝트는 `main`과 `special-education-web.vercel.app`을 유지한다.
- 이음진 검증은 `codex/ieumjin-greenfield` 브랜치와 이음진 이름의 별도 Vercel 프로젝트에서만 한다.
- 이음진 코드를 Classic 운영 브랜치에 합치지 않는다.
- 새 기능은 기존 실험 잔재를 옮기지 말고, 필요할 때 clean-room 구조로 새로 만든다.

## 다음 구현 기준

1. 용어사전 중심 첫 여정을 만든다.
2. 개념, 기출, 동형문제는 새 경로와 새 상태 모델에서 연결한다.
3. 기존 실험 localStorage/sessionId는 이음진 데이터 모델로 승계하지 않는다.
