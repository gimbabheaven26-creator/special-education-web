## 서문: 우리가 여기까지 온 길

2026년 3월, 우리는 8일 만에 155,000줄의 코드를 완성했다.
332개 파일, 118개 커밋. 전통적 팀이라면 8명이 6개월 걸릴 규모다.

이 속도는 축복이었고, 동시에 위험이었다.

Gemini, 시니어 리뷰어, Claude App — 세 개의 독립 리뷰가 서로 다른 버그를 발견했다.
`db.ts` 싱글톤처럼 가장 위험한 문제가 가장 마지막에 발견됐다.
각 리뷰어가 전체를 보지 못했다는 뜻이고, 아직 발견되지 않은 문제가 더 있다는 신호이기도 했다.

우리는 이 사실을 솔직하게 받아들인다.
그리고 그럼에도 불구하고, 이 프로젝트를 계속한다.

---

## 1부. 우려에 대한 응답

### "보안 구멍이 있다"

맞다. API 4개가 인증 없이 열려있고, `db.ts` 싱글톤이 RLS를 무력화하고 있다.
이 두 가지는 사용자를 받기 전에 반드시 닫아야 하는 문이다.

**우리의 입장**: 이것은 프로토타입 속도가 만든 기술 부채다. 이미 T-01~T-03으로 명문화했고,
다음 스프린트의 1순위다. "모르고 있었다"에서 "알고 고친다"로 상태가 바뀌었다. 이것이 성숙이다.

### "콘텐츠 품질을 누가 보장하나"

AI가 생성한 문항이 전문가 검증 없이 제공되는 문제를 Gemini가 가장 날카롭게 짚었다.
임용 수험생 커뮤니티는 좁고, 오류 하나가 퍼지면 신뢰가 무너진다.

**우리의 입장**: 이것이 우리가 가진 가장 큰 자산이자 책임이다.
12년 특수교육 현장 경험이 코드에 새겨져 있다는 건 —
`answer-checker.ts`의 "또는" 복수 정답 처리, `elaboration.ts`의 법령 키워드 보존 로직,
BDS 시나리오의 PBS 4단계 구조 —
외부 개발팀 10명이 고용돼도 만들 수 없는 것이다.
이 도메인 지식이 콘텐츠 검증의 기준이 된다.
540개 KICE 기출이 검증의 닻이고, AI 생성 문항은 점진적 크라우드소싱으로 정제한다.

### "기능을 10개로 줄여야 한다"

여러 리뷰어가 "41개 페이지를 10개 남짓으로 줄이라"고 제언했다.
이 제언에 동의하지 않는다.

**우리의 입장**: 41개 페이지 중 약 1/3은 중복되거나 분산된 기능이 있다는 것을 인정한다.
그러나 해법은 삭제가 아니라 **통합과 정교화**다.

- 중복되는 기능은 하나로 합친다
- 분산된 UX는 일관된 흐름으로 재연결한다
- 각 기능을 더 깊고 완성도 있게 만든다

더 적은 기능이 아니라, 더 잘 작동하는 기능이 목표다.
이것이 앞으로의 로드맵 방향이다.

### "1인 개발이면 버스 팩터가 1이다"

CI/CD가 없고, 에러 모니터링이 없고, 문서화가 부족하다.
창업자가 2주 자리를 비우면 무슨 일이 일어나는지 아무도 모른다.

**우리의 입장**: 이것이 에이전트 팀 체제를 구축한 이유다.
강선생, 클루디, 스미스 프라임이 각자의 메모리와 핸드오프 문서를 유지하며,
세션이 끊겨도 작업이 이어지는 구조를 실험하고 있다. 완벽하지 않지만, 방향은 맞다.

## 2부. 우리가 가진 것

### 시장은 비어있다

특수교육 임용 수험생을 위한 디지털 학습 플랫폼이 없다.
이것은 "시장을 만들어야 하는" 문제가 아니라, "이미 존재하는 수요에 도구가 없는" 상황이다.
연간 약 2만 명의 수험생이 PDF와 카페 게시물로 공부하고 있다.

### 코드는 고칠 수 있지만, 통찰은 복제할 수 없다

`elaboration.ts`에서 숫자 제거 정규식이 "제15조"를 "제 조"로 만들어버리는 버그를 발견한 것 —
이것은 법령 기반 채점이 어떻게 작동하는지 아는 사람만 찾아낼 수 있다.
BDS 시나리오에서 FBA와 PBS를 순서대로 연결한 것,
스페이스드 시나리오에서 DRO 3변형을 정교하게 구현한 것 —
이것들이 이 서비스의 해자(Moat)다.

보안 구멍은 하루면 막는다. 이 통찰은 15년이 걸린다.

### 빠른 실행력은 계속 작동한다

8일에 155,000줄을 만든 속도는 줄이지 않는다. 방향을 바꾼다.
더 빠르게 더 넓게가 아니라, **더 빠르게 더 깊게**.

## 3부. 로드맵: 11월 시험일까지

### Phase 0 — Foundation (4월 1~2주)
**목표**: 사용자를 안전하게 받을 수 있는 상태

```
✅ T-01: API 인증 가드 ×4 (완료 2026-03-22)
✅ T-02: db.ts — createClient() 방식으로 이미 전환됨
✅ T-04~07: 성능 + 시드 버그 + 입력 검증 + 보안 (완료 2026-03-22)
✅ T-08~10: ExamCountdown + elaboration 법령 키워드 + useMounted 훅 (완료 2026-03-22)
⬜ T-03: review-db.ts 대량 삭제 방지 (클루디)
⬜ T-11: display_id 컬럼 (클루디)
⬜ 데이터 품질: ox answer 오류 15건 + 기타 위반 130건 (클루디)
```

> **베타 테스터가 이미 활동 중이다.** 이 단계는 기존 사용자 보호이기도 하다.

> **이미 상당부분 완료**: Zustand→Supabase 양방향 동기화, 게스트→회원 마이그레이션 UI (2026-03-22 완료)

- ~~localStorage → Supabase 5개 스토어 동기화~~ → 완료
- ~~게스트→회원 마이그레이션 (데이터 손실 없음)~~ → 완료

- 완료 기준: 베타 테스터 3명 이상 "폰→PC 이어보기" 확인

### Phase 2 — Consolidation Sprint (5~6월)
**목표**: 중복 기능 통합 + 핵심 루프 정교화

1/3에 해당하는 중복·분산 기능을 진단하고 통합한다.

```
통합 방향:
- 학습 통계 관련 다중 페이지 → 하나의 일관된 대시보드
- 분산된 복습 진입점 → 단일 루프로 재연결
- 중복 UI 패턴 → 공통 컴포넌트로 수렴
```

- PWA + 오프라인 캐시 (지하철에서 작동)
- 복습 리마인더 푸시 알림
- 완료 기준: D-7 리텐션 15% 달성 (교육앱 업계 평균), 중복 페이지 1/3 통합
  - *30%는 상위 10% 수준 — 베타 사용자 피드백 1달 후 재조정*

**이 단계에서 새 페이지를 만들지 않는다. 통합과 정교화만.**

### Phase 3 — Data Trust (7~8월)
**목표**: 콘텐츠를 믿을 수 있게 된다

- KICE 기출 540개를 검증의 기준으로 공개
- AI 생성 문항에 "검토 중" 배지 시스템
- 사용자 오류 신고 → 포인트 지급 크라우드소싱
- 완료 기준: 신고된 오류 72시간 내 수정율 95%

### Phase 4 — Exam Sprint (9~10월)
**목표**: 시험 2개월 전, 수험생의 마지막 루틴이 된다

- D-30 / D-7 커리큘럼 자동 생성
- KICE 모의고사 타이머 강화
- 취약 과목 집중 드릴
- 합격 확률 시뮬레이션 (mastery.ts simulatePass 활용)

### Phase 5 — After November (11월 이후)
**목표**: 합격한 교사가 현직에서도 이 서비스를 쓴다

- 현직 특수교사 교원 연수 콘텐츠
- BDS 시나리오 → NISE/교육청 라이선스 검토 (B2G)
- 2027년 수험생을 위한 플랫폼 재개

**이 단계는 지금 설계하지 않는다. 11월 시험일 이후에 결정한다.**

## 4부. 우리가 배운 것

### 개발 과정의 교훈

**"빠른 프로토타이핑은 숨겨진 부채를 만든다"**
8일 속도는 많은 결정을 검토 없이 통과시켰다. `db.ts` 싱글톤이 대표적이다.
세 번의 리뷰가 모두 놓쳤다. 앞으로: 아키텍처 변경은 반드시 `/plan` 먼저, 승인 전 코딩 없음.

**"다양한 리뷰어가 다른 것을 본다"**
Gemini는 기능적 결함을, 클로딩 평가는 시스템 부채를, Claude App은 아키텍처 패턴을 잡아냈다.
단일 리뷰어를 신뢰하지 않는다. 교차 검증이 원칙이다.

**"창시자의 반박이 때로 AI보다 옳다"**
`generateStaticParams`의 빌드 타임 파일 읽기를 AI가 잘못 지적했을 때,
창시자가 프레임워크를 정확히 이해하고 방어했다.
도메인과 기술을 모두 아는 사람의 판단이 최종 기준이다.

**"AI 제언은 필터링해야 한다"**
"41개 페이지를 10개로 줄여라"는 제언은 기술적으로는 합리적이지만
제품 방향성과 맞지 않았다. AI의 제언은 입력이고, 결정은 창시자가 한다.

**"컴팩션은 세션 종료가 아니다"**
작고 당연해 보이지만, 이것 때문에 Notion 자동화가 3주간 작동하지 않았다.
인프라는 가정하지 말고 검증한다.

### 에이전트 협업의 교훈

강선생은 빠르게 실행하고, 클루디는 데이터를 검증하고, 스미스 프라임은 방향을 잡는다.
이 역할 분리가 흔들릴 때 품질이 흔들렸다. 역할 경계를 지킨다.
