# SEW Next Live Smoke + User Validation Guide

작성: 2026-05-23 / 루멘

## 결론

배포 URL `https://special-education-web.vercel.app` 기준으로 SEW Next 핵심 루프가 실제 브라우저에서 렌더링된다. HTTP 200만 본 것이 아니라 Playwright로 DOM 렌더링, 핵심 문구, 콘솔 에러, page error를 함께 확인했다.

## 라이브 Smoke 결과

| Route | Status | 확인한 화면 신호 | Console/Page Error |
| --- | ---: | --- | --- |
| `/next` | 200 | `SEW Next`, `2027 특수교육 임용 Readiness`, `오늘의 처방` | 0 / 0 |
| `/next/qbank` | 200 | `SEW Next Qbank`, `실제 DB 문제은행`, `대표 문항` | 0 / 0 |
| `/next/practice?mode=mock` | 200 | `SEW Next Practice`, `Mock Exam Drill`, `Mock timer`, `영역 배분` | 0 / 0 |
| `/record` | 200 | `기록` | 0 / 0 |

## 실제 사용자 검증 시나리오

1. Readiness 첫인상
   - 사용자가 `/next` 진입 후 10초 안에 "오늘 무엇을 하면 되는지" 말할 수 있는지 본다.
   - 성공 신호: readiness 수치보다 `오늘의 처방`과 추천 이유를 먼저 설명한다.
   - 위험 신호: "이 점수가 무슨 뜻인지 모르겠다", "기존 홈/퀴즈와 차이가 없다"는 반응.

2. Qbank 자기주도 구성
   - `/next/qbank`에서 domain/difficulty/format을 바꾸고 `커스텀 세션 시작`까지 진행하게 한다.
   - 성공 신호: 사용자가 필터를 "내 약점/시험 대비 목적" 언어로 해석한다.
   - 위험 신호: 필터가 시험 과목 체계와 연결되지 않거나, 대표 문항이 신뢰를 주지 못한다.

3. Practice 피드백 루프
   - adaptive/custom 세션에서 2문항을 풀고 해설, AI Answer Coach, 세션 요약을 읽게 한다.
   - 성공 신호: 사용자가 "다음에 다시 볼 부분"을 스스로 말한다.
   - 위험 신호: 해설은 읽지만 다음 행동이 떠오르지 않는다.

4. Mock Exam 리포트
   - `/next/practice?mode=mock`을 끝까지 진행하고 영역별 리포트를 보게 한다.
   - 성공 신호: 사용자가 점수보다 `영역별 결과`, `함정 선지`, `시간 관리`를 보고 다음 학습을 정한다.
   - 위험 신호: 실제 임용고시 모의고사라고 느끼기에는 문항 수, 시간 압박, 영역 배분이 약하다는 반응.

5. Record 재방문 동기
   - 세션 완료 후 `/record`로 이동해 최근 SEW Next 흐름을 확인하게 한다.
   - 성공 신호: 사용자가 "내 실력이 쌓이고 있다"는 감각을 받는다.
   - 위험 신호: 기록이 단순 활동 로그로 보여서 다음 학습 행동을 만들지 못한다.

## 관찰 지표

| 지표 | 좋은 방향 |
| --- | --- |
| 첫 처방 이해 시간 | `/next` 진입 후 10초 이내 |
| Qbank 필터 완주율 | 필터 변경 후 custom session 시작 |
| 해설 체류 | 정오답 확인 직후 이탈하지 않고 coach/복습 예약까지 확인 |
| 세션 후 재방문 | practice 완료 후 `/record` 또는 `/next`로 이동 |
| 신뢰 발화 | "기출/각론/영역이 맞다", "내 약점에 맞다"는 표현 |

## 다음 제품 판단 기준

- 사용자 3명 이상이 Qbank를 "내가 시험 직전에 골라 풀 도구"로 이해하면 Supabase 영속 저장과 개인화 추천을 우선한다.
- Mock Exam에서 "진짜 모의고사 느낌이 약하다"는 피드백이 2명 이상 나오면 문항 수, 제한 시간, 답안 검토, 리포트 밀도를 먼저 키운다.
- Readiness 수치가 납득되지 않는다는 반응이 반복되면 점수 산식보다 근거 설명, 최근 기록, 오답 위험 영역을 먼저 드러낸다.
- Record가 다시 학습하게 만들지 못하면 "기록 화면"이 아니라 "다음 처방을 생성하는 화면"으로 재설계한다.

## 후속 구현 후보

1. SEW Next 세션 결과를 Supabase에 저장해 기기 간 continuity를 확보한다.
2. Mock Exam을 실제 시험형 묶음으로 확장하고 제한 시간/영역별 배점을 도입한다.
3. Readiness 산식에 기출 연도, 각론 영역, 최근 망각 간격을 반영한다.
