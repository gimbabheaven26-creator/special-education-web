# 이음진 First Screen UX Review

Date: 2026-06-06
Target: https://ieumjin.vercel.app/
Scope: 공개 Production 첫 화면, 모바일/데스크톱 첫인상, 첫 클릭 도착 경험

## 한 줄 판단

이음진은 이름과 세계관의 씨앗은 살아났지만, 현재 첫 화면은 아직 "새로운 학습 서비스"가 아니라 "분위기 좋은 입구 화면"에 머문다. 살릴 것은 브랜드 방향과 용어 중심 학습 축이고, 버릴 것은 현재의 정적인 카드 배치와 Classic으로 끊기는 첫 클릭이다.

## Evidence

- Desktop 1440x1000: `/` HTTP 200, title/H1 `이음진`, console/page error 0.
- Mobile 390x844: `/` HTTP 200, title/H1 `이음진`, console/page error 0.
- Root text scan: `SEW Next` 0, root의 `특수교육 공부방` 0.
- Mobile layout: body height 2012px, 첫 viewport는 전체의 42%만 표시. `오늘의 이음 시작` CTA는 y=1811px에 있어 첫 화면에 보이지 않는다.
- First click destination: `/terms?q=기능적 행동평가` HTTP 200, console/page error 0이지만 화면과 상단 chrome은 `특수교육 공부방` Classic으로 전환된다. 도착 화면에는 `이음진`이 없다.

## 살릴 것

1. `이음진`이라는 이름
   - 지명/공간 감각이 있고, 기존 Classic 이름과 충분히 분리된다.
   - 기능명 조합보다 기억 가능성이 높다.

2. 어두운 수험지 톤
   - Classic의 밝은 학습 도구 느낌과 분리된다.
   - "입장한다"는 감각이 생긴다.

3. 용어를 첫 단위로 잡은 것
   - 사용자가 양보하지 않겠다고 한 핵심과 맞다.
   - 특수교육 임용에서 용어는 개념, 법령, 기출 선지, 서술형 답안으로 이어지는 좋은 출발점이다.

4. 용어당 → 개념교 → 기출진 → 출제공방의 뼈대
   - 서비스의 장기 루프가 한 줄로 설명된다.
   - 다만 지금은 버튼 이름 수준이라 실제 학습 이점으로 번역해야 한다.

5. `/next` 제거와 독립 배포
   - 과거 실험의 맛을 재학습하지 않게 하는 결정은 맞다.
   - 이 원칙은 계속 유지해야 한다.

## 버릴 것

1. 현재 모바일 첫 화면 구조
   - 핵심 CTA가 첫 화면에 없다.
   - 사용자는 "오늘 무엇을 하면 되는지"보다 "아래에 뭐가 더 있나"를 먼저 느낀다.

2. 데스크톱의 큰 빈 공간
   - H1 아래부터 카드까지 여백이 너무 커서 긴장감이 빠진다.
   - 수험 서비스라기보다 전시용 포스터처럼 보인다.

3. 오늘의 통행 용어 3장 + 하단 4개 관문의 중복
   - 같은 레벨의 카드가 두 번 나온다.
   - 사용자는 "용어 카드부터 누를까, 용어당부터 누를까"를 고민한다.

4. 첫 클릭 후 Classic으로 바뀌는 경험
   - 가장 큰 신뢰 손상 지점이다.
   - 이음진에서 출국 수속을 마쳤는데 바로 기존 학교 건물로 들어가는 느낌이다.

5. `출제공방`이 `/admin/ai-generate`로 연결되는 구조
   - 사용자가 보는 서비스의 핵심 기능이어야 하는데 관리자 도구 냄새가 난다.
   - "AI 동형문제"는 별도 사용자용 표면으로 만들어야 한다.

6. 현재 카피의 추상성
   - "개념의 다리를 건너고"는 분위기는 좋지만 성적 향상 약속은 약하다.
   - 수험생은 첫 화면에서 "몇 분 안에 무엇이 나아지는지"를 알고 싶어 한다.

## P0 / P1 / P2

### P0: 전환 차단

- 모바일 첫 화면에 시작 CTA가 없다.
- 첫 클릭 후 Classic chrome으로 전환된다.
- `출제공방`이 사용자용 기능이 아니라 admin route로 연결된다.

### P1: 핵심 약속 약함

- "용어에서 출제로"라는 방향은 좋지만 오늘 수행할 미션이 없다.
- 망각 곡선, 기출 빈도, 오답, 동형문제 같은 학습 이점이 첫 화면에 없다.
- 첫 화면이 사용자의 최근 상태를 기억하지 않는다. 게스트라도 "오늘은 기능적 행동평가 7분 코스"처럼 즉시 시작해야 한다.

### P2: 시각/문구 다듬기

- 데스크톱 우측 패널은 공간은 좋지만 정보 밀도가 낮다.
- 카드 border와 색 대비가 차분하지만, 오래 머물 이유를 만들 만큼 생동감은 없다.
- `용어당`, `개념교` 같은 내부 지명은 좋지만 첫 방문자는 보조 설명 없이는 행동을 예측하기 어렵다.

## 바로 바꿀 방향

1. 첫 화면을 "입진증"이 아니라 "오늘의 이음권"으로 바꾼다.
   - 한 용어, 예상 소요 시간, 기출 연결 수, 동형문제 생성 가능 여부를 첫 화면에 노출한다.
   - 예: `기능적 행동평가 7분 이음권`, `용어 1개 → 개념 3개 → 기출 선지 5개 → 동형 2문항`.

2. 모바일 첫 viewport 안에 CTA를 넣는다.
   - H1, 한 줄 약속, 오늘 미션 카드, 시작 버튼까지 844px 안에 들어와야 한다.
   - 오늘의 용어 3장은 아래로 내리거나 한 장만 대표로 남긴다.

3. 첫 클릭 도착지를 이음진 내부 표면으로 만든다.
   - `/terms`를 그대로 쓰더라도 이음진에서 진입한 경우 이음진 chrome/상단 미션 바를 유지한다.
   - 더 낫게는 `/ieum/terms/[term]` 같은 clean-room route를 만든다.

4. `출제공방`은 사용자용 생성 흐름으로 분리한다.
   - admin route 연결은 제거한다.
   - 첫 버전은 실제 생성 전이라도 `기출 근거 선택 → 변형 조건 → 초안 보기` 형태의 안전한 사용자 경험으로 시작한다.

5. 첫 화면을 "읽는 화면"이 아니라 "시작하는 화면"으로 바꾼다.
   - 사용자가 10초 안에 누를 버튼은 하나여야 한다.
   - 나머지는 보조 경로다.

## Recommended Cut

현재 화면에서 유지할 것은 다음뿐이다.

- `이음진` 이름
- 어두운 수험지 톤
- 용어 중심
- 용어 → 개념 → 기출 → 동형문제 구조
- `용어당`, `개념교`, `기출진`, `출제공방` 같은 공간명 후보

현재 카드 배치, 우측 패널 위치, 첫 클릭 Classic 연결, admin route 연결은 버린다. 다음 구현은 "기능적 행동평가 7분 이음권" 하나를 모바일 첫 화면 안에 완성하는 것부터 시작하는 편이 맞다.
