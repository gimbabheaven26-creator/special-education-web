# 이음진 FBA 용어 렌즈 설계

Date: 2026-06-08
Status: Approved direction from Kairan; design draft for implementation planning

## 1. 왜 여기서 시작하는가

이음진의 첫 감각은 전체 앱 구조가 아니라 `기능적 행동평가` 하나에서 시작한다.

이 용어는 다음 조건을 모두 만족한다.

- NISE 용어 데이터에 `기능적 행동평가` 항목이 있다.
- SEW 개념 MDX에 FBA, PBS, ABC 관찰, 가설 설정, 행동지원계획이 이미 정리되어 있다.
- KICE 원본 기출에 직접 연결되는 강한 문항이 있다. 특히 2019 전공A 14번은 `긍정적 행동지원`, `기능적 행동평가`, `MAS`, `ABC 관찰`, `교체기술`, `대체행동`을 한 문항에 묶는다.
- 최근 예측/분석 문서에서도 행동지원, 기능 분석, ABC, 차별강화가 반복적으로 등장한다.

따라서 FBA 하나를 완성하면 이음진의 기본 구조를 검증할 수 있다.

```text
공식 출처
→ 용어 씨앗
→ 용어 렌즈
→ 20분 학습
→ 검수·갱신
```

## 2. 용어 정리

### 공식 출처

서비스가 믿는 원본이다.

현재 단계에서는 기존 SEW 자료만 사용한다.

- `data/terminology/nise-dictionary.json`
- `data/kice-기출/`
- `src/content/concepts/`
- `docs/kice-structure-analysis.md`
- `docs/kice-keyword-weight-analysis.md`

카이란이 새로 확보할 특수교육 용어사전 책은 이후 `source pack`으로 추가한다. 지금 설계는 그 책을 받아들일 수 있어야 하지만, 지금 작업을 그 책에 의존하지 않는다.

### 용어 씨앗

용어 씨앗은 화면이 아니다. DB 테이블명도 아니다. 이음진에 어떤 용어를 들여오기 위한 최소 자격 요건이다.

`TermLensSeed`라는 개발자식 이름은 내부 코드에서 필요할 때만 쓰고, 제품 언어로는 `용어 씨앗`이라고 부른다.

용어 씨앗이 가져야 할 최소 내용:

- 대표 용어: 기능적 행동평가
- 별칭: FBA, 기능평가, functional behavior assessment
- 공식 정의 후보
- 출처: NISE doc_id, 기존 개념 파일, KICE 문항
- 관련 개념: PBS, ABC 관찰, 기능 분석, 가설 설정, 행동지원계획, 대체행동
- 관련 기출: 2019 전공A 14번, 2021 전공A 10번, 2025 전공A 11번, 2026 전공A 3번 등
- 검수 상태: `source-linked` 또는 `verified`

### 용어 렌즈

용어 렌즈는 사용자가 실제로 경험하는 완성 단위다.

용어 씨앗이 원재료라면, 용어 렌즈는 그 원재료를 시험 공부 경험으로 조립한 것이다.

사용자는 렌즈 하나에서 다음을 모두 한다.

1. 용어를 자기 말로 회상한다.
2. 공식 정의를 확인한다.
3. 헷갈리는 개념과 구분한다.
4. 관련 개념을 짧게 연결한다.
5. 실제 기출에서 어떻게 나왔는지 본다.
6. 답안 문장으로 바꾼다.
7. 동형 문제를 푼다.
8. 복습 예약을 남긴다.

### 기출 DNA

기출 DNA는 기출문제의 껍데기 정보가 아니라, 변형 가능한 구조다.

FBA 렌즈에서 2019 전공A 14번의 DNA는 다음처럼 읽는다.

- 교시/번호: 2019 전공A 14번
- 유형: 서술형
- 배점: 4점
- 영역: 행동지원
- 장면: 정서·행동장애 학생 S를 위한 긍정적 행동지원 내용
- 핵심 자료: 문제행동, 기능적 행동평가, 가설 설정, 지원 계획, 평가 계획
- 물음: 교체기술 특성, 대처기술 명칭, 수준 변화 분석, 즉각성 분석
- 변형 축: 행동 기능, 대체행동, ABC 관찰, 단일대상연구 그래프 분석

### 자동 변환 스크립트

자동 변환 스크립트는 기존 서비스를 새 서비스로 복사하는 도구가 아니다.

목표는 `후보 생산`이다.

```text
NISE 용어 + KICE JSON + MDX 개념
→ 용어 씨앗 후보
→ 근거/기출/개념 연결 점수
→ 검수 대기 목록
```

자동 변환 스크립트가 해도 되는 일:

- 같은 용어가 NISE, KICE keyword, MDX kiceKeywords에 함께 있는지 찾기
- 관련 기출 후보를 점수순으로 정렬하기
- 관련 개념 파일 후보를 제안하기
- 출처 누락, 용어 불일치, 오래된 항목을 표시하기

자동 변환 스크립트가 하면 안 되는 일:

- 검수 없이 사용자 화면에 공개하기
- Classic UI나 기존 라우트 구조를 같이 옮기기
- AI가 만든 문항을 verified처럼 취급하기
- 출처 없는 정의를 공식 정의처럼 저장하기

### 공식 출처 감사 스크립트

공식 출처 감사 스크립트는 서비스가 오래되어 틀리는 것을 막는 장치다.

처음에는 자동 크롤러보다 `체크리스트형 lint`로 시작한다.

- 법령 시행일이 문서에 기록되어 있는가?
- KICE 원문 URL이 연결되어 있는가?
- NISE doc_id 또는 새 용어사전 source id가 있는가?
- 교육부/시도교육청 공고 기준 시험 구조가 최신인가?
- `verified`인데 출처가 하나도 없는 항목은 없는가?
- `ai-draft`가 사용자 화면에 노출되지 않는가?

## 4. 첫 화면 경험

FBA 렌즈는 사전 페이지처럼 시작하면 안 된다. 처음 5초 안에 사용자가 할 일이 보여야 한다.

첫 화면 구조:

```text
기능적 행동평가
전공A/B 답안 가능도: source-linked

[3분 회상]
선행사건, 행동, 후속결과라는 세 단어를 써서 FBA를 설명해 보세요.

[공식 정의]
행동의 이유나 목적을 확인하기 위해 선행 사건, 행동, 후속 결과 정보를 수집·분석하는 방법.

[오늘 연결]
2019 전공A 14번: PBS 내용 속 FBA, MAS, ABC 관찰, 교체기술
```

버튼은 기능명이 아니라 다음 행동이어야 한다.

- `정의 말해보기`
- `ABC로 쪼개기`
- `2019 A-14 보기`
- `동형 한 문항 풀기`
- `복습 예약`

## 5. 20분 학습 흐름

```text
0~3분: 정의 회상
3~6분: 공식 정의와 핵심어 확인
6~9분: FBA / 기능 분석 / ABC 관찰 / PBS 구분
9~14분: 2019 전공A 14번 기출 DNA 분석
14~18분: 동형 문제 1개 풀이
18~20분: 답안 문장 저장과 복습 예약
```

사용자는 이 20분 뒤에 다음 문장을 쓸 수 있어야 한다.

> 기능적 행동평가는 문제행동의 기능을 확인하기 위해 선행사건, 행동, 후속결과 정보를 수집·분석하고, 그 결과를 바탕으로 행동지원계획과 대체행동 교수를 설계하는 과정이다.

## 6. 데이터 흐름

### 1단계: 수작업 샘플

먼저 FBA 하나는 손으로 만든다.

이 단계의 목적은 자동화가 아니라 감각 확정이다.

산출물:

- `fba` 용어 씨앗 샘플
- 관련 기출 3~5개 수동 연결
- 관련 개념 2~4개 수동 연결
- 렌즈 화면에 필요한 문장과 카드 순서

### 2단계: 타입과 읽기 전용 인덱스

FBA 샘플이 만족스럽다면 그 모양을 타입으로 고정한다.

타입은 DB 계약이 아니라 앱 내부 read-only 모델로 시작한다.

### 3단계: 자동 변환 후보 생성

그다음 기존 데이터를 훑어서 나머지 용어 후보를 만든다.

자동 변환은 다음 점수를 낸다.

- NISE exact match
- KICE keyword match
- MDX kiceKeywords match
- 최근 기출 출현
- 복합 영역 연결
- 출처 신뢰도

### 4단계: 검수 대기열

자동 변환 결과는 `verified`가 아니라 `source-linked` 또는 `ai-draft`로 들어간다.

카이란이 새 용어사전 책 데이터를 넣으면 이 단계에서 source priority를 다시 계산한다.

## 7. 설계 선택지

### A. 수작업 FBA 먼저

가장 느리지만 가장 정확하다. 이음진의 맛을 정한다.

추천한다.

### B. 자동 변환 먼저

빠르게 많은 후보를 만들 수 있지만, 기존 SEW의 흔적과 노이즈가 같이 들어올 가능성이 높다.

지금은 하지 않는다.

### C. UI 먼저

보기에 빠르게 진전된 것처럼 보이지만, 다시 Classic 표피 변경으로 흐를 위험이 크다.

지금은 하지 않는다.

## 8. 수락 기준

FBA vertical slice는 다음을 통과해야 한다.

1. 기존 SEW Classic 화면으로 이동하지 않는다.
2. NISE 정의 후보와 KICE 기출이 명시적으로 연결된다.
3. 사용자는 20분 안에 FBA의 정의, 구분, 기출 적용, 동형 풀이를 끝낼 수 있다.
4. 동형 문제는 원본 기출 DNA를 따르되 검수 전에는 `ai-draft`로만 존재한다.
5. 새 용어사전 책이 들어와도 `source pack`으로 교체·보강 가능하다.
6. 자동 변환은 공개가 아니라 검수 대기 후보 생성을 목표로 한다.

## 9. 구현 전 결정

다음 구현 계획은 아래 순서로 작성한다.

1. FBA 용어 씨앗 샘플 파일을 만든다.
2. FBA 렌즈를 읽는 작은 도메인 모듈을 만든다.
3. `/terms?q=기능적 행동평가`를 샘플 파일 기반 렌즈로 교체한다.
4. 자동 변환 스크립트는 FBA 렌즈가 만족스러운 뒤 만든다.

이 순서를 바꾸지 않는다.
