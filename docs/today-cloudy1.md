## 4. 작업 경로 (파일 존재 시)

### STEP 1 — 구조 분석 (Opus 권장)
- `벤티) 01.자폐스펙트럼장애.pdf.md` 전체 읽기
- `벤티) 01.자폐스펙트럼장애_quiz.md` (있으면) 병행 읽기 → kiceKeywords 추출
- 섹션 수와 각 섹션의 핵심 개념 목록 작성
- MDX 파일 분할 계획 결정 (정서행동장애는 8개였으나 자폐는 내용에 따라 조정)

### STEP 2 — MDX 생성 (Sonnet으로 전환 가능)
각 파일을 순서대로 생성:

**frontmatter 필수 필드**:
```yaml
---
title: 자폐스펙트럼장애 — {섹션명}
description: {한 줄 요약}
subject: 자폐스펙트럼장애
slug: {영문 or 한글 슬러그}
order: {숫자}
kiceKeywords:
  - {키워드1}
  - {키워드2}
lastUpdated: 2026-03-19
---
```

**본문 규칙**:
- H2(`##`)로 대섹션, H3(`###`)으로 소섹션
- 진단 기준: 숫자, 개수, 기간 **반드시 명시** (예: "12개월 이상", "3개 중 2개")
- Mermaid 다이어그램 → 마크다운 표로 변환
- 비교 대상이 있으면 항상 표로 정리
- `> ⚠️ **기출 포인트**: ...` 블록 — 임용에 자주 나온 개념에 사용
- 의학적 분류(DSM-5 기준) vs 교육적 분류 구분 명시

### STEP 3 — 검증
```bash
cd ~/Projects/special-education-web
npm run build
```
빌드 exit 0 확인 필수.

### STEP 4 — 완료 보고
`~/.claude/notion-pending.json` 작성:
```json
{
  "title": "완료보고: 클루디1 자폐스펙트럼장애 MDX 2026-03-19",
  "type": "세션기록",
  "tags": ["클루디1", "s-e-w", "MDX"],
  "content": "## 완료된 것\n- 자폐스펙트럼장애 MDX {N}개 생성\n- 빌드 exit 0 ✅\n\n## 생성된 파일\n{파일 목록}\n\n## 미결\n{있으면 기록}"
}
```

## 5. 금지 사항

- `docs/contract.md` 수정 금지 — 스미스 프라임에게 요청
- `src/content/concepts/지적장애/`, `학습장애/` 건드리지 않음 (클루디2, 3 영역)
- `data/mdx/`, `scripts/` 수정 금지
- Supabase 삽입 금지 — MDX 파일 생성만 담당 (삽입은 별도 지시 후)
- 빌드 실패 시 수정하지 않고 카이란에게 보고

## 6. 우선순위

1. **블로커 확인** (파일 없으면 즉시 알릴 것) — 가장 먼저
2. **기출 포인트 정확도** — 틀린 내용보다 누락이 낫다
3. **분량** — 정서행동장애 파일 당 평균 200~400줄 참고
4. **빌드 통과** — MDX 문법 오류 없어야 함
