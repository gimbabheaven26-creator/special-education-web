# 장기 비전: 교수용 문제 플랫폼

> 작성: 2026-03-24 | 상태: 아이디어 메모 | Phase: 4+ (장기)

## 핵심 아이디어

전국 특수교육과 교수들이 쪽지시험/중간·기말고사 출제 시 사용할 수 있는 문제 데이터베이스 플랫폼.

## 역할 구조

| 역할 | 우리 플랫폼 | 교수 |
|------|-----------|------|
| 문제 생성 | O (창고) | X (소비자) |
| 문제 선택/편집 | 도구 제공 | 간단한 수정 |
| 피드백 | 수집 | 짧은 코멘트 |
| 학생 배포 | 로컬 권한 개방 | 자기 학생에게 안내 |

**핵심: 교수는 만드는 사람이 아니라 고르는 사람. 우리가 문제를 가지고 있어야 가치가 있다.**

## 문항 특성

- 각론서 내용 기반의 지엽적 문제 (임용시험에는 안 나오는 수준)
- 주로 OX퀴즈, 단답형 (쪽지시험 용도)
- 난이도: 기존 단답형 기준 상급
- 교수가 원하는 영역/주제 단위로 요청 → 플랫폼에서 제공

## 우리가 얻는 것

1. **교수 피드백** — 문항 품질의 최고 검증 시그널
2. **학생 유입** — 교수가 자기 수업에서 플랫폼을 언급/권장
3. **문항 성능 향상** — 지엽 문항이 역으로 OX/단답형 전체 품질을 끌어올림

## 교수 권한 모델: 준관리자

**신분 인증 → 준관리자 권한 부여. 권한 자체가 참여 인센티브.**

교수가 돈이나 포인트가 아니라 "이 플랫폼 안에서의 영향력"을 얻는 구조.
각 학교의 교수들이 적극적으로 서비스에 참여하도록 만드는 핵심 장치.

| 권한 | 일반 사용자 | 교수 (인증 후) |
|------|-----------|--------------|
| 문제 풀기 | O | O |
| 문항 DB 열람/검색 | 제한적 | 전체 |
| 문항 편집/커스터마이즈 | X | O |
| 피드백/코멘트 | X | O |
| 학생 로컬 권한 개방 | X | O (자기 학교 학생) |
| 문항 세트 구성/내보내기 | X | O |
| 통계 열람 | 본인만 | 자기 학생 + 영역별 |

인증 방법: 대학 이메일(.ac.kr), 교원 증명, 학교 홈페이지 재직 확인 등

## 교수가 얻는 것

1. 쪽지시험 문제를 처음부터 안 만들어도 됨
2. 간단한 커스터마이즈만으로 자기 수업에 맞춤
3. 학생들에게 복습 도구 제공 가능
4. 플랫폼 내 준관리자 영향력 (문항 품질에 기여하는 전문가로서의 역할)

## 통계/측정 참고

- 교수 이용은 로컬 권한 개방이므로 서비스 통계에 직접 잡히지 않음
- 간접 지표: 교수 피드백 수, 교수 경유 학생 가입 수

## 연결 포인트

- Phase 2 #6(콘텐츠 비율 정규화)의 장기적 해법
- 지엽 문항 축적 → 기존 OX/단답형 DB 확장
- 양면 네트워크: 교수 ↔ 문제 DB ↔ 수험생

## 미결 사항

- 초기 문항 어떻게 확보? (AI 생성 + 전문가 검수? 기존 각론서 기반?)
- 교수 온보딩 채널 (학회? 개별 접촉? 대학 제휴?)
- 로컬 권한의 기술적 구현 (학교별 서브도메인? 초대 코드?)
- 수익화 시점과 방식
