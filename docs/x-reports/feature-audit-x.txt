## 코어 정의

이 서비스의 코어는 3-Layer 기출 문제 분석 시스템이다:

- Layer 1 (기출 원본): 연도별 ~40문항의 원본 기출 문제 열람, 풀이, 키워드 태깅
- Layer 2 (기출 동형): 기출과 동일한 깊이와 패턴을 다른 각도에서 접근하는 문제
- Layer 3 (기출 예측): 출제 경향 분석, 키워드 빈도, 영역 예측

---

| 코어 지원 | 코어를 간접적으로 강화하는 기능 (학습 추적, 오답 분석 등 — 예측에 피드백 루프를 제공할 잠재력이 있음) |

## 1. 코어 직결 기능

### 1-1. KICE 기출 열람 및 풀이 (/kice)

파일: src/app/kice/page.tsx (L1-73), src/app/kice/KiceClient.tsx, src/app/kice/KiceByArea.tsx, src/app/kice/KiceSearch.tsx
데이터: data/kice-기출/ (2016~2027, 연도별 JSON)
비즈니스 로직: src/lib/kice.ts (L1-105)

이 기능은 Layer 1과 Layer 2를 구현하는 핵심이다. getAvailableExams() 함수가 data/kice-기출/ 디렉토리를 스캔하여 연도별 시험 목록을 반환하고, getExam(year, session)이 개별 시험 JSON을 파싱한다. 기출 원본(전공A.json, 전공B.json)과 동형 문제(전공A-동형.json, 전공B-동형.json), 예상 문제(전공A-예상.json)가 파일명 컨벤션으로 구분되며, isIsomorphic과 isPredicted 플래그로 코드 내에서 분류된다.

KiceClient는 연도/세션 선택, 키워드 필터, 비교 모드(동형 vs 원본 나란히 보기), 1문제씩 연습 모드를 제공한다. KiceByArea는 영역별 기출 분류, KiceSearch는 전체 기출에서 키워드/과목 검색을 제공한다.

이것이 코어 직결인 이유: Layer 1(기출 원본)과 Layer 2(기출 동형)의 데이터 저장/로딩/표시를 직접 구현한다. 현재 2016~2026년 원본 + 2025~2026년 동형 + 2027년 예상이 존재하여 3개 Layer 모두가 데이터로 표현되어 있다.

### 1-2. KICE 모의고사 (/kice/exam)

파일: src/app/kice/exam/page.tsx (L1-39), src/app/kice/exam/ExamClient.tsx, src/app/kice/exam/ExamResultScreen.tsx
컴포넌트: src/components/kice/QuestionCard.tsx, src/components/kice/DialogueBlock.tsx, src/components/kice/ModelAnswers.tsx, src/components/kice/SubItemsBlock.tsx

실제 시험 환경을 시뮬레이션하는 모의고사 모드이다. 기출/동형/예상 시험을 선택하여 타이머와 함께 전체 시험을 풀고 결과를 확인할 수 있다. QuestionCard 컴포넌트가 기출 문항의 복잡한 구조(대화체 지문, 하위 문항, 모범답안)를 렌더링한다.

이것이 코어 직결인 이유: Layer 1/2 기출 문제를 실전과 동일한 조건에서 풀어볼 수 있는 핵심 학습 경로이다.

### 1-3. 출제 경향 분석 (/kice/analytics)

파일: src/app/kice/analytics/page.tsx (L1-7), src/app/kice/analytics/AnalyticsClient.tsx
비즈니스 로직: src/lib/kice-analytics.ts (L1-167)

computeAnalytics() 함수가 원본 기출만 필터링한 뒤 다음 5가지를 계산한다:
1. 과목별 출제 빈도 히트맵 (SubjectFrequency[]): 과목 x 연도 매트릭스
2. 상위 30개 키워드 빈도 (KeywordFrequency[]): 출제 횟수, 등장 연도, 관련 과목, 연속 출제 횟수
3. 3년 이상 연속 출제 키워드 (recentStreakKeywords): 다음 시험 예측의 직접 근거
4. 5년 이상 미출제 키워드 (neverTestedKeywords): "공백 영역" 분석
5. 연도별 요약 (YearSummary[]): 문항 수, 과목/유형 분포, 평균 배점

AnalyticsClient가 히트맵 시각화, 키워드 랭킹, 연속출제 키워드 강조를 UI로 제공한다.

이것이 코어 직결인 이유: Layer 3(기출 예측)의 핵심 엔진이다. 키워드 빈도 분석, 연속 출제 패턴, 공백 영역 탐지가 모두 이 모듈에서 이루어진다. /kice 페이지의 tab=analytics 탭과 별도 /kice/analytics 라우트 양쪽에서 접근 가능하다.

### 1-4. KICE 기출 JSON 데이터 (data/kice-기출/)

위치: data/kice-기출/{year}/{session}.json

2016~2027년 총 12개 연도, 연도당 2~4개 세션 파일(전공A, 전공B, 동형, 예상)이 존재한다. 각 JSON은 KiceExam 타입으로 문항 배열(questions)을 포함하며, 각 문항에 subjects, keywords, points, type, context, subItems 등이 태깅되어 있다.

이것이 코어 직결인 이유: 3-Layer 시스템의 데이터 원천이다. 키워드 태깅이 Layer 3 분석의 품질을 직접 결정한다.

17개 과목 폴더에 걸쳐 MDX 파일로 구성된 개념 학습 시스템이다. 각 MDX 파일의 frontmatter에 kiceKeywords 필드가 있어 기출 키워드와 개념이 연결된다. ChapterTracker 컴포넌트가 학습 완료를 기록하고, BookmarkButton이 북마크를 지원한다. 이전/다음 네비게이션과 LearningTimeline 컴포넌트가 학습 경로를 안내한다.

### 2-2. 과목별 퀴즈 (/quiz)

파일: src/app/quiz/page.tsx (L1-81), src/app/quiz/[subject]/page.tsx, src/app/quiz/ox/page.tsx, src/app/quiz/short/page.tsx

상태 관리: src/stores/useQuizStore.ts (L1-298)

Supabase quiz_questions 테이블에서 과목별/유형별 문제를 가져와 퀴즈를 제공한다. 11개 과목 카드 선택 화면, 과목별 퀴즈 실행, 전체 OX/단답 모아풀기를 지원한다. QuizClient 컴포넌트가 정답 확인, 해설 표시, 오답 자동 등록을 처리한다.

이것이 코어 지원인 이유: 퀴즈 데이터 자체가 기출 분석에서 도출된 핵심 개념을 연습하는 도구이다. 오답 데이터가 useQuizStore에 축적되어 mastery 분석과 취약 영역 식별에 사용되고, 이는 장기적으로 개인화된 예측(Layer 3+)에 활용될 수 있다.

### 2-3. 오답노트 (/wrong-notes)

파일: src/app/wrong-notes/page.tsx (L1-31), src/app/wrong-notes/quiz/page.tsx (L1-25)
상태 관리: src/stores/useQuizStore.ts — wrongNotes[], addWrongNote(), markMastered()

틀린 문제를 과목/챕터별로 분류하여 표시하고, 오답 재시험(wrong-notes/quiz)으로 반복 학습을 제공한다. 최대 500개 오답을 저장하며 마스터 표시로 완료 관리가 가능하다. 오답 등록 시 useLeitnerStore에 자동으로 복습 카드가 생성된다 (useQuizStore.ts L117-129).

이것이 코어 지원인 이유: 기출 문제를 풀다 틀린 것들이 여기에 모이고, 이 데이터가 마스터리 계산과 취약 영역 분석에 직접 사용된다. 기출 기반 학습 효과를 측정하는 핵심 피드백 루프이다.

### 2-4. 마스터리 트리 (/mastery)

파일: src/app/mastery/page.tsx (L1-352)
비즈니스 로직: src/lib/mastery.ts (L1-218)
데이터: src/data/exam-structure.json — subjectWeights

calculateChapterMasteries()가 퀴즈 히스토리와 오답노트를 분석하여 챕터별 숙련도(not_started → learning → practicing → proficient → mastered)를 계산한다. calculateSubjectMasteries()가 이를 과목 단위로 집계하고, simulatePass()가 과목별 가중치(exam-structure.json의 배점 비율)를 적용하여 80점 만점 기준 예상 점수와 합격 가능성을 시뮬레이션한다. 합격선은 56점(70%)으로 설정되어 있다.

이것이 코어 지원인 이유: Layer 3 예측의 "이 학생이 현재 어디에 있는가"를 정량화한다. 과목별 숙련도와 합격 시뮬레이션은 기출 분석의 결과를 학습자의 현재 상태에 매핑하는 핵심 모듈이다.

## 3. 후순위 기능

### 3-1. 커뮤니티 문제 (/community)

파일: src/app/community/page.tsx (L1-22), src/app/community/create/page.tsx, src/app/community/mine/page.tsx, src/app/community/[id]/page.tsx
비즈니스 로직: src/lib/community-db.ts (L1-115)
API: src/app/api/community/route.ts, src/app/api/community/[id]/vote/route.ts, src/app/api/community/[id]/report/route.ts

수험생이 직접 문제를 출제하고 공유하는 소셜 기능이다. Supabase community_questions 테이블에 저장되며, 투표(좋아요), 신고, 공식 채택 기능이 있다.

이것이 후순위인 이유: 사용자 생성 콘텐츠(UGC)는 3-Layer 기출 분석 시스템과 직접적 연관이 없다. 커뮤니티 문제는 기출 데이터의 체계적 분석이 아닌 사용자 간 사회적 학습에 해당한다.

코어 직결이 되려면: 커뮤니티에서 "공식 채택"된 문제가 Layer 2(동형 문제) 풀에 자동 편입되고, 커뮤니티 문제의 키워드가 Layer 3 분석에 포함되어야 한다. 즉, 커뮤니티가 동형 문제 생성의 크라우드소싱 채널로 작동하는 구조가 필요하다.

### 3-2. BDS 행동 의사결정 시뮬레이터 (/scenarios)

파일: src/app/scenarios/page.tsx (L1-129), src/app/scenarios/[id]/page.tsx, src/app/scenarios/spaced/[groupId]/page.tsx
데이터: src/data/scenarios/ (15개 시나리오 파일)
비즈니스 로직: src/lib/spaced-scenario.ts (L1-90)
상태: src/stores/useStudyStore.ts — scenarioProgress, spacedScenarioSchedules

분기형 시나리오(FBA, PBS, 통합교육 등)를 체험하는 시뮬레이터이다. 단일 시나리오와 스페이스드(간격 반복) 시나리오 두 가지 모드가 있다. 행동지원, 통합교육, 진단평가, 교육과정, 전환교육, 관련 법령 등 7개 과목에 걸쳐 난이도별 시나리오를 제공한다.

이것이 후순위인 이유: 시나리오 시뮬레이션은 기출 문제의 "분석"과 다른 축의 학습 도구이다. 기출 분석이 "무엇이 출제되는가"를 다룬다면, 시나리오는 "실제 교실에서 어떻게 판단하는가"를 다룬다.

코어 직결이 되려면: 시나리오의 주제가 기출 키워드 분석에서 도출되어야 하고, 기출 분석에서 "이 키워드 관련 시나리오 연습하기"로 직접 연결되는 경로가 필요하다.

## 6. 핵심 발견 사항

### 6-1. Layer 3 (예측) 의 현재 수준

kice-analytics.ts의 computeAnalytics()는 과목별 빈도, 키워드 빈도, 연속 출제, 공백 영역을 계산하는 탄탄한 엔진이다. 그러나 분석 결과가 학습 경로에 피드백되는 루프가 약하다. 구체적으로:

- 분석 결과에서 "이 키워드를 공부하러 가기" 링크가 개념학습으로 연결되지 않음
- 예상 문제(data/kice-기출/2027/전공A-예상.json)가 존재하지만 분석 결과에서 자동 생성된 것이 아닌 수동 작성물임
- 마스터리 시뮬레이션이 기출 분석의 과목 가중치(exam-structure.json)를 사용하지만, 키워드 단위의 세밀한 연결은 없음

### 6-2. Layer 1/2 데이터 품질

data/kice-기출/ 에 2016~2027년 데이터가 있으며, 2025~2026년에 동형 문제가 존재한다. 2027년에는 예상 문제만 있다. 각 문항에 subjects, keywords, points, type이 태깅되어 있어 분석 엔진의 입력 데이터 구조는 갖추어져 있다.

### 6-3. 코어 지원 → 코어 직결 승격 후보

마스터리 트리가 가장 강력한 승격 후보이다. 현재 simulatePass()가 과목별 가중치를 사용하여 합격 확률을 계산하는데, 여기에 기출 키워드 분석의 "이 키워드가 3년 연속 출제됐는데 학습자가 관련 챕터를 아직 안 풀었다"는 경고를 추가하면 Layer 3 예측과 개인 학습 상태가 직접 연결된다.

### 6-4. 후순위 중 잠재력 높은 기능

온보딩의 학습 계획 생성기(study-planner.ts)가 computeAnalytics()의 결과를 입력으로 받으면, "기출 분석 기반 맞춤형 학습 계획"이라는 강력한 가치 제안이 만들어진다. 현재는 단순 과목 배분만 하지만, 기출 빈도 기반 우선순위 조정은 구조적으로 어렵지 않다.

## 7. 다음 세션 에이전트를 위한 지침

이 문서를 읽는 다음 에이전트에게:

1. 코어 직결 기능 수정 시 src/lib/kice.ts, src/lib/kice-analytics.ts, data/kice-기출/ 을 반드시 함께 확인하라. 데이터 구조 변경이 분석 엔진에 직접 영향을 미친다.

3. 상태 관리 전체 구조: 5개 Zustand 스토어(study, quiz, leitner, bookmark, onboarding)가 localStorage에 persist되고 Supabase user_data에 sync된다. 스토어 간 의존성은 useQuizStore → useLeitnerStore (오답 시 자동 카드 생성) 한 건이다.

4. KICE 데이터 확장 시 data/kice-기출/{year}/{session}.json 형식을 유지하되, 파일명에 동형 또는 예상이 포함되면 isIsomorphic/isPredicted 플래그가 자동 설정된다. computeAnalytics()는 원본만 필터링하므로 동형/예상 문제는 분석에 포함되지 않는다.

5. 테스트: src/lib/__tests__/ 에 단위 테스트가 존재한다. 기출 관련 변경 시 테스트도 확인하라.
