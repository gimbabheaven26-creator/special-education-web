## 1. 전역 패턴 문제

### 문제점

**A. 에러 메시지가 전부 동일 구조 — 냉정하고 기술적**

모든 error.tsx 파일이 "X를 불러올 수 없습니다 / X 데이터를 가져오는 중 오류가 발생했습니다 / 다시 시도 / 홈으로" 패턴을 반복합니다. 수험생이 느끼는 것은 "내가 뭔가 잘못한 건가?", "데이터가 날아간 건가?"라는 불안입니다.

**B. 빈 상태(empty state) 메시지가 방향을 주지 않는 곳이 많음**

"데이터가 없습니다" 계열의 메시지가 사용자에게 "그래서 뭘 해야 하는데?"라는 질문을 남깁니다.

**C. Not Found 페이지가 모두 차갑고 방향 없음**

"찾을 수 없습니다 / 존재하지 않습니다 / 돌아가기" — 사용자가 왜 여기에 왔는지, 대안이 뭔지 알려주지 않습니다.

**D. 로딩 상태가 무음 — 기다리는 동안 불안**

skeleton만 보여주고, 무엇을 기다리는 중인지 알려주지 않습니다.

### 개선안 (전역 적용)

에러 메시지 템플릿:
```
제목: 잠깐, 연결이 불안정해요
본문: 인터넷 연결을 확인하고 다시 시도해주세요. 학습 기록은 안전하게 보관되어 있으니 걱정 마세요.
버튼1: 다시 시도해볼게요
버튼2: 홈으로 돌아가기
```

Not Found 템플릿:
```
제목: 앗, 여기는 비어 있네요
본문: 찾으시는 페이지가 이동되었거나 삭제되었을 수 있어요. 아래에서 원하는 기능을 찾아보세요.
(관련 페이지 3개 추천 링크)
```

## 3. 로그인

`src/app/login/page.tsx`, `src/components/auth/LoginForm.tsx`, `src/components/auth/SocialLoginButtons.tsx`

### 현재 상태
- 제목: "특수교육 공부방"
- 부제: "학습 데이터를 기기 간에 동기화하세요"
- 에러: "로그인 중 오류가 발생했습니다. 다시 시도해주세요."
- 게스트 링크: "로그인 없이 계속하기"
- 카카오 버튼: "카카오로 계속하기" (비활성, 툴팁 "준비 중")

### 문제점
1. **"학습 데이터를 기기 간에 동기화하세요"** — 기술적 언어. 수험생에게 "동기화"보다는 "어디서든 이어서 공부"가 와닿음
2. **"로그인 없이 계속하기"** 위치가 맨 아래로 밀려 있음 — 게스트 사용도 환영한다는 느낌이 부족
3. **LoginForm 에러 메시지**: "이메일 또는 비밀번호가 올바르지 않습니다" — 어느 쪽이 틀렸는지 힌트 없음 (보안상 의도적이지만, 사용자 안내 추가 가능)
4. **"확인 이메일을 보냈습니다. 이메일을 확인해주세요."** — 어디로 보냈는지 이메일 주소 표시하면 더 친절
5. **비밀번호 placeholder "6자 이상"** — 최소 조건만 표시, 강도 힌트 없음

### 개선안

| 위치 | 현재 | 제안 |
|------|------|------|
| `login/page.tsx:29` 부제 | `학습 데이터를 기기 간에 동기화하세요` | `어디서든 이어서 공부할 수 있어요` |
| `login/page.tsx:35` 에러 | `로그인 중 오류가 발생했습니다. 다시 시도해주세요.` | `로그인에 실패했어요. 잠시 후 다시 시도해주세요.` |
| `login/page.tsx:59` 게스트 | `로그인 없이 계속하기` | `로그인 없이 바로 공부 시작하기` |
| `LoginForm.tsx:34` 에러 | `이메일 또는 비밀번호가 올바르지 않습니다.` | `이메일 또는 비밀번호를 다시 확인해주세요. 회원가입을 안 하셨다면 위에서 가입할 수 있어요.` |
| `LoginForm.tsx:44` 성공 | `확인 이메일을 보냈습니다. 이메일을 확인해주세요.` | `${email}로 확인 메일을 보냈어요. 메일함을 확인해주세요. (스팸함도 확인해보세요!)` |
| `SocialLoginButtons.tsx:47` 구글 | `Google로 계속하기` | `Google 계정으로 시작하기` |

## 4. 온보딩

`src/app/onboarding/page.tsx`

### 현재 상태
- 5단계 플로우: 시험일 → 학습 수준 → 취약 과목 → 하루 목표 → 확인
- 대체로 잘 설계되어 있음. 레벨 옵션에 이모지와 설명이 있고 따뜻한 톤

### 문제점
1. **"다음" 버튼이 모든 스텝에서 동일** — 다음 스텝에서 뭘 할 건지 예고하지 않음
2. **취약과목 "모르겠어요 (건너뛰기)"** — 좋은 UX지만, "모르겠어도 괜찮아요, 학습하면서 자동 파악돼요" 같은 안심 메시지 없음
3. **최종 확인 스텝 "학습 계획을 만들까요?"** — "만들까요?"보다 기대감 있는 문구가 나을 수 있음
4. **하루 목표 선택에서 "꾸준히 유지할 수 있는 양을 선택하세요"** — 좋지만, 추천 마크(예: "대부분의 합격자가 선택") 없음

### 개선안

| 위치 | 현재 | 제안 |
|------|------|------|

## 5. 오늘 학습 (Daily)

`src/app/daily/page.tsx`

### 현재 상태
- 3단계 학습: OX → 단답 → 서술
- 로딩: spinner만 (텍스트 없음)
- 에러: "문제를 불러오지 못했습니다." + "다시 시도"
- 완료 화면: "{타임슬롯} 학습 완료!" + 점수 표시

### 문제점
1. **로딩 상태가 무음** — spinner만 있고 "문제를 준비하고 있어요..." 같은 텍스트 없음 (`daily/page.tsx:303`)
2. **에러 상태가 최소** — "문제를 불러오지 못했습니다"만 있음. 홈으로 갈 수 있는 링크도 없음 (`daily/page.tsx:311-317`)
3. **완료 화면 "최소 경로 달성"** — 기술적 용어. 사용자에게 의미 없음 (`daily/page.tsx:334`)
4. **OX 정답률이 낮을 때 감성 메시지 없음** — 0%여도 동일한 "학습 완료!" 표시
5. **Step 간 전환 안내 부족** — "단답형으로 넘어가기" 버튼은 있으나, "잘했어요! 다음은 단답형이에요" 같은 전환 메시지 없음
6. **"채점하기" 버튼 라벨** — 단답형은 "답안 확인", 서술형은 "정답 키워드 확인"으로 각각 다름. 일관성은 있으나 "채점하기"가 긴장감을 줄 수 있음
7. **placeholder "답을 작성해보세요..." / "답을 입력해보세요..."** — 좋은 톤이지만 "핵심 키워드를 포함해서 작성해보세요" 같은 가이드가 더 나음

### 개선안

| 위치 | 현재 | 제안 |
|------|------|------|
| `daily/page.tsx:303-306` 로딩 | `<Loader2>` (무음) | `오늘의 문제를 준비하고 있어요...` (Loader2 아이콘 + 텍스트) |
| `daily/page.tsx:312` 에러 텍스트 | `문제를 불러오지 못했습니다.` | `문제를 불러오지 못했어요. 인터넷 연결을 확인하고 다시 시도해주세요.` |
| `daily/page.tsx:334` 부제 | `OX 10 + 단답 5 + 서술 3 — 최소 경로 달성` | `OX {OX_COUNT} + 단답 {FILL_IN_COUNT} + 서술 {DESCRIPTIVE_COUNT}문제를 모두 풀었어요!` |
| `daily/page.tsx:332` 완료 제목 | `{TIMESLOT.label} 학습 완료!` | 정답률에 따라 분기: >=80% `훌륭해요! {TIMESLOT.label} 학습 완료!` / >=50% `잘 하고 있어요! {TIMESLOT.label} 학습 완료!` / <50% `{TIMESLOT.label} 학습을 끝까지 해내셨어요! 반복하면 분명 올라갈 거예요.` |
| `daily/page.tsx:158` 서술 placeholder | `답을 작성해보세요...` | `핵심 키워드를 포함하여 서술해보세요...` |
| `daily/page.tsx:443` 채점 버튼 | `채점하기` | `정답 확인하기` (덜 긴장됨) |
| `daily/page.tsx:496` 안내문 | `문제를 읽고 답을 생각해보세요. 답안 확인 후 자기 채점합니다.` | `먼저 답을 적어보고, 정답과 비교해보세요. 스스로 채점하는 것이 기억에 더 남아요.` |

3. **"챕터 N개"** — 문제 수가 아닌 챕터 수만 보여줌. 문제가 몇 개인지, 예상 소요 시간이 없음

## 7. OX 퀴즈 / 단답형

`src/app/quiz/ox/page.tsx`, `src/app/quiz/short/page.tsx`

### 현재 상태
- metadata: "OX 퀴즈" / "단답형"
- description: "전 과목 OX 문제 모아 풀기" / "전 과목 단답형 문제 모아 풀기"
