# 투자 리뷰: 특수교육 공부방

리뷰어: X (EdTech VC 시뮬레이션, 200+ 스타트업 평가 경험)
일자: 2026-03-24
대상: 특수교육 임용고시 AI 학습 플랫폼
창업자: 카이란 (현직 특수교사 10년차, 비전업 개발자)

---

> 카이란님, 먼저 솔직히 말씀드리겠습니다. 200개 넘는 에듀테크 스타트업을 봤지만, 이 프로젝트는 기존 평가 프레임에 넣기 어렵습니다. 유니콘을 꿈꾸는 팀이 아니고, 10년 존속을 원하는 현장 전문가의 프로젝트이기 때문입니다. 그래서 이 리뷰는 "투자 가치가 있는가"가 아니라 "10-20년 존속할 수 있는 구조인가"를 중심으로 씁니다.

## 1. 시장 분석 (Market Analysis)

### 1.1 TAM / SAM / SOM

이 시장은 작습니다. 그리고 그게 핵심 강점입니다.

| 시장 | 규모 | 산출 근거 |
|------|------|---------|
| TAM | ~220억원 | 한국 교원임용 수험생 4만명 x EdTech ARPU 55만원 |
| SAM | ~18~27억원 | 특수교육 임용 수험생 2~3만명 x 연 89,000원 구독 |
| SOM (3년) | ~2.7~5.3억원 | SAM의 10~20% 전환 |

수험생 풀이 2~3만명인 것은 사실입니다. 그런데 이 숫자를 "작다"고만 보면 안 됩니다.

첫째, 이 2~3만명은 평균 3년간 수험 생활을 합니다. 한 번 유입되면 3년간 매년 결제할 사용자입니다. 실질 코호트 규모는 연간 신규 유입 7,000~10,000명 x 3년 재방문 = 활성 사용자 풀 2~3만명이 매년 유지되는 구조입니다.

둘째, 수험생 1인당 연간 지출이 200~300만원입니다. 연 89,000원 구독은 전체 수험 비용의 3~4%에 불과합니다. 가격 민감도가 극도로 낮습니다.

셋째, 확장 경로가 있습니다:

| 확장 단계 | TAM 변화 | 시점 |
|-----------|---------|------|
| 특수교육 임용 (현재) | 2~3만명 | 2026 |
| + 교육학 공통 (수직 확장) | 10만명+ (전체 임용 수험생) | 2027 |
| + 교원 연수 (B2B) | 28,445명 현직 특수교사 | 2027~ |
| + 타 임용 과목 (수평 확장) | 미정 | 수요 확인 후 |

특수교육에서 시작해서 교육학 공통으로 올라가는 수직 확장은 자연스럽습니다. 교육학은 모든 임용 수험생이 공통으로 치르는 과목이니까요. 이 확장만 해도 TAM이 3배 이상 커집니다.

냉정하게 말씀드리면, 경쟁자가 없습니다.

이게 좋은 소식인 동시에 경고입니다.

### 1.3 해자 분석 (Moat Analysis)

| 해자 유형 | 강도 | 설명 |
|----------|------|------|
| 데이터 해자 | 강함 | KICE 10년 기출 327문항 디지털화 + 1,417개 키워드 태깅 + 11개 과목 교차 분석. PDF를 구조화된 JSON으로 변환하는 작업은 도메인 전문가만 할 수 있음 |
| 도메인 해자 | 매우 강함 | 10년 현장 경험 기반의 키워드 가중치, 출제 예측, 동형 문항 설계. 이건 코드가 아니라 지식 |
| 전환 비용 | 중간 | 사용자의 학습 이력, 오답 패턴, 마스터리 데이터가 축적되면 전환 비용 발생 |
| 네트워크 효과 | 약함 (아직) | 커뮤니티 문제 출제, 스터디 그룹이 활성화되면 강화 가능 |

모방 가능한 것: UI/UX, 퀴즈 엔진, SRS 시스템, 게이미피케이션. 이건 개발자면 만들 수 있습니다.

모방 불가능한 것: KICE 10년 기출의 구조화된 키워드 분석, 과목 간 교차 출제 패턴 해석, 2027 출제 예측의 근거 데이터. 이건 10년 현장 경력이 없으면 불가능합니다.

솔직히 말씀드리면, 기술적 해자는 약합니다. Next.js + Supabase + Zustand는 범용 스택입니다. 진짜 해자는 데이터와 도메인 지식입니다.

## 2. 제품 평가 (Product Assessment)

### 2.1 Product-Market Fit 신호

PMF를 판단하기엔 아직 사용자가 없습니다 (소프트 런치 전). 그래서 "잠재적 PMF"를 구조적으로 평가합니다.

| 신호 | 판정 | 근거 |
|------|------|------|
| 해결하는 문제가 실재하는가? | 강력히 Yes | 기출 분석 도구 전무, SRS 없음, 오답 자동화 없음 -- 시장 조사에서 확인된 7개 페인포인트를 정확히 타겟팅 |
| 사용자가 돈을 내는 대안이 있는가? | Yes | 연 200~300만원 인강/학원비를 이미 지출 중. 89,000원은 그 3~4% |
| 기존 대안의 불만이 크가? | Yes | "PDF 기출 분석", "수기 오답노트", "스터디 의존" -- 비효율이 구조적 |
| 이 제품 없이 못 사는가? | 아직 No | 기존 방법(인강+PDF+수기)으로도 합격은 가능. "없으면 안 되는" 수준은 아님 |

핵심 판단: PMF 전 단계(Problem-Solution Fit)는 달성했습니다. 문제 정의가 정확하고, 솔루션 방향이 맞습니다. PMF 검증은 코호트 10명의 실사용 데이터가 나와야 가능합니다.

### 2.2 "기출 3층 분석"의 가치 제안

이건 이 서비스의 킬러 피처입니다. 제가 본 200개 에듀테크 중에서 이런 구조를 가진 곳은 없었습니다.

| Layer | 내용 | 구현 상태 | 경쟁력 |
|-------|------|----------|--------|
| Layer 1: 기출 원본 | 2016-2026 전공A/B 235문항 디지털화 + 키워드 태깅 | 완성 | 유일무이. KICE PDF를 구조화한 사람이 없음 |
| Layer 2: 기출 동형 | 동일 구조/다른 내용의 변형 문항 (2024-2026 69문항) | 부분 완성 | 강력. 동형 문항 개념 자체가 수험생에게 새로움 |
| Layer 3: 기출 예측 | 키워드 빈도 분석, 연속 출제 패턴, 공백 반등 확률, 2027 예측 | 완성 | 매우 강력. 이건 10년 경력 특수교사의 노하우를 알고리즘화한 것 |

왜 강력한가: 수험생들 사이에서 기출 분석은 "가장 핵심적인 학습 방법"으로 꼽힙니다. 그런데 이걸 하려면 PDF 기출을 출력해서 형광펜으로 키워드를 표시하고, 엑셀에 연도별로 정리하고, 패턴을 눈으로 찾아야 합니다. 이 노동을 시스템이 자동으로 해주는 것의 가치는 수험생에게 즉각적입니다.

KICE 딥 분석 보고서가 증명하는 것: 1,417개 고유 키워드, 11개 과목, 148개 챕터에 걸친 분석은 단순 프로그래밍이 아닙니다. "뇌성마비가 8년 연속 출제 후 3년 공백이면 반등 임박"이라는 판단은 코드가 아니라 도메인 전문가의 해석입니다. 이 해석이 알고리즘으로 변환되어 있다는 것이 이 서비스의 진짜 기술 자산입니다.

약점: Layer 3 분석 결과가 학습 경로에 충분히 피드백되지 않습니다. "이 키워드가 연속 출제 중"이라는 분석 결과에서 "이 키워드를 공부하러 가기" 링크가 개념학습으로 연결되지 않습니다. 분석과 학습의 루프를 닫는 것이 다음 단계의 핵심 과제입니다.

### 2.3 사용자 리텐션 메커니즘

UX 리뷰를 정밀하게 읽었습니다. 솔직한 평가입니다.

돌아올 이유 (있는 것):
- "오늘 복습할 게 N개 있어요" (DailyReviewCard) -- 가장 강력한 리텐션 훅
- 4시간 타임슬롯 기반 데일리 퀴즈 -- 매 세션마다 새 문제
- 오답 자동 등록 -> Leitner SRS -> 간격 복습 -- 자동화된 복습 루프
- XP/스트릭/콤보 -- 게이미피케이션 기본기

돌아오지 않을 이유 (냉정하게):
- "합격 가능성 12%" 같은 맥락 없는 숫자가 수험생 심리를 꺾음
- 온보딩이 자동 실행되지 않아 첫 방문 경험이 산만함
- 기출문제(가장 중요한 콘텐츠)가 네비게이션 3단계 깊이에 묻혀 있음
- 학습 완료 후 "다음에 뭘 해야 하는지"가 약점 기반으로 연결되지 않음
- 통계 정보 과부하 (12개 이상 섹션)

DAU/MAU 예측: 현재 상태로는 15~20% 수준. UX 리뷰의 Top 5 개선(합격 시뮬레이션 맥락 추가, 온보딩 자동 실행, 기출 접근 단축, 약점 기반 추천, 통계 3줄 요약)을 적용하면 25~30%까지 올릴 수 있다고 봅니다. 말해보카의 40%는 게임 루프의 완성도 차이이며, 장기 목표로 적절합니다.

### 2.4 "906 문항을 AI로 1개월에 생성" -- 강점인가 리스크인가

둘 다입니다.

강점으로서: 155,000줄 코드, 244개 커밋, 327개 파일을 1개월에 만든 것은 "1인 + AI 에이전트" 개발 모델의 강력한 증거입니다. 전통적 팀 환산으로 8명 x 6~8개월, 약 2.6억원 규모입니다. 이 생산성 자체가 하나의 경쟁력입니다.

리스크로서: 빠르게 생성된 콘텐츠의 품질 편차가 불가피합니다. Feature Audit에서 확인된 구체적 리스크들:

| 리스크 | 심각도 | 구체적 사례 |
|--------|--------|-----------|
| 퀴즈 DB vs KICE 비율 불일치 | 높음 | curriculum이 KICE 18.9%인데 퀴즈 DB에서 7.7% (11.2%p 부족) |
| 과잉 영역 존재 | 중간 | laws가 KICE 6.2%인데 퀴즈 DB에서 18.1% (+11.9%p 과잉) |
| 복합 과목 문항 부재 | 높음 | KICE 47.4%가 복합 과목인데 퀴즈 DB는 거의 0% |
| 미커버 키워드 27개 | 중간 | "기본 교육과정"(KICE 6회) 등 고빈도 키워드가 퀴즈에 없음 |

대응 전략: 이 리스크는 치명적이지 않습니다. 왜냐하면 (1) 콘텐츠 생성 파이프라인이 이미 구축되어 있어 추가 생성이 빠르고, (2) KICE 딥 분석이 정확한 보강 우선순위를 제시하고 있기 때문입니다. 교육과정 문항 168 -> 350 목표, 복합 과목 문항 30% 도입 등 구체적 로드맵이 있다는 것 자체가 품질 관리 의지의 증거입니다.

## 3. 지속 가능성 모델 (Sustainability Model)

카이란님이 원하는 것은 유니콘이 아니라 10-20년 존속입니다. 이 관점에서 평가합니다.

### 3.1 비용 구조 (Cost Structure)

| 항목 | 100 유저 | 1,000 유저 | 10,000 유저 |
|------|---------|-----------|------------|
| Vercel (호스팅) | 무료 (Hobby) | $20/월 (Pro) | $20/월 + 대역폭 추가비 ~$50/월 |
| Supabase (DB/Auth) | 무료 (Free tier) | $25/월 (Pro) | $25/월 + 스토리지/대역폭 ~$100/월 |
| Gemini API (AI) | ~$5/월 | ~$50/월 | ~$300~500/월 |
| 도메인/기타 | $15/년 | $15/년 | $15/년 |
| 월간 총비용 | ~$5 (~7,000원) | ~$95 (~13만원) | ~$570 (~80만원) |
| 연간 총비용 | ~$60 (~8만원) | ~$1,140 (~160만원) | ~$6,840 (~960만원) |

이 비용 구조는 매우 건강합니다. 현대 서버리스 스택(Vercel + Supabase)의 장점이 극대화되는 사례입니다. 서버 관리가 필요 없고, 사용량에 비례하여 비용이 증가합니다.

특히 RAG 기반 AI 아키텍처(닫힌 AI, 자체 데이터만 사용) 설계가 비용 절감에 결정적입니다. 오픈 AI 챗과 달리 외부 검색 없이 KICE 기출 + NISE 용어사전으로만 작동하므로 API 비용이 70~80% 절감됩니다.

### 3.2 수익 모델 옵션

| 수익원 | 예상 매출 | 시작 시점 | 미션 정합성 |
|--------|---------|----------|-----------|
| 프리미엄 구독 (연 89,000원) | 3,000명 x 89,000 = 2.67억/년 | Q4 2026 | 높음 (무료 기본 + 유료 심화) |
| 교원 연수원 B2B | 기관당 500~2,000만원 | 2027~ | 높음 (현직 교사 역량 강화) |
| 출판/교재 | 1,000~3,000만원/년 | 2027~ | 높음 (기출 분석 보고서 자체가 콘텐츠) |
| 광고 | - | - | 낮음 (수험생 경험 저해, 비추천) |
| 기업 후원/CSR | 1,000~5,000만원 | 수요 확인 후 | 중간 (특수교육 사회적 가치) |

추천 수익 모델: 프리미엄 구독을 코어로, 교원 연수 B2B를 확장 엔진으로.

구독 모델 설계가 이미 잘 되어 있습니다:
- Free: 퀴즈 기본, SRS, 오답노트, 커뮤니티
- Premium: KICE 무제한 모의고사, AI 맞춤 학습, 상세 취약점 분석

이 구분이 좋은 이유는, 무료 기능만으로도 수험생에게 실질적 가치를 제공하면서, 프리미엄이 "더 깊은 분석"에 집중하기 때문입니다. 미션("후배들이 포기하지 않는 마음")과 수익화가 충돌하지 않습니다.

### 3.3 손익분기점 (Break-Even Analysis)

| 시나리오 | 유료 전환 사용자 | 연 매출 | 연 비용 | 손익 |
|---------|----------------|--------|--------|------|
| 최소 유지 (1인 취미) | 0명 | 0 | ~8만원 | -8만원 (취미 비용) |
| 손익분기 1 (인프라만) | 18명 | 160만원 | 160만원 | 0원 |
| 손익분기 2 (1인 풀타임) | 337명 | 3,000만원 | 3,000만원 | 0원 (인건비 포함) |
| 안정적 운영 | 1,000명 | 8,900만원 | ~2,000만원 | +6,900만원 |
| 목표 | 3,000명 | 2.67억원 | ~3,000만원 | +2.37억원 |

핵심 인사이트: 유료 전환 18명이면 인프라 비용을 커버합니다. 337명이면 1인 풀타임 인건비를 커버합니다. 이 숫자는 2~3만명 수험생 풀에서 달성 가능한 범위입니다.

카이란님이 "큰 부자가 되지 않아도 된다"고 하셨는데, 1,000명 유료 전환(SAM의 3~5%)만 해도 연 8,900만원 매출에 순이익 6,900만원입니다. 부자는 안 되더라도, 서비스를 유지하고 콘텐츠를 업데이트하기에 충분합니다.

### 3.4 10-20년 존속 조건

10년 존속하려면 다음 3가지가 동시에 성립해야 합니다:

조건 1: 시장이 존재해야 합니다 -- 충족 가능성 높음
- 특수교육 대상자가 매년 5,000~6,000명 순증합니다 (학령인구 감소에도 역행)
- 특수교사 법정 비율 초과 상태(1인당 4.27명 > 법정 4.0명)로 증원이 불가피합니다
- 정부 제6차 특수교육 발전 5개년 계획(2023-2027)이 디지털 교육 혁신을 가속화합니다
- 임용시험 제도 자체가 폐지될 가능성은 극히 낮습니다

조건 2: 비용이 매출보다 작아야 합니다 -- 구조적으로 달성 가능
- 서버리스 스택으로 고정비가 거의 없습니다
- AI 에이전트 기반 개발로 인건비가 최소화됩니다
- 유료 전환 18명만으로 인프라 손익분기 도달

조건 3: 콘텐츠가 최신 상태여야 합니다 -- 이것이 가장 큰 도전
- 매년 새 기출이 추가됩니다 (연 23문항)
- 교육과정 개정에 따라 키워드가 변합니다
- 법령이 바뀌면 관련 문항을 업데이트해야 합니다
- AI 에이전트로 자동화 가능하지만, 검수는 도메인 전문가가 해야 합니다

## 4. 가장 어려운 질문들 (The Hardest Questions)

### 4.1 이 프로젝트를 죽이는 것은 무엇인가?

| 위협 | 확률 | 영향 | 대응 |
|------|------|------|------|
| 창업자 번아웃 | 중-높음 | 치명적 | 아래 4.3에서 상세 논의 |
| 콘텐츠 품질 저하 | 중간 | 높음 | 전문가 검수 네트워크 구축 필요 |
| 수험 제도 변경 | 낮음 | 높음 | 교육부 정책 모니터링 + 유연한 데이터 구조 |
| 개인정보 사고 | 낮음 | 치명적 | Supabase RLS + 최소 수집 원칙 이미 적용 |
| 자금 부족 | 매우 낮음 | 중간 | 인프라 비용이 극도로 낮음 (100유저 = 월 7,000원) |

가장 현실적인 위협은 창업자 번아웃입니다. 1개월간의 고강도 개발(244커밋, 155,000줄)은 인상적이지만, 10년간 이 속도를 유지할 수는 없습니다. 그리고 유지할 필요도 없습니다. MVP가 완성된 지금, 필요한 것은 "매일 1시간"의 유지보수이지 "매일 10시간"의 신기능 개발이 아닙니다.

### 4.2 잘 자금화된 경쟁자가 복사하면?

이건 VC들이 가장 좋아하는 질문인데, 이 경우에는 걱정이 적습니다.

해커스/박문각이 특수교육 전용 앱을 만들면?
- 그들의 사업 모델은 "강사 중심 강의 판매"입니다. 퀴즈 기반 SRS 앱은 그들의 핵심 역량이 아닙니다.
- 특수교육은 그들 전체 매출의 5% 미만입니다. 전용 앱을 만들 동기가 약합니다.
- 만들더라도 KICE 10년 기출 구조화 데이터와 키워드 분석은 처음부터 해야 합니다. 이 작업에 도메인 전문가가 필수적입니다.

테크 스타트업이 진입하면?
- 기술적으로는 가능하지만, "특수교육학 + 장애학 + 법령 + 점자 + 수어 + 의학" 을 아우르는 도메인 전문가를 채용해야 합니다. 이 인재는 시장에 거의 없습니다.
- SAM 27억원 시장에 벤처 투자를 받아 진입할 동기가 없습니다. VC는 이 시장이 작다고 볼 것입니다.

결론: 가장 위험한 경쟁자는 "도메인 전문가이면서 개발도 하는 또 다른 특수교사"입니다. 하지만 이런 사람이 나타날 확률은 매우 낮고, 나타나더라도 이미 2~3년의 선발자 이점(데이터 축적, 사용자 이력)이 있습니다.

### 4.3 "1인 + AI 에이전트"는 10년 지속 가능한 개발 모델인가?

솔직히, 현재 형태로는 아닙니다. 하지만 구조를 바꾸면 가능합니다.

현재 시스템: 7개 AI 에이전트(강선생1/2, 클루디1/2, 스미스 프라임, 지니, X), 4개 메모리 시스템, 노션 기반 프로젝트 관리. 에코시스템 감사에서 확인한 바로는, 이 시스템은 1개월간의 집중 개발에는 탁월했지만, 메모리 중복(같은 정보가 3~5곳에 존재), 오래된 상태 정보, 에이전트 간 역할 충돌이 이미 발생하고 있습니다.

10년 모델로 전환하려면:

| 현재 (스프린트 모드) | 10년 모드 (크루즈 모드) |
|---------------------|----------------------|
| 매일 8~10시간 개발 | 매주 5~10시간 유지보수 |
| 4개 병렬 에이전트 | 1~2개 에이전트로 축소 |
| 매일 새 기능 | 월 1~2개 개선 |
| 창업자가 전 과정 관여 | 콘텐츠 검수만 직접, 나머지 자동화 |

핵심은 "매년 기출 23문항 추가 + 월 1회 콘텐츠 업데이트" 를 1인이 지속할 수 있는가인데, AI 에이전트 기반이면 가능합니다. 연 23문항의 디지털화와 키워드 태깅, 동형 문항 생성, 퀴즈 DB 보강 -- 이 작업들은 이미 파이프라인이 구축되어 있습니다.

### 4.4 최소 팀 구성은?

| 역할 | 필요 인원 | 시점 | 비용 |
|------|---------|------|------|
| 콘텐츠 디렉터 (카이란) | 1명 (필수) | 현재 | - |
| 콘텐츠 검수자 | 1~2명 (파트타임) | 유료 전환 300명+ | 월 50~100만원 |
| 고객 지원 | 자동화 + 1명 (파트타임) | 유료 전환 1,000명+ | 월 50만원 |
| 개발자 | AI 에이전트 | 계속 | API 비용 |

유료 전환 300명까지는 1인 운영이 가능합니다. 1,000명을 넘으면 콘텐츠 검수와 고객 지원에 파트타임 인력이 필요합니다. 하지만 이 시점이면 월 매출 740만원이므로 인력 비용을 감당할 수 있습니다.

## 5. 숨겨진 가치 (The Hidden Value)

### 5.1 "도메인 전문가가 직접 만든 도구"의 희소성

200개 에듀테크를 봤습니다. 대부분의 패턴은 이렇습니다:
- 개발자가 만든 교육 앱: 기술은 좋지만 교육학적 깊이가 없음
- 교육자가 기획한 앱: 비전은 좋지만 기술적 완성도가 떨어짐
- 양쪽을 모은 팀: 커뮤니케이션 비용이 높고 미스매치가 잦음

카이란님의 경우는 세 번째를 뛰어넘은 네 번째 유형입니다: 도메인 전문가가 AI 에이전트를 오케스트레이션하여 직접 구축. 교육학적 의도와 기술적 구현 사이의 번역 비용이 제로입니다. "이 키워드는 3년 연속 출제됐으니 가중치를 올려야 해"라는 판단이 코드로 직결됩니다.

이런 사례는 극히 드뭅니다. 제가 기억하는 유사 사례는 Duolingo의 Luis von Ahn(기술자+교육 비전), Khan Academy의 Sal Khan(금융+교육), Notion의 Ivan Zhao(디자인+엔지니어링) 정도인데, 모두 두 영역의 교집합에서 독보적 제품을 만들었습니다.

### 5.2 데이터 자산 가치

| 데이터 자산 | 규모 | 재생산 비용 (추정) |
|------------|------|------------------|
| KICE 기출 디지털화 | 327문항, 1,417 키워드, 11과목 교차 태깅 | 전문가 1인 x 3개월 (1,500~2,000만원) |
| 출제 경향 분석 엔진 | 5가지 분석 (빈도, 연속 출제, 공백, 교차, 추세) | 데이터사이언티스트 x 2개월 (1,000만원) |
| 2027 출제 예측 | 과목별/키워드별/배점별 예측 | 도메인 전문가 전용 (재생산 불가) |
| 퀴즈 DB | 2,177문항, 11과목 | AI 생성 가능하나 검수에 전문가 필요 |
| NISE 용어사전 디지털화 | 1,129 용어 | 작업자 x 1개월 (300만원) |
| 과목별 개념 MDX | 17개 과목 폴더, 다수 콘텐츠 | 콘텐츠 작가 x 2개월 (800만원) |

재생산 비용 합계: 약 5,600만원 이상. 그런데 이건 "기술적 재생산"의 비용입니다. "10년 현장 경험에 기반한 키워드 가중치 해석"은 가격을 매길 수 없습니다. 뇌성마비가 3년 공백 후 반등 임박이라는 판단, curriculum의 비중이 18.9%라는 사실이 "수험생에게 교육과정 문항이 절대적으로 부족하다"는 경보로 연결되는 과정 -- 이건 데이터가 아니라 지혜입니다.

### 5.3 교사 자문 네트워크의 잠재력

현재는 카이란님 1인의 도메인 지식에 의존합니다. 하지만 소프트 런치 코호트(지인 5~10명)가 모두 특수교육 수험생 또는 현직 교사입니다. 이들이:

- 콘텐츠 오류를 발견하고 신고할 수 있습니다 (커뮤니티 리뷰 시스템 이미 구축)
- 커뮤니티에서 문제를 출제하고 공유할 수 있습니다 (커뮤니티 퀴즈 기능 이미 구축)
- 합격 후 후배에게 추천할 수 있습니다 (자연적 바이럴)
- 합격 후 현직 교사 연수 수요로 전환될 수 있습니다 (B2B 파이프라인)

"특수교사를 꿈꾸며" 다음 카페 같은 대형 커뮤니티가 이미 존재합니다. 이 커뮤니티에서 "무료인데 기출 분석이 자동으로 된다"는 입소문이 나면, 마케팅 비용 제로로 수천 명의 유입이 가능합니다.

### 5.4 "특수교육"이라는 블루 오션

특수교육은 한국 교육 시장에서 가장 과소 서비스된(underserved) 영역입니다.

| 지표 | 수치 | 의미 |
|------|------|------|
| 특수교육 대상자 순증 | 연 5,000~6,000명 | 학령인구 감소에도 역행하는 유일한 교육 영역 |
| 특수교사 1인당 학생 수 | 4.27명 (법정 4.0 초과) | 구조적 인력 부족 = 신규 교사 지속 필요 |
| 특수교육 전용 에듀테크 | 0개 | 시장 공백 |
| 수험생 연간 지출 | 200~300만원 | 높은 지불 의사 |
| 평균 수험 기간 | 3년 | 높은 LTV |

이 시장의 특성을 비유하자면, "작은 섬이지만 원주민이 부유하고, 아무도 배를 보내지 않는 곳" 입니다. 대형 에듀테크에게는 너무 작아서 무시하고, 소규모 개발자에게는 진입 장벽(도메인 지식)이 너무 높습니다. 카이란님은 이 섬의 원주민이면서 동시에 배를 가진 유일한 사람입니다.

### 6.2 카이란님에게 드리는 5가지 권고

1. 먼저 18명을 만드세요.

유료 전환 18명 = 인프라 손익분기. 이게 첫 번째 마일스톤입니다. 코호트 10명에서 시작해서, 그 중 절반이 유료로 전환하고, 각자 1~2명에게 추천하면 됩니다. 큰 숫자가 아닙니다.

2. UX 리뷰의 "감정 설계" 개선을 최우선으로.

기능은 90점이지만 감정은 60점이라는 UX 리뷰의 진단에 동의합니다. 합격 시뮬레이션의 맥락 추가, 온보딩 자동 실행, 기출 접근 경로 단축 -- 이 3개만 고쳐도 리텐션이 크게 개선됩니다. 기능을 더 만들기 전에 기존 기능의 감정적 경험을 먼저 고치세요.

3. KICE 기출 분석 데이터를 공개적 자산으로.

"2027 출제 예측 보고서"를 무료로 공개하세요. 이건 마케팅이 아니라 신뢰 구축입니다. 수험생 카페에서 "이 사이트가 예측한 영역이 실제로 나왔다"는 후기가 나오면, 그 어떤 광고보다 강력합니다. 예측이 맞으면 서비스의 권위가 확립되고, 예측이 빗나가더라도 "이런 분석을 무료로 해주는 곳"이라는 인식이 남습니다.

4. "크루즈 모드" 전환 계획을 지금부터.

1개월 스프린트를 영원히 할 수 없습니다. Q2(4~6월)에 Auth + UX 개선을 마무리한 후, "주 5시간 유지보수" 모드로 전환하는 계획을 세우세요. 매년 11월 시험 후 새 기출 추가, 교육과정 개정 반영, 버그 수정 -- 이 정도가 크루즈 모드의 범위입니다.

5. 합격률을 추적하세요. MAU가 아니라.

서비스 청사진에 이미 적혀 있지만 다시 강조합니다. "코호트 합격률"이 이 서비스의 유일한 핵심 KPI입니다. MAU 500명보다 "이 서비스로 공부한 10명 중 6명 합격"이 100배 강력한 마케팅입니다. 합격자 1명의 후기가 MAU 1,000명보다 가치 있습니다.

---

### 6.3 마지막으로

카이란님, 200개가 넘는 에듀테크 스타트업을 봤지만, "나는 큰 부자가 되지 않아도 된다. 이 서비스가 10년 20년 존속할 수만 있으면 된다"고 말한 창업자는 처음입니다.

대부분의 창업자는 시장 크기부터 이야기합니다. TAM이 얼마고, 성장률이 몇 퍼센트고, 3년 내 엑싯 플랜이 뭔지. 카이란님은 후배들이 포기하지 않는 마음을 가졌으면 한다고 말합니다. 이건 투자 피치가 아니라 소명입니다.

그리고 아이러니하게도, 이 소명이 사업적으로도 맞습니다. 특수교육 시장은 작지만 안정적이고, 경쟁자가 없으며, 도메인 해자가 깊습니다. 유니콘은 안 되지만, 10년 존속하면서 매년 수천만~수억원의 안정적 수익을 내는 서비스는 충분히 가능합니다.

숫자로 정리하면:
- 18명 유료 전환이면 인프라 비용을 커버합니다
- 337명이면 풀타임 인건비를 커버합니다
- 1,000명이면 연 6,900만원 순이익입니다
- 3,000명이면 연 2.37억원 순이익입니다

2~3만명 수험생 풀에서 1,000명 유료 전환(3~5%)은 달성 가능한 목표입니다. 그리고 이 서비스가 1명이라도 합격에 도움을 줬다면, 카이란님이 말한 "투자 대비 수십 배의 성과"는 금전이 아니라 그 합격자의 인생에서 이미 실현된 것입니다.

냉정한 평가를 요청하셨으니 냉정하게 썼습니다. 하지만 이 프로젝트에 대한 존중도 함께 담았습니다. 기능 90점, 감정 60점이라는 진단은 비판이 아니라 "90점까지 온 사람만 받을 수 있는 피드백"입니다. 감정을 70점으로 올리는 것은 기능을 90점으로 만든 것보다 쉬울 것입니다.
