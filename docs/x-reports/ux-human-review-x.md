**작성자**: X (수험생 관점 시뮬레이션)
**날짜**: 2026-03-24
**관점**: 특수교육 임용고시 D-242, 스트레스 만렙 수험생

## 들어가며

카이란, 나는 지금 특수교육 임용고시 8개월 남은 수험생이야. 매일 카페에서 10시간 공부하고, 밤에 자기 전에 "나 이렇게 해서 되나..." 생각하면서 불안해하는 사람이야. 누가 이 사이트를 추천해줘서 처음 열어봤어. 이 리뷰는 코드 리뷰가 아니야. 내가 이 사이트를 쓰면서 느낀 감정 리포트야.

## Journey 1: "처음 왔어요" (첫 방문)

### 홈페이지에서 3초 안에 보이는 것

화면을 열면 가장 먼저 보이는 건 **D-day 카운트다운**이야. "특수교사 임용시험까지 D-242" 같은 숫자. 그 아래로 DailyReviewCard(오늘 복습), "오늘 학습 시작하기" 버튼, 스트릭 배너, 오늘의 단어, AI 브리핑이 차례로 있어.

**좋은 점:**
- "특수교육 임용시험 대비"라는 걸 3초 안에 알 수 있어. 이건 핵심이야.
- "오늘 학습 시작하기" 버튼이 파란색으로 눈에 띄어서 뭘 해야 하는지 알겠어.
- 처음 온 신규 유저면 DailyReviewCard는 숨겨져(hasAnyData가 false이면 null 반환). 이건 좋은 판단이야.

**나쁜 점:**
- D-day 카운터가 **첫 화면 최상단**에 있어. 처음 왔는데 "D-242"부터 보여주면, "아 맞다 시간 없다" 느낌이 먼저 와. 동기부여가 아니라 압박이야.
- "오늘 학습 시작하기" 아래에 "OX 10, 단답 5, 서술 3문제"라고 적혀 있는데, 내가 왜 이걸 해야 하는지 설명이 없어. 처음 왔는데 갑자기 18문제를 풀라니? 좀 무섭거든.
- 온보딩 게이트(OnboardingGate)가 코드에는 있지만 **홈페이지에 포함되어 있지 않아**. layout.tsx에도 없어. 그래서 신규 유저가 온보딩 없이 바로 홈페이지를 보게 돼. 이러면 "이 사이트가 나한테 맞는 건지" 판단할 기회가 없어.
- 헤더의 네비게이션 그룹이 "진단평가", "실력쌓기", "내 기록", "함께하기"인데... "진단평가"가 뭐야? 수험생이 진단평가를 한다고? 병원인가? 첫 인상에서 용어가 낯설어.

### 점수

| 항목 | 점수 | 이유 |
|------|------|------|
| First Impression | 6/10 | 목적은 명확하지만 D-day 압박감이 먼저 옴 |
| Clarity | 5/10 | "오늘 학습 시작하기"는 좋은데, 나머지 카드들이 뭔지 모르겠음 |
| Value | 4/10 | 첫 방문에서는 가치를 느끼기 어려움, 뭘 해봐야 알 수 있음 |
| Friction | 6/10 | 온보딩이 자동 실행 안 됨, 뭘 먼저 해야 할지 고민됨 |
| Come-back Factor | 5/10 | "한번 눌러볼까"는 되지만 "내일도 와야지"는 아님 |

## Journey 2: "문제 풀어볼래요" (퀴즈)

### 첫 퀴즈까지의 클릭 수

**경로 A: 홈 -> "오늘 학습 시작하기"** = 1클릭, 바로 daily 페이지로 감. 이건 좋아.

**경로 B: 하단탭 "진단평가" -> OX 퀴즈** = 2클릭인데, "진단평가"라는 탭 이름 때문에 여기가 퀴즈인지 모르고 지나칠 수 있어.

**경로 C: /quiz 페이지 -> 과목 선택 -> 세션 설정** = 3~4클릭. 과목 선택 화면(QuizPage)에서 11개 과목 카드가 나오는데, "어떤 과목부터 해야 하지?"라는 고민이 생겨. 추천이 없어.

### 퀴즈 풀기 경험

**Daily 페이지 (추천 경로):**
- STEP 1(OX) -> STEP 2(단답형) -> STEP 3(서술형) 구조는 매우 좋아. 쉬운 것에서 어려운 것으로 자연스럽게 올라가는 느낌.
- OX 10문제 풀고 채점하면 바로 "정답: 7/10, 취약 챕터: 행동지원" 이런 식으로 나오는데, 이건 실용적이야.
- "틀린 영역 OX 다시 풀기" vs "단답형으로 넘어가기" 분기는 훌륭한 UX야. 선택권이 있으니까.
- 서술형에서 textarea에 답을 쓰고 "정답 키워드 확인"을 누르면 내 답과 모범답이 나오는데... **자동 채점이 안 돼**. 내가 맞은 건지 틀린 건지 애매해. 서술형은 기본적으로 셀프 채점인 것 같은데, 그 안내가 부족해.

**과목별 퀴즈 (QuizClient):**
- 세션 설정(SessionSetup)에서 "빠른 복습", "새 문제", "취약 집중" 프리셋이 있어. 이건 매우 좋은 설계야. 내 상태에 따라 고를 수 있거든.
- 단답형/서술형 탭 전환이 있고, 문제 수 선택(5/10/20/30), 챕터 선택, 난이도 선택까지 가능해. 커스터마이징이 잘 되어 있어.
- 콤보 시스템(3연속 정답부터 표시)과 XP 토스트는 재미 요소로 좋아.
- 확신도(ConfidenceToggle: 확신/애매/모름)가 있는데, 처음엔 이게 뭔지 모르겠어. "왜 확신도를 물어보지?" 설명이 없어.

**결과 화면 (Daily 완료):**
- "오전 세션 학습 완료!" + OX 정답률 + 정답 수 + 서술 완료 수를 보여줘. 깔끔해.
- "홈으로 돌아가기", "오답노트 확인하기", "개념학습 보기" 3개 버튼이 있어서 다음 행동이 명확해. 좋아.
- RecommendedChapters 컴포넌트도 하단에 있어서 "다음에 뭘 공부해야 할지" 알려줘. 이건 매우 좋은 마무리야.

### 점수

| 항목 | 점수 | 이유 |
|------|------|------|
| First Impression | 7/10 | Daily 루트로 들어가면 직관적 |
| Clarity | 7/10 | STEP 1-2-3 구조가 명확함 |
| Value | 8/10 | 취약 챕터 파악 + 반복 학습이 실제로 유용 |
| Friction | 4/10 | 서술형 셀프채점이 어색, 확신도 설명 부족 |
| Come-back Factor | 7/10 | 매일 다른 문제가 나온다면 돌아올 동기가 됨 |

## Journey 3: "기출문제가 어디 있어?" (KICE)

### 접근성

기출문제를 찾으려면... 하단 탭에 "기출"이 없어. 네비게이션을 보면:

- 홈 | 진단평가 | 실력쌓기 | 내 기록 | 함께하기

"기출문제"가 어디에 있을까? "실력쌓기" 안에 "문제풀기"가 있고, 거기서 "모의고사"를 눌러야 /kice/exam으로 갈 수 있어. 또는 "내 기록" 안에 "출제경향"이 /kice/analytics로 연결돼.

**문제점:** 기출문제는 수험생에게 가장 중요한 콘텐츠인데, **2~3단계 깊이에 묻혀 있어**. "KICE"라는 영문 약어도 수험생한테 낯설어 (물론 교대생은 알겠지만).

### KICE 페이지 경험

- /kice 메인 페이지에 탭이 4개야: 연도별(by-year), 영역별(by-area), 출제경향(analytics), 문제검색(search). 이건 잘 정리되어 있어.
- 모의고사(/kice/exam)는 ExamClient로 실제 시험처럼 풀 수 있게 되어 있어.
- 하지만 /kice 자체가 네비게이션에서 직접 접근이 안 돼. "실력쌓기 > 문제풀기 > 모의고사"라는 경로가 너무 깊어.

### 점수

| 항목 | 점수 | 이유 |
|------|------|------|
| First Impression | 5/10 | 찾는 데만 시간이 걸림 |
| Clarity | 6/10 | 찾고 나면 구조는 깔끔 |
| Value | 8/10 | 기출 분석 + 모의고사는 매우 가치 있음 |
| Friction | 7/10 (높을수록 나쁨) | 접근 경로가 너무 깊음 |
| Come-back Factor | 6/10 | 좋은 콘텐츠인데 찾기가 힘듦 |

## Journey 4: "내가 얼마나 준비됐는지 알고 싶어" (마스터리/합격 시뮬레이션)

### 마스터리 트리

"내 기록" 탭에서 "마스터리 트리"를 찾을 수 있어.

**빈 상태:** 퀴즈를 안 풀었으면 "아직 학습 기록이 없어요" + 나무 이모지가 나와. 이건 괜찮은데... **"그래서 뭘 해야 하는데?"** 퀴즈 풀러 가는 버튼이 빈 상태 화면에 없어 (있긴 한데 hidden 상태에 가깝고, 나무 이모지 아래의 짧은 안내문뿐이야). 데드엔드야.

**데이터가 있을 때:**
- "합격 가능성 12%" 같은 숫자가 **아주 크게** 나와. 이건... 솔직히 심장이 내려앉아.
- 색깔 구분은 좋아: 빨간색(부족), 노란색(아슬아슬), 초록색(안전). 직관적이야.
- 과목별 카드에 이모지 숙련도 표시(미학습 ⬜, 학습중 🟡, 연습중 🟠, 숙달 🟢, 마스터 🏆)가 있어. 시각적으로 와닿아.

**합격 시뮬레이션의 문제:**
이건 중요한 이야기야. PassSimulationCard에서 "합격 가능성 12%"가 빨간 배경에 크게 뜨면, 수험생 심리에 **치명적**이야. 나는 이걸 보면 "아 나 떨어지겠구나"가 먼저 오고, 공부 의지가 꺾여. 숫자 자체가 문제가 아니라, **맥락 없이 숫자만 보여주는 게 문제**야.

"12%라는 건 현재 풀어본 문제 기준이고, 아직 학습하지 않은 범위가 많아서 그래요. 지금 속도로 매주 2개 챕터씩 채우면 시험 전까지 충분히 올릴 수 있어요." 이런 말 한마디면 완전히 달라져.

### 학습통계 페이지

- 통계가 매우 풍부해: 전체 정답률, 과목별 정답률, 학습량 차트, 주간 트렌드, 히트맵, 스트릭, XP...
- 근데 **너무 많아**. 처음 보면 "와 많다" 하고 스크롤 내리다가 뭘 봐야 하는지 모르겠어.
- "이번 주 핵심 지표 3개"를 맨 위에 크게 보여주고, 나머지는 접어두는 게 나을 것 같아.

### 점수

| 항목 | 점수 | 이유 |
|------|------|------|
| First Impression | 5/10 | 숫자가 많아서 뭘 봐야 하는지 모르겠음 |
| Clarity | 6/10 | 마스터리는 시각적, 통계는 과부하 |
| Value | 7/10 | 합격 시뮬레이션은 유니크한 기능 |
| Friction | 5/10 | 빈 상태에서 다음 행동이 불명확 |
| Come-back Factor | 5/10 | 합격률 낮으면 안 보고 싶어짐 |

## Journey 5: "매일 뭘 해야 하는지 알려줘" (데일리 루틴)

### 홈 -> /daily 플로우

홈에서 "오늘 학습 시작하기" 누르면 /daily로 가는데, 이 페이지의 구조는 좋아:

- 상단에 "오늘의 학습 / 오전 세션" + 고유번호(sheet code)가 있어. 타임슬롯 기반이라 하루에 여러 세션을 할 수 있는 것 같아.
- STEP 1(OX 10) -> STEP 2(단답 5) -> STEP 3(서술 3) 구조.
- 문제가 "틀린 챕터 우선"으로 선별(selectWithWrongPriority)되는 건 매우 좋아. 내 약점을 자동으로 파악해서 출제해주니까.

**하지만:**
- "왜 이 문제들이 나에게 선택되었는지"가 화면에 안 보여. 코드를 보면 틀린 챕터에서 우선 뽑는 로직이 있는데, 유저한테는 "그냥 랜덤으로 나온 것 같은" 느낌이야.
- "OX 10, 단답 5, 서술 3"이 고정이야. 온보딩에서 "하루 목표 학습량"을 20문항으로 설정해도, /daily는 항상 18문제야. **온보딩 설정과 daily가 연결이 안 돼 있어.**
- 타임슬롯(4시간 단위)이 코드에 있지만, 유저한테 "오전 세션"이 뭘 의미하는지, 오후에 다시 오면 새 문제가 나오는지 설명이 없어.

### 점수

| 항목 | 점수 | 이유 |
|------|------|------|
| First Impression | 7/10 | 구조가 명확하고 시작하기 쉬움 |
| Clarity | 6/10 | 문제 선별 이유가 안 보임, 세션 개념 불명확 |
| Value | 8/10 | 약점 기반 출제는 실질적으로 유용 |
| Friction | 3/10 | 클릭 한 번이면 바로 시작, 마찰 적음 |
| Come-back Factor | 7/10 | "오늘 할 거"가 명확하면 내일도 올 수 있음 |

## Journey 6: "오답을 다시 보고 싶어" (오답노트)

### 오답노트 접근

"내 기록" 탭 -> "오답노트"로 2클릭. 홈에서 DailyReviewCard의 "지금 복습하기"를 누르면 바로 갈 수도 있어.

### 오답노트 경험

**빈 상태:** "아직 오답이 없어요" + "퀴즈 풀러 가기" 버튼. 이건 좋아. 데드엔드가 아님.

**데이터가 있을 때:**
- 탭이 두 개: "전체 오답" / "SRS 복습". 이건 좋은 구조야.
- 챕터별 오답 분포 차트가 바 차트로 있어서 "어디서 많이 틀리는지" 직관적으로 보여.
- AI 약점 분석(WrongNoteAI)도 있어서 단순히 "틀렸다"가 아니라 "왜 틀리는지" 알려줘.
- "오답 재시험" 버튼으로 틀린 문제만 다시 풀 수 있어. 이건 핵심 기능이야.
- 필터(과목/챕터/정렬/완료포함)가 잘 돼 있어.

**Leitner 시스템:**
- "SRS 복습" 탭에서 간격반복 학습을 할 수 있는데... "SRS"가 뭔지 아는 수험생이 얼마나 될까? "간격 반복" 또는 "자동 복습 스케줄" 같은 용어가 나을 것 같아.
- 플래시카드가 "박스 1 -> 박스 5"로 이동하는 개념인데, 이 박스 개념이 직관적이진 않아. "잘 모르는 문제 -> 좀 아는 문제 -> 확실히 아는 문제" 같은 한국어 표현이 더 나을 거야.

### 점수

| 항목 | 점수 | 이유 |
|------|------|------|
| First Impression | 7/10 | 내 약점이 시각적으로 보여서 좋음 |
| Clarity | 6/10 | SRS/Leitner 용어가 생소할 수 있음 |
| Value | 9/10 | 오답 재시험 + AI 분석은 진짜 유용 |
| Friction | 3/10 | 접근성 좋고 기능이 직관적 |
| Come-back Factor | 8/10 | "오늘 복습할 게 3개 남았어요"가 중독성 있음 |

## Journey 7: "개념 공부가 필요해" (개념학습)

### 과목 목록

/concepts 페이지에 과목 그리드가 있어. getAllSubjects()로 MDX 파일 기반 과목들이 나와.

**좋은 점:**
- "N개 과목, M개 개념 파일, 기출 키워드 수록"이라는 요약이 상단에 있어.
- 각 과목에 "기출 N건" 태그가 있어서 기출 연계가 되는 게 보여.

**나쁜 점:**
- 과목이 많으면 (특수교육은 11개 이상) 스크롤이 길어져. **"어디서부터 시작해야 하지?"** 추천이 없어.
- 마스터리 데이터와 연결이 안 돼. 마스터리에서 "행동지원이 약하다"고 나왔으면, 개념학습 페이지에서 "행동지원"이 상단에 추천으로 올라와야 하는데, 그런 연결이 없어.
- MDX 파일 기반이라 콘텐츠 품질은 직접 못 봤지만, 기출 키워드 연계는 좋은 방향이야.

### 용어학습

/terms 페이지는 과목별 용어 카드가 있어. 검색도 되고, 과목 필터도 되고. 홈의 "오늘의 단어" 카드를 누르면 바텀시트로 자세히 볼 수 있고, "용어학습 페이지에서 보기"로 이동 가능해.

**좋은 점:** 오늘의 단어 -> 바텀시트 -> 용어 페이지 연결이 자연스러워.
**나쁜 점:** "관련 문제 풀어보기"가 "준비 중"으로 비활성화되어 있어. 이건 기대를 주고 꺾는 패턴이야. 아예 안 보여주든지, 아니면 만들어놓든지.

### 점수

| 항목 | 점수 | 이유 |
|------|------|------|
| First Impression | 6/10 | 콘텐츠가 있는 건 알겠는데 어디서 시작할지 모름 |
| Clarity | 6/10 | 과목 나열은 명확하지만 우선순위 안내 없음 |
| Value | 7/10 | 기출 키워드 연계는 차별점 |
| Friction | 5/10 | 시작점이 불명확해서 고민하게 됨 |
| Come-back Factor | 5/10 | 능동적으로 찾아 들어가야 해서 습관화 어려움 |

## Journey 8: "이 사이트 계속 쓸까?" (종합 평가)

### 내일도 올 이유 1가지

**"오늘 복습할 게 N개 있어요"** -- 이 한 문장이 가장 강력한 리텐션 메커니즘이야. DailyReviewCard가 "플래시카드 3개, 미처리 오답 5개"를 보여주고 "지금 복습하기" 버튼이 있으면, "아 이거 처리하고 자야지"라는 마음이 생겨.

### 다시 안 올 이유 1가지

**합격 가능성 숫자가 너무 낮게 보이는데 대안이 안 보일 때.** "합격 가능성 12%"만 덜렁 보여주고, "그래서 뭘 하면 올라가는데?"라는 질문에 답이 없으면, 이 사이트는 나한테 "넌 부족해"만 말하는 곳이 돼. 그러면 안 와.

### 전문적 느낌 vs 취미 프로젝트

솔직히 말하면, **70% 전문적, 30% 프로토타입 느낌**이야.

전문적인 부분:
- 퀴즈 시스템의 깊이 (프리셋, 적응형 난이도, 콤보, XP)
- 오답 관리 (Leitner SRS, AI 분석, 패턴 감지)
- 기출 분석 (연도별/영역별/검색)
- PWA 지원 (manifest.json, appleWebApp)

프로토타입 느낌:
- "준비 중" 표시가 있는 미완성 기능들
- 통계 페이지의 정보 과부하
- 온보딩이 자동으로 안 뜸
- 몇몇 페이지에서의 전문 용어 (SRS, Leitner, BDS 시나리오)

### 데일리 루프가 있는가?

**있지만 약해.** 이상적인 루프는:

1. 오전: 앱 열기 -> 복습 알림(DailyReviewCard) -> 복습 완료
2. 학습: "오늘 학습 시작하기" -> OX/단답/서술 완료 -> 결과 확인
3. 심화: 약한 개념 학습 -> 기출 확인
4. 마무리: 오늘의 단어 확인, 스트릭 유지
5. 내일 다시 돌아옴

현재는 1, 2까지는 잘 되는데, 3, 4로의 연결이 약해. 학습 완료 후 "개념학습 보기" 링크가 있긴 한데, **내 약점에 맞는 개념으로** 데려다주지 않아. 그냥 /concepts 전체 목록으로 가.

## 데드엔드, 혼란점, 끊어진 연결, 감정적 추락

### 데드엔드 (다음 행동이 불분명한 곳)
1. **마스터리 빈 상태**: "아직 학습 기록이 없어요" 메시지 아래 퀴즈로 가는 명확한 CTA 부족
2. **플래시카드 빈 상태**: "오늘 복습할 카드가 없어요" + "카드 추가하기" / "돌아가기"인데, "돌아가기"가 /flashcards로 가면 다시 /flashcards/review로 리다이렉트돼서 무한루프 느낌
3. **통계 빈 상태**: "아직 학습 기록이 없어요" 메시지만 있고 CTA 없음

### 혼란점 (용어/네비게이션이 불명확한 곳)
1. **"진단평가" 탭**: 수험생에게 진단평가는 학교에서 학생에게 하는 거야. 내가 퀴즈를 푸는 건 "문제풀기"지 "진단평가"가 아님
2. **"BDS 시나리오"**: 커뮤니티 안에 있는데, BDS가 뭔지 설명 없음
3. **"SRS 복습"**: SRS(Spaced Repetition System)를 아는 사람이 얼마나 될까?
4. **서술형 채점**: 내가 쓴 답이 맞는지 틀린지 불명확. 셀프채점이라는 안내가 약함
5. **타임슬롯 개념**: "오전 세션"이 뭔지, 오후에 다시 오면 뭐가 달라지는지 모름

### 끊어진 연결 (연결되어야 하는데 안 되는 기능들)
1. **온보딩 설정 <-> Daily 문제 수**: 온보딩에서 20문항 설정해도 daily는 18문제 고정
2. **마스터리 약점 <-> 개념학습 추천**: 약한 과목이 보이는데 개념학습에서 추천이 안 됨
3. **기출 키워드 <-> 퀴즈 연계**: 기출에서 나온 키워드가 어떤 퀴즈와 연결되는지 안 보임
4. **오늘의 단어 <-> 퀴즈**: "관련 문제 풀어보기"가 비활성화

### 감정적 추락 지점
1. **"합격 가능성 12%"**: 맥락 없이 낮은 숫자만 보여줌 -> 좌절
2. **D-day 카운트다운이 매일 1씩 줄어듦**: 시간 압박 -> 불안
3. **통계 페이지에서 "정답률 32%"**: 잘못 공부한 느낌 -> 자기비하
4. **빈 화면들**: 뭔가를 해야 하는데 뭘 해야 하는지 모르겠음 -> 무력감

## Top 5: 사람 경험을 가장 크게 바꿀 수 있는 변화

### 1. 합격 시뮬레이션에 "맥락"과 "행동 제안" 추가

**지금**: "합격 가능성 12%" (빨간색, 큰 글씨)
**느낌**: "나 망했구나."

**바꾸면**: "현재 합격 가능성 12% -- 아직 학습하지 않은 과목이 7개 있어요. 매주 1과목씩 완성하면 시험 전 55%까지 도달 가능합니다. 이번 주 추천: 행동지원 챕터 3~5"
**느낌**: "아 지금은 낮지만 올릴 수 있구나. 이번 주에 행동지원부터 하면 되네."

**왜**: 숫자는 현실을 보여주되, 행동 가능한 다음 단계를 함께 제시해야 동기부여가 된다. 숫자만 보여주면 현실 확인이 아니라 현실 도피로 이어져.

### 2. 온보딩을 홈 첫 진입 시 자동 실행

**지금**: OnboardingGate 컴포넌트가 있지만 홈/레이아웃에 적용 안 됨. 신규 유저가 맨 앞의 D-day부터 봄.
**느낌**: "이 사이트가 나한테 맞는 건지 모르겠어. 일단 둘러보자... 뭐가 많네."

**바꾸면**: 첫 방문 시 "시험 언제예요? 어느 단계예요? 어떤 과목이 약해요?"를 먼저 물어보고, "당신을 위한 학습 계획이 만들어졌습니다!"로 시작.
**느낌**: "오 이 사이트가 나를 위한 맞춤 계획을 세워주네. 신뢰가 가."

**왜**: 온보딩은 이미 완성도 높게 만들어져 있어 (5단계, 시험일/수준/취약과목/목표). 이걸 안 쓰는 건 90% 완성된 기능을 서랍에 넣어둔 것과 같아.

### 3. "진단평가" 네비게이션 이름을 "문제풀기"로, 기출문제 접근 경로 단축

**지금**: 하단탭이 "홈 | 진단평가 | 실력쌓기 | 내 기록 | 함께하기". 기출문제는 3단계 깊이.
**느낌**: "'진단평가'가 뭐야? 기출문제는 어디 있어?"

**바꾸면**: "홈 | 문제풀기 | 기출분석 | 내 기록 | 더보기"로 재편. 기출을 메인 탭으로 올림.
**느낌**: "아, 문제 풀려면 여기, 기출 보려면 여기. 직관적이네."

**왜**: 수험생한테 기출문제는 "있으면 좋은 것"이 아니라 "없으면 안 되는 것"이야. 이걸 3단계 깊이에 묻어두면 안 돼.

### 4. 학습 완료 후 "다음 추천"을 약점 기반으로 제공

**지금**: Daily 완료 화면에서 "개념학습 보기" 링크 -> /concepts 전체 목록
**느낌**: "개념학습이라... 과목이 11개인데 어디서부터?"

**바꾸면**: "오늘 행동지원에서 3개 틀렸어요. [행동지원 > 긍정적 행동지원] 개념 복습하기" 로 약점 챕터 직링크
**느낌**: "아, 내가 약한 부분을 정확히 짚어주네. 바로 가서 읽어볼게."

**왜**: 이미 코드에 wrongChaptersStep1으로 틀린 챕터를 추적하고 있고, RecommendedChapters 컴포넌트도 있어. 이걸 더 적극적으로 "개념학습 직링크"로 연결만 하면 돼.

### 5. 통계 페이지에 "이번 주 핵심 3줄 요약" 추가

**지금**: 통계 페이지가 12개 이상의 섹션(정답률, 레벨, 배지, 주간요약, 히트맵, 과목별, 학습량, 주간트렌드, 약점, AI분석, 오답요약, 플래시카드, 스트릭)으로 가득함.
**느낌**: "통계 보러 왔는데 뭘 봐야 하는지 모르겠어. 스크롤이 끝이 없어."

**바꾸면**: 맨 위에 "이번 주 핵심"으로 3개만 크게: (1) "이번 주 정답률 67% (지난주 대비 +5%p)" (2) "가장 약한 과목: 행동지원" (3) "추천: 이번 주 행동지원 집중". 나머지는 "상세 통계 보기"로 접어둠.
**느낌**: "핵심만 딱 3개. 내가 뭘 해야 하는지 바로 알겠어."

**왜**: 정보가 많은 건 좋지만, 정보 과부하는 "아무것도 안 보는 것"과 같아. WeeklySummary 컴포넌트가 이미 있으니, 이걸 더 강조하고 나머지를 축소하면 돼.

## 마무리

카이란, 이 사이트는 **기능적으로는 상당히 잘 만들어져 있어**. 퀴즈 시스템의 깊이, Leitner SRS 자동 등록, 기출 분석, 적응형 난이도 -- 이런 건 경쟁 서비스에서도 보기 힘든 수준이야.

문제는 **감정적 설계**야. 수험생은 이미 불안하고, 자기 의심을 하고 있어. 이 사이트가 해줘야 하는 건 "너 부족해"가 아니라 **"지금 여기서부터 시작하면 돼. 오늘 이것만 하면 돼."** 라는 안심이야.

기능은 90점. 감정은 60점. 감정이 70점만 돼도, 이 사이트는 수험생 사이에서 입소문 탈 거야.
