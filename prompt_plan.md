## Tier 1: 단기 (Phase 5) — 신규 사용자 획득 + 학습 효율 극대화

신규 사용자 온보딩과 기존 사용자 효율 모두 잡는 기능들. 3~4세션.

### 1-0. 🆕 학습 여정 온보딩 + D-day 플래너 (최우선)
**개념**: "뭐부터 해야 할지 모르는" 초보 수험생을 위한 첫 경험 설계

**온보딩 플로우** (첫 방문 또는 /onboarding):
1. "시험까지 며칠 남았나요?" → 시험일 입력 (D-day 자동 계산)
2. "현재 어느 단계인가요?" → 입문(0회독) / 기본(1~2회독) / 심화(3회독+)
3. "어떤 과목이 가장 약한가요?" → 과목 2~3개 선택 (또는 "모르겠어요")
4. → **맞춤 학습 계획 자동 생성**

**학습 계획 생성 로직**:
- 임용고시 공고문 기반 과목별 출제 비중 데이터 (`data/exam-structure.json`)
  - 특수교육학(행동지원, 교육과정, 진단평가 등) 비중이 높음
  - 장애유형별 과목은 균등 배분
- D-day까지 남은 주(weeks) 계산
- 단계별 추천:
  - 입문(6개월+): 주차별 과목 순환 + 기초 퀴즈 → "이번 주: 특수교육개론 1~3챕터"
  - 기본(3~6개월): 기출 분석 병행 + 취약 과목 집중
  - 심화(3개월 미만): 모의고사 + 오답 집중 + 예상 문제
- 주간 마일스톤: "이번 주 목표: 행동지원 3챕터 완독 + 퀴즈 30문제"

**매일의 학습 추천** (`src/components/dashboard/TodayStudyPlan.tsx`):
- "오늘 할 일" 카드: (1) 개념 읽기 → (2) 퀴즈 풀기 → (3) 오답 복습
- 단계별로 난이도 자동 조절
- 완료 시 격려 메시지 + XP

**격려 시스템**:
- 학습 시작 전: "오늘은 [과목명]의 [챕터]를 공부합니다. 시험까지 D-XX일!"
- 진행 중: 진행률 프로그레스 바 + "절반 왔어요!"
- 완료 후: 주간 진행률 시각화 + "이번 주 목표의 60% 달성!"
- 미학습 시: "어제 쉬었네요. 오늘 10분만 투자해볼까요?"

**파일**:

## Tier 3: 장기 (Phase 7~8) — 창조적 혁신 + 교원 연수 기반

시장에 없는, 우리만의 킬러 피처. Phase별 1~2세션씩.

### 3-1. ✅ "행동 의사결정 시뮬레이터" (BDS) — 완료 (2026-03-16)
> **이것이 이 프로젝트의 창조적 핵심이다.**

**개념**: IEP/FBA/행동중재를 분기형 시나리오로 학습
- 의료교육의 Body Interact, Mursion에서 영감
- 하지만 VR이 아닌 **텍스트+선택지 기반 분기형 시나리오**

**시나리오 예시**:
```
[상황] 자폐스펙트럼 3학년 학생이 수업 중 자해행동(손으로 머리 치기)을 반복
  → [선택1] 즉시 물리적으로 제지한다
  → [선택2] ABC 데이터를 먼저 수집한다
  → [선택3] 대체행동을 가르친다

[선택2 선택 시]
  "좋은 선택입니다. 먼저 행동의 기능을 파악해야 합니다."
  → ABC 분석 결과: 선행(수학 과제 제시) → 행동(자해) → 결과(과제 회피)
  → "이 행동의 기능은 무엇입니까?"
    → [감각적 자극] → "이 상황에서는 과제 회피가 더 유력합니다. 다시 생각해보세요."
    → [과제 회피] → "정확합니다! 부적 강화에 의해 유지됩니다."
    → 다음 단계: BSP 작성...
```

**데이터 모델**:
```typescript
interface ScenarioNode {
  id: string
  type: 'situation' | 'choice' | 'feedback' | 'assessment'
  content: string
  choices?: { label: string; nextId: string; isOptimal: boolean; feedback: string }[]
  relatedConcepts: string[]  // 연결된 임용 키워드
  legalBasis?: string        // 관련 법령 조항
}
```

- **ScenarioCompositeChoice 구조를 확장**하여 구현 (기존 코드 활용)
- 시나리오 JSON을 `data/scenarios/` 디렉토리에 저장
- **교원 연수 확장**: 같은 시나리오 엔진으로 연수 콘텐츠 제공 가능
- **ABA 킬러 콘텐츠**: FBA→BSP 전 과정을 분기형으로 체험

### 3-2. ✅ "정교화 질문 모드" (Elaborative Interrogation) — 완료 (2026-03-16)
> **EdTech 리서치에서 발견한, 어디에도 없는 기능**

**개념**: O/X나 선다형 정답 후 "왜?" 후속 질문
- 기존: "정답: O" → 해설 표시 → 다음 문제
- 신규: "정답: O" → **"왜 이것이 맞는지 한 줄로 설명해보세요"** → AI 또는 키워드 매칭으로 설명 평가
- generation effect(스스로 생성) + retrieval practice(인출 연습)의 결합
- **창조적 변형**: edumind에도 gosari에도 없는, 학습과학 기반의 독자적 기능

**구현 방식**:
- 퀴즈 결과 확인 후 20% 확률로 정교화 질문 트리거
- `quiz_questions.explanation`에서 핵심 키워드 3개 자동 추출
- 사용자 입력에서 키워드 매칭 → 포함률로 이해도 평가 (0/3, 1/3, 2/3, 3/3)
- 향후 LLM API 연동으로 정밀 평가 가능

### 3-3. ✅ "스페이스드 시나리오" (Spaced Scenarios) — 완료 (2026-03-16)
> **EdTech 리서치의 "인터리빙 + 간격반복 + 시나리오"의 결합**

**개념**: 같은 원리를 다른 맥락에서 간격을 두고 반복
- 1회차: "자폐 초등학생의 자해행동 → DRO 적용"
- 3일 후 2회차: "지적장애 중학생의 이탈행동 → DRO 적용?"
- 7일 후 3회차: "정서행동장애 고등학생의 공격행동 → DRO vs DRA 비교"
- **핵심**: 같은 원리(DRO)를 다른 변수(장애유형, 학교급, 행동유형)로 전이(transfer) 훈련
- Leitner/FSRS 스케줄링과 결합
- **교원 연수에 직접 활용 가능** — 실제 교실 상황의 변형 시나리오

### 3-4. ✅ MDX 인터랙티브 컴포넌트 도입 — 완료 (2026-03-16)
**개념**: 정적 학습 콘텐츠를 능동적 학습으로 전환
- gosari의 7종 MDX 컴포넌트에서 선별 도입:
  - **MatchingExercise**: 특수교육 용어-정의 매칭 (예: 장애유형-특성 연결)
  - **FillBlank**: 법령 조항 빈칸 채우기
  - **StepGuide**: IEP 작성 절차 아코디언
- **창조적 변형**: gosari는 범용이지만, 우리는 **도메인 특화** — 법령 인용 자동 완성, 장애 특성 드래그 분류 등

### 3-5. ✅ 마스터리 트리 + 합격 시뮬레이션 — 완료 (2026-03-16)
**개념**: XP/스트릭을 넘어, 실질적 합격 가능성 시각화
- 11과목 × 39챕터 스킬 트리 (gosari의 React Flow 시각화 참고)
- 각 노드에 숙련도 (학습 중 / 연습 중 / 마스터)
- **합격 시뮬레이션**: 현재 과목별 정답률 + 커버리지 → "예상 점수 XX점 / 합격선 YY점"
- **D-day 학습 플래너**: 시험일까지 남은 일수 기반 일일 추천 학습량
- **교원 연수 확장**: "연수 수료 요건까지 N단계 남음" 으로 재활용

### 3-6. ✅ AI 서술형 채점 가이드 — 완료 (2026-03-16)
**개념**: 서술형 답안에 대한 구조화된 피드백
- AutoSCORE 방식: 채점 루브릭 → 핵심 요소 추출 → 요소별 매칭
- 1단계 (LLM 없이): 모범답안의 핵심 키워드 n개 중 사용자 답안에 포함된 비율 시각화
- 2단계 (LLM 연동): 키워드 + 논리 구조 + 법령 인용 정확성 평가
- **특수교육 특화**: IEP 목표 작성, 행동지원계획 등 정형화된 답안 형식에 특히 효과적
