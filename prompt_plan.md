## 요구사항

M2에서 "기능이 존재하는 것"과 "사용자가 만족하는 것"의 갭을 확인했다.
M3는 새 기능 추가보다 **기존 기능의 깊이와 품질**에 집중한다.
카이란이 직접 써보며 "이거 고쳐"라고 하면 즉시 수정하는 **실시간 리뷰 루프** 방식.

## 카이란 피드백 기반 백로그 (우선순위순)

### B1. /concepts 버그 수정 ★ BLOCKER

> 카이란: "개념학습만 작동 안 됨"

- [ ] 원인 조사 — MDX 로딩, 인코딩, 렌더링 중 어디서 실패하는지
- [ ] 수정 및 검증

### B2. 허브 → 통합 페이지 전환 (네비 2클릭 해소)

> 카이란: "탭 1클릭으로 원하는 기능에 도달해야 한다"

- [ ] `/diagnosis` → 서브기능 임베드 (퀴즈 시작 + 최근 결과)
- [ ] `/practice-hub` → 서브기능 임베드 (진도 + 바로가기)
- [ ] `/my` → 서브기능 임베드 (통계 요약 + 각 "더보기")

### B3. 플래시카드 UX 재설계

> 카이란: "클릭하여 답을 맞추는 구조가 이상함"

- [ ] 인터랙션 단순화 — 답 버튼을 flip 전에도 보이게 or swipe-only
- [ ] 진행률 표시 개선

### B4. 커뮤니티 AI 보조 문제 만들기

> 카이란: "수동 4단계 위저드가 뭘 어떻게 하라는지 모호함"

- [ ] AI를 앞으로 이동 — 과목+유형 선택 후 AI가 초안 생성
- [ ] 사용자는 수정/확인 → 제출

### B5. 출제경향 탭 간소화

> 카이란: "4탭 과다" (이미 2탭으로 축소됨 — 현재 상태 확인 필요)

- [ ] 카이란과 논의 후 결정

### B6. 북마크 용도 안내

> 카이란: "뭘 북마크하는지 모호함"

- [ ] "어디서 북마크하나요?" 안내 + 가이드

### B7. localStorage 안내 문구

> 기기 간 데이터 미동기화 임시 대응

- [ ] "학습 기록은 이 기기에서만 유지됩니다" 안내

## 리뷰 루프 운영

카이란 사용 → 피드백 → X 즉시 수정(1커밋) → 재확인 → 만족/재수정

## Completion Contract

V(평가자)가 이 기준으로 PASS/FAIL을 판정한다. 80% 이상 통과해야 PASS.

### 기능 기준
- [ ] /concepts/[subject] 페이지에서 MDX 콘텐츠가 정상 렌더링된다
- [ ] /concepts/[subject]/[slug] 페이지에서 학습 콘텐츠가 표시된다
- [ ] /diagnosis에서 1클릭으로 퀴즈 시작 가능 (허브 카드 없이)
- [ ] /my에서 학습통계·오답·북마크 현황을 바로 확인 가능
- [ ] 커뮤니티 문제 만들기에서 AI가 초안을 생성한다

### UX 기준
- [ ] 플래시카드에서 "다음에 뭘 해야 하는지" 명확하다
- [ ] 북마크 페이지에 안내가 있다
- [ ] 내기록 페이지에 "이 기기에서만 유지" 안내가 있다
- [ ] 출제경향이 2~3탭 이하로 간결하다

### 만족도 기준 (카이란 재평가)
- [ ] 진단평가 50%+ (현재 30%)
- [ ] 실력쌓기 40%+ (현재 30%)
- [ ] 내기록 35%+ (현재 20%)
- [ ] 함께하기 20%+ (현재 5%)

### 빌드 기준
- [ ] `npm run build` exit 0
- [ ] `npm run test` 전체 통과
