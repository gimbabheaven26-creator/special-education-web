## 1. ABA 기반 교수법 비교

### 1-1. 비연속 개별시도 교수 (DTT: Discrete Trial Training)

ABA 원리를 **구조화된 환경**에서 집중적으로 적용. 한 번에 하나의 기술을 분리하여 반복 교수.

| 구성 요소 | 내용 |
|---------|------|
| **변별 자극 (SD)** | 교사의 지시 ("이것 보여줘", 그림 제시) |
| **반응 (R)** | 학생의 행동 |
| **결과 (SR)** | 정반응 → 강화 / 오반응 → 교정 |
| **시행 간 간격 (ITI)** | 다음 시도 전 짧은 휴식 (1~5초) |

**특징:**
- 구조화된 환경 (1:1 또는 소집단)
- 집중적 반복 (동일 기술 다수 시행)
- 즉각적이고 일관된 피드백
- 과제 분석 후 작은 단계로 분해

**한계:** 일반화가 어려움 → **자연적 맥락에서의 연습 필요**.

---

### 1-2. 중심축 반응 훈련 (PRT: Pivotal Response Training)

자연적 환경과 아동의 **동기·선택**을 활용하여 핵심 반응 영역(pivotal areas)을 중재.

| 중심축 영역 | 설명 |
|-----------|------|
| **동기 (Motivation)** | 아동의 관심·선택을 따름 → 학습 동기↑ |
| **복수 단서 반응** | 여러 환경 단서에 동시 반응하는 능력 |
| **자기 관리** | 자신의 행동을 스스로 모니터·강화 |
| **자기 주도 사회적 상호작용** | 스스로 사회적 상호작용 시작 |

| 구분 | DTT | PRT |
|-----|-----|-----|
| **환경** | 구조화된 교수 환경 | 자연적·놀이 환경 |
| **주도** | 교사 주도 | 아동 관심 따름 |
| **강화** | 임의적 강화물 | 자연적·직접적 강화 |
| **일반화** | 추가 프로그램 필요 | 자연스럽게 촉진 |

## 2. 의사소통 지원

### 2-1. 그림 교환 의사소통 체계 (PECS: Picture Exchange Communication System)

사회적 교환의 원리를 활용하여 **자발적·기능적 의사소통**을 체계적으로 교수. 언어 발달이 어려운 학생에게 AAC 지원.

> ⚠️ **기출 포인트**: PECS 6단계 순서와 각 단계의 목표를 정확히 숙지할 것.

| 단계 | 목표 | 핵심 활동 |
|-----|------|---------|
| **1단계: 교환 방법** | 원하는 물건의 그림을 상대에게 건네 교환함 | 보조자(촉진자)가 뒤에서 신체 촉진, 상대가 앞에서 수용 |
| **2단계: 거리와 지속성** | 거리가 멀어져도 자발적으로 의사소통 파트너에게 다가가 교환 | 의사소통 책(binder) 가지고 파트너에게 이동 |
| **3단계: 변별** | 여러 그림 중 원하는 그림을 변별하여 교환 | 동일/유사 그림 변별, 오류 교정 절차 |
| **4단계: 문장 구조** | "나는 ~을/를 원해요" 문장 띠(sentence strip) 구성 후 교환 | 문장 시작 카드 + 원하는 물건 카드 |
| **5단계: 요구에 반응** | "무엇을 원해?" 질문에 문장으로 반응 | 자발적 요구 + 질문에 반응 모두 |
| **6단계: 논평** | "무엇이 보여?"와 같은 논평 질문에 반응 | "나는 ~이/가 보여요" 등 다양한 기능으로 확장 |

---

### 2-2. 기능적 의사소통 훈련 (FCT: Functional Communication Training)

문제행동과 **동일한 기능**을 수행하면서 사회적으로 용인되는 **적절한 의사소통 대체 행동**을 교수.

**절차:**
1. 기능행동평가(FBA) → 문제행동의 기능 파악 (관심, 회피, 자기자극, 접근)
2. 동일 기능의 적절한 의사소통 반응 선정
3. 적절한 반응 교수 + 강화
4. 문제행동에 대한 강화 소거

| 문제행동 기능 | 대체 의사소통 예시 |
|------------|----------------|
| 관심 얻기 | "저 봐주세요" 카드 제시 또는 탭하기 |
| 과제 회피 | "쉬고 싶어요" 카드 또는 "도와주세요" |
| 물건 요구 | PECS 그림 교환 또는 AAC 기기 사용 |

## 3. 사회적 기술 지원

### 3-1. 사회적 상황 이야기 (Social Story)

특정 사회적 상황과 관련된 **사회적 단서와 적절한 반응**을 설명해 주는 개별화된 인지적 중재.

**Gray(그레이)의 5가지 문장 유형:**

| 문장 유형 | 비율 | 역할 | 예시 |
|---------|------|------|-----|
| **서술 문장** | 필수 (다수) | 상황·배경·사람들의 생각·감정을 객관적으로 기술 | "점심시간에 교실에서 밥을 먹는다." |
| **관점 문장** | 권장 | 다른 사람의 생각·느낌·믿음 기술 | "선생님은 학생들이 조용히 먹기를 원한다." |
| **지시 문장** | 0~1개 | 학생이 해야 할 행동을 직접 제안 | "나는 조용히 먹으려고 노력할 것이다." |
| **확인 문장** | 선택 | 중요한 내용 재확인 | "규칙을 기억하는 것은 좋은 일이다." |
| **통제 문장** | 선택 | 기억 전략 제공, 학생이 직접 작성 | "이 이야기는 나를 도와줄 것이다." |

> ⚠️ **기출 포인트**: 지시 문장은 0~1개. 사회적 상황 이야기는 규칙을 강요하는 것이 아니라 상황을 **이해**하도록 돕는 것이 목적.

**작성 원칙:**
- 1인칭으로 작성 (학생 관점)
- 긍정적 문장 사용 ("~하지 마라" 대신 "~할 것이다")
- 학생의 시각적 강점 활용 (그림·사진 포함)

---

### 3-2. 파워카드 전략 (Power Card Strategy)

학생의 **특별한 관심 영역** (영웅, 캐릭터, 좋아하는 인물)을 활용하여 사회적 상황에 적합한 **스크립트**를 지도하는 시각적 접근.

**구성:**
1. **영웅 시나리오** (짧은 이야기): 관심 인물도 같은 사회적 어려움을 경험했고, 3~5단계로 해결한다는 내용
2. **파워카드** (명함 크기): 관심 인물 그림 + 3~5단계 전략 요약

**예시:** 기차를 좋아하는 학생 → "기차 기관사도 차례를 지킨다 → (1) 줄 서기, (2) 기다리기, (3) 내 차례가 오면 탑승"

## 4. 구조화된 교수 (TEACCH)

### 4-1. 환경의 구조화 (Physical Structure)

물리적 환경을 명확히 구분하여 **각 영역의 기능을 예측가능하게** 만드는 것.

| 원칙 | 내용 |
|-----|------|
| **영역 구분** | 각 활동 영역을 시각적으로 명확히 구분 (칸막이, 카펫, 테이프 등) |
| **방해자극 최소화** | 과제 관련 없는 시각적·청각적 자극 제거 |
| **예측 가능성** | 항상 같은 자리에서 같은 활동 |

### 4-2. 시각적 일정표 (Visual Schedule)

> ⚠️ **기출 포인트**: 시각적 일정표는 예측가능성 제공 → 변화에 대한 저항 감소. 자폐성 장애 학생은 예측가능성이 일관성보다 더 중요하므로, 변화를 시각적으로 사전 고지하는 것이 효과적이다.

| 형태 | 적용 수준 |
|-----|---------|
| 실물 | 인지 수준이 낮은 경우 |
| 사진 | 중간 수준 |
| 그림/그림문자 | 그림 이해 가능한 경우 |
| 글자 | 문자 이해 가능한 경우 |

### 4-3. 작업 시스템 (Work System)

독립적 작업을 위해 다음 4가지 정보를 시각적으로 제공:

| 제공 정보 | 질문 | 시각적 표현 예시 |
|---------|------|--------------|
| **무엇을** | 어떤 과제를 해야 하나? | 과제 바구니 라벨 |
| **얼마나** | 얼마나 많이/얼마나 오래? | 과제 수 표시 (1, 2, 3…) |
| **언제 끝나나** | 작업이 언제 완료되나? | 완료 바구니, "완료" 표시 |
| **다음은** | 끝나면 무엇을 하나? | 다음 활동 카드 |

**핵심:** 작업 시스템은 **이미 숙달한 과제**로 구성 → 독립적 수행 연습.

### 4-4. 과제 구조화 (Task Organization)

각 과제 내에서 정보를 명확히 조직하여 독립적 수행 지원.

| 전략 | 내용 |
|-----|------|
| **시각적 지시** | 단계별 그림·사진 지시서 |
| **왼쪽→오른쪽** | 과제를 왼쪽에서 오른쪽으로 수행하도록 구조화 |
| **위→아래** | 과제를 위에서 아래로 순서화 |
| **완료 바구니** | 완료된 과제를 명확히 분리 |
