## 1. 보조공학 기기 분류

### 1-1. 기술 수준별 분류

| 유형 | 정의 | 예시 |
|------|------|------|
| **No-tech** | 기기 없음 | 몸짓, 표정, 발성 |
| **Low-tech** | 전력 불필요 또는 단순 전자기기 | 그림 카드, 의사소통판, 손잡이 달린 숟가락 |
| **Mid-tech** | 단순 전자기기 | 단일 메시지 음성 출력기(BIGmack), 디지털 알람 |
| **High-tech** | 복잡한 전자기기·소프트웨어 | SGD(음성생성기기), 전동 휠체어, 스위치+AAC 앱 |

> ⚠️ **기출 포인트**: "보조공학 기기를 기술 수준에 따라 분류"하는 문항 자주 출제. 예시와 함께 암기.

---

### 1-2. 기능별 분류 (Cook & Polgar)

| 기능 | 예시 기기 |
|------|----------|
| **이동 보조** | 수동/전동 휠체어, 워커, 목발 |
| **자세 보조** | 자세보조기기(PBS), 특수 의자, 쐐기형 방석 |
| **의사소통 보조** | SGD, AAC 앱, 의사소통판 |
| **컴퓨터 접근** | 특수 키보드, 스위치, 트랙볼, 마우스 키 |
| **일상생활 보조** | 유니버설 컵, 수정 식기, 드레싱 에이즈 |
| **학습 보조** | 전자 독서대, 페이지 터너, 음성 녹음기 |

## 2. SETT 프레임워크

**목적**: 보조공학 기기 선택 시 체계적 요구 평가

### 2-1. 4요소

| 요소 | 내용 | 핵심 질문 |
|------|------|----------|
| **S — Student(학생)** | 학생의 능력·제한·요구 | "학생이 무엇을 할 수 있고 무엇이 필요한가?" |
| **E — Environment(환경)** | 물리적·사회적·교수 환경 | "학생이 학습하는 환경의 특성은?" |
| **T — Task(과제)** | 참여해야 할 기능적 활동 | "수행해야 할 과제는 무엇인가?" |
| **T — Tools(도구)** | 필요한 보조공학 기기·서비스 | "어떤 보조공학이 참여를 가능하게 하는가?" |

> ⚠️ **기출 포인트**: SETT는 기기 선택 **전** 실시하는 과정. 도구(Tools)가 마지막 단계인 것이 핵심 — 기기부터 결정하지 않음.

### 2-2. SETT 적용 순서

```
1. 학생 평가 (능력, 요구, 선호)
2. 환경 분석 (교실 배치, 지원 인력, 기술 인프라)
3. 과제 명시 (수행해야 하는 구체적 활동 목록)
4. 도구 선택 (앞의 세 요소를 종합하여 최적 AT 결정)
```

## 3. 컴퓨터 접근 기술

### 3-1. 입력 장치 유형

| 유형 | 접근 방법 | 대상 |
|------|----------|------|
| **스위치** | 단일 스위치(One-switch) → 스캐닝 | 손 사용 불가, 미세운동 제한 심함 |
| **트랙볼** | 볼 회전으로 커서 이동 | 손 떨림·미세운동 제한 |
| **조이스틱** | 레버 방향 이동 | 손 근력 약하지만 방향 조절 가능 |
| **터치스크린** | 직접 화면 접촉 | 조준 능력이 있는 경우 |
| **안구 추적(Eye Gaze)** | 시선으로 커서 제어 | 상지 기능 전무, 경추 손상 |
| **구강 스틱(Mouth Stick)** | 입으로 막대 조작 | 손 기능 없지만 구강 조절 가능 |

---

### 3-2. 스위치 접근법

**스캐닝(Scanning)**: 스위치 1개로 순차적 선택지를 검토하여 원하는 항목에서 스위치를 누름

| 스캐닝 유형 | 방법 |
|------------|------|
| **자동 스캐닝** | 일정 시간마다 자동 이동 → 원하는 곳에서 활성화 |
| **단계 스캐닝(Step Scanning)** | 스위치 한 번 = 한 칸 이동, 두 번 = 선택 |
| **역 스캐닝(Inverse Scanning)** | 스위치 누른 동안 이동, 놓으면 선택 |

**스위치 배치 부위 예시**:
- 발판 스위치(Foot Switch): 발로 조작
- 헤드 스위치: 머리 기울임으로 조작
- 근전도(EMG) 스위치: 근육 수축 신호로 조작
- 호흡 스위치(Sip-Puff): 빨고 불기로 조작

---

### 3-3. 소프트웨어 접근성 지원

- **화상 키보드(On-Screen Keyboard)**: 마우스·스위치로 화면 키보드 입력
- **예측 입력(Word Prediction)**: 단어 첫 글자로 후보 표시 → 입력 횟수 감소
- **드웰 클릭(Dwell Click)**: 커서 일정 시간 정지 → 자동 클릭 (손 떨림 있는 경우)
- **접착 키(Sticky Keys)**: 단축키 동시 누름 불가 시 순차 입력 허용

## 4. AAC 접근 기법

### 4-1. 직접 선택 vs 간접 선택

| 구분 | 직접 선택 | 간접 선택(스캐닝) |
|------|----------|-----------------|
| **방법** | 신체 부위로 직접 항목 지목 | 자동/수동 스캐닝 후 스위치 활성화 |
| **속도** | 빠름 | 느림 |
| **요구 운동** | 조준 능력 필요 | 타이밍 능력 필요 |
| **적용** | 손/발/머리로 가리키기 가능한 경우 | 미세운동 심한 제한, 스위치 사용 |

### 4-2. 코딩(Encoding) 기법

| 기법 | 설명 |
|------|------|
| **알파뉴메릭 코딩** | 숫자+문자 조합 코드로 메시지 호출 |
| **아이콘 시퀀싱** | 그림 아이콘 연속 선택으로 메시지 구성 |
| **모스 코드** | 짧고 긴 신호 조합 |

> 코딩 기법은 **상징 수가 적은 기기에서 많은 어휘를 표현**하기 위한 방법.

## 5. 보조공학 팀 접근

```
특수교사 — 교육적 필요 파악
물리치료사 — 자세·이동 보조기기
작업치료사 — 상지기능·일상생활 보조기기, 컴퓨터 접근
언어치료사 — AAC·의사소통 기기
부모/보호자 — 가정 환경·선호 정보
학생 본인 — 선호·피드백 (자기결정)
```

> ⚠️ **기출 포인트**: 보조공학 팀 접근에서 **학생 본인 포함** 여부 자주 출제. 자기결정권 원칙과 연결됨.
