## 1. 한국형 웹 접근성 지침 (KWCAG)

한국형 웹 접근성 지침은 장애인과 고령자를 포함한 **모든 사용자**가 웹 콘텐츠에 동등하게 접근하고 이용할 수 있도록 보장하는 것을 목표로 한다.

> ⚠️ **기출 포인트**: 4원칙의 명칭과 각 원칙의 구체적 적용 사례(예: 건너뛰기 링크 → 운용의 용이성)를 연결 짓는 문제가 자주 출제된다. 원칙별 예시를 반드시 암기할 것.

---

## 2. 웹 접근성 4원칙

**한국형 웹 접근성 지침 2.1** 기준: **4원칙 — 13지침 — 24검사항목**

| 원칙 | 핵심 개념 | 대표 예시 |
|------|-----------|-----------|
| **인식의 용이성** | 모든 콘텐츠를 장애 유무와 관계없이 동등하게 인식 가능하도록 제공 | 멀티미디어 자막·원고 제공, 색상 외 구분 수단 제공 |
| **운용의 용이성** | 웹 사이트의 모든 기능을 장애 유무와 관계없이 운용 가능하도록 제공 | 키보드만으로 운용, 건너뛰기 링크, 번쩍임 제한 |
| **이해의 용이성** | 웹 사이트 콘텐츠를 장애 유무와 관계없이 이해 가능하도록 제공 | 명확한 오류 안내, 일관된 내비게이션 |
| **견고성** | 현재 및 미래 기술에서도 콘텐츠를 오류 없이 이용 가능하도록 제공 | 표준 준수, 보조 기술 호환성 확보 |

## 3. 원칙별 세부 지침

### 3.1 인식의 용이성

- 멀티미디어 콘텐츠(영상, 음성)에 **자막 또는 원고** 등 대체 수단을 제공해야 한다.
- 웹 정보는 **색깔을 배제하더라도 구분** 가능해야 한다 (색맹·색약 사용자 고려).
- 이미지에 **대체 텍스트(alt text)**를 제공하여 화면 낭독기 사용자가 내용을 인식할 수 있도록 한다.

### 3.2 운용의 용이성

- 웹 상의 모든 운용은 **키보드만으로 가능**해야 한다.
- `Tab` 키 이용 장애 학생을 위해 콘텐츠를 **왼쪽→오른쪽, 위→아래의 논리적 순서**로 선형화해야 한다.
- 팝업창의 자동 깜빡거림을 제한하여 **번쩍임이 없도록** 해야 한다 (광과민성 발작 예방).
- 반복 내비게이션 링크를 건너뛰어 핵심 부분으로 바로 이동할 수 있는 **건너뛰기 링크**를 제공해야 한다.
- 링크 텍스트는 '여기'가 아닌 **목적지를 명확히 알 수 있는 텍스트**로 제공해야 한다.

### 3.3 이해의 용이성

- 사용자가 웹 사이트에서 제공하는 콘텐츠를 **이해할 수 있도록** 제공해야 한다.
- 오류 발생 시 명확한 안내 메시지를 제공하여 사용자가 스스로 수정할 수 있도록 한다.

### 3.4 견고성

- **관련 표준을 준수**하여 앞으로 사용될 기술에서도 오류가 발생하지 않도록 해야 한다.
- 보조 기술(화면 낭독기, 점자 디스플레이 등)과의 **호환성**을 확보해야 한다.

## 4. 웹 접근성 준수의 필요성

| 측면 | 내용 |
|------|------|
| **법적 요구사항 준수** | 웹 접근성은 법적 의무사항 (장애인차별금지법 등) |
| **이용권 확대** | 장애인, 노인 등 다양한 이용자의 동등한 웹 이용 보장 |
| **이용 상황 확대** | 새로운 장소·기기 환경 등 다양한 이용 상황 대응 |
| **설계 효율성 제고** | 접근성을 고려한 설계는 전반적인 효율성 향상 |
| **비용 절감** | 사전 접근성 고려 시 사후 수정 비용 절감 |
| **홍보 효과** | 기업·기관의 긍정적 이미지 형성 |
| **균등 기회 보장** | 모든 사람에게 동등한 정보 접근 기회 제공 |
