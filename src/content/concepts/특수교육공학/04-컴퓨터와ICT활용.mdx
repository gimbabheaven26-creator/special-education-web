## 4. 특수교육과 컴퓨터의 활용

컴퓨터 보조수업(CAI)과 멀티미디어 활용 수업은 장애 학생의 개별화된 학습을 지원하고 동기를 유발하며, 효과적인 교육용 소프트웨어는 학습 목표에 적합하고 학습자의 특성을 고려하여 개발 및 평가되어야 한다.

| 주요 영역 | 핵심 개념 |
|---|---|
| 컴퓨터 보조수업(CAI) | 7가지 유형, 장애 학생 적합 조건, 장단점 |
| 멀티미디어 활용 수업 | Mayer의 인지 이론, 7가지 설계 원칙 |
| 교육용 소프트웨어 | 외부·내부 평가, 효과적 프로그램 특징 |
| ICT 활용 교육 | 5가지 특성, 장단점 |

---

### 4.1. 컴퓨터 보조수업(CAI)

**컴퓨터 보조수업(CAI)**은 컴퓨터를 직접 교수 매체로 활용하여 교과 내용을 가르치고 평가하는 방법이다.

#### CAI의 7가지 유형

| 유형 | 주요 특징 | 활용 시점 |
|---|---|---|
| **개인교수형** | 학습 목표 제시 → 내용 제시 → 연습 문제 → 피드백 순서로 진행 | 새로운 지식·기술을 처음 가르칠 때 |
| **반복연습형** | 학습 내용 정착·숙련도 향상 목적 | 새로운 지식·기술 습득 이후 |
| **시뮬레이션형** | 실제와 유사한 환경을 컴퓨터로 구현 | 비용·위험 부담이 높은 과제 |
| **게임형** | 경쟁·도전·흥미 요소 포함, 능동적 학습 참여 유도 | 동기 유발이 필요한 상황 |
| **문제해결형** | 정보 수집 → 문제 진술 → 가설 설정 → 실험 → 해결안 도출 과정 | 복잡한 문제 해결 학습 |
| **발견학습형** | 귀납적 방법, 학습자가 가설 설정 후 데이터베이스에 질문하며 검증 | 탐구·시행착오 학습 |
| **자료안내형** | 필요한 자료를 안내하고 탐색하도록 지원 | 정보 탐색·참고 활동 |

> ⚠️ **기출 포인트**: CAI 유형 7가지의 명칭과 특징이 출제된다. 특히 **발견학습형**의 '귀납적 방법·가설 검증', **시뮬레이션형**의 '비용·위험 부담' 적용 조건을 구별할 것.

#### 장애 학생의 요구에 적합한 CAI 조건

| 조건 영역 | 세부 내용 |
|---|---|
| **프로그램 내용** | 교수 목표 명백히 제시, 학습 내용 단계적·연관성 있음. 교사가 쉽게 변형 가능한 옵션 제공. 개별 학생 IEP 목표와 일치 |
| **학습자 요구** | 학습 수준·집중력·신체 능력에 적합. 보조기기 지원 가능, 화면 문자 크기 조절, 응답 시간 충분히 제공 |
| **교수 제시 방식** | 신호 체계(화살표·커서) 눈에 띄게 표시. 시작·중지·다음 단계 전환 용이. 도움말·그림 수시로 열람 가능. 진행 속도·수준 조절 가능. 오답 피드백 충실, 정반응 시 강화 제공 |
| **기술적 특징** | 그림·색깔·소리가 내용과 일치. 잘못된 키보드 입력 처리 및 소프트웨어 결손 여부 확인 |

#### CAI의 장단점

| 구분 | 내용 |
|---|---|
| **장점** | 개인별 수준·속도에 맞는 학습 가능, 흥미로운 동기 유발, 주의 집중 향상, 비위협적 학습 환경, 장애·비장애 학생 간 상호작용 촉진, 성취도 기록 용이, 즉각적 피드백 |
| **단점** | 고품질 프로그램 개발·선정 어려움, 기계적 정답 찾기로 전락 가능성, 교수-학습 상황에서 정서적 교류 부족 |

### 4.2. 멀티미디어 활용 수업

**멀티미디어**는 다양한 매체를 이용하여 언어적 정보와 회화적 정보를 함께 제시하는 것이다.

| 구분 | 내용 |
|---|---|
| **장점** | 실제와 유사한 간접 경험 제공, 풍부한 자료 제공, 학습 개별화 촉진, 동기 유발·자신감 부여, 비용 절감, 안전성 증대, 훈련 시간 단축 |
| **단점** | 하드웨어·소프트웨어 구입 비용 높음, 장비·프로그램 사용 연한 짧음, 새로운 기술 습득 필요, 정의적 영역 학습 효과 미미 |

#### Mayer의 멀티미디어 학습 인지 이론

유의미 학습을 위해 학습자들은 언어적(청각적) 정보와 시각적(회화적) 정보를 통합해야 하며, **함께 제시할 때 가장 효과적**이다.

**세 가지 가정**

| 가정 | 내용 |
|---|---|
| **이중 경로** | 인간은 시각적 정보와 언어적 정보를 분리된 경로를 통해 저장한다 |
| **제한된 능력** | 한 경로 내에서 동시에 처리할 수 있는 정보의 양은 제한적이다 |
| **역동적 과정** | 관련 정보 습득·조직화, 선행 지식과의 통합 등에 적극적으로 참여함으로써 학습한다 |

**Mayer의 멀티미디어 설계 7원칙**

| 원칙 | 내용 |
|---|---|
| **멀티미디어의 원칙** | 언어적 정보와 시각적 정보가 **동시에** 제시될 때 더 나은 학습 |
| **공간적 접근의 원칙** | 언어적 정보와 시각적 정보가 **공간적으로 인접**할 때 더 나은 학습 |
| **시간적 접근의 원칙** | 언어적 정보와 시각적 정보가 **시간적으로 동시에(유사하게)** 제시될 때 더 나은 학습 |
| **일관성의 원칙** | 학습과 **관련 없는** 단어·삽화·소리가 제외될 때 더 나은 학습 |
| **형식의 원칙** | 애니메이션과 **내레이션**이 동시에 제시될 때 더 나은 학습 |
| **잉여의 원칙** | 애니메이션+내레이션 조합이 애니메이션+텍스트+내레이션 조합보다 더 나은 학습 |
| **개인차의 원칙** | 학습 **숙련도가 낮은** 학습자에게 더욱 효과적 |

> ⚠️ **기출 포인트**: Mayer의 7가지 설계 원칙은 이름과 내용을 매칭하는 형태로 자주 출제된다. **잉여의 원칙**(텍스트를 빼는 것이 낫다), **일관성의 원칙**(불필요한 요소 제거)을 특히 구별할 것.

### 4.3. 교육용 소프트웨어의 선정과 평가

교육용 프로그램의 선정은 **초학문적 팀**(특수교사, 일반교사, 학부모, 공학 전문가)에 의해 이루어져야 하며, 외부 평가와 내부 평가를 거쳐 최종 선정된다.

#### 프로그램 선정 고려사항 5가지

정확성 · 피드백 · 학습자 통제 · 선수 학습 · 사용의 용이성

#### 외부 평가 vs 내부 평가

| 구분 | 외부 평가 | 내부 평가 |
|---|---|---|
| **주체** | 외부 전문가 팀 | 학급 단위 교사 |
| **초점** | 프로그램의 기술·공학 (거시적·종합적) | 수업 관련 일반 사항 + 공학 기기 적합성 (개별 학생 중심) |
| **평가자 자질** | 특수교육 대상자 전문 지식, 교과 지도 경험, 특수교육-공학 상호관계 이해 | — |

**내부 평가 고려사항**

| 영역 | 세부 항목 |
|---|---|
| **수업 정보** | 학습자와 교수자의 특성 |
| **교육 적절성** | 교수-학습 장면 적합성, 학습자 특성 부합, 상위 사고 기능 촉진, 반복 학습 고려 |
| **공학 기기 적합성** | 화면 구성, 입력 장치 조정, 기타 장비 필요성, 학습 능력·참여 증진, 비용·시간 효과성 |

#### 기술적 평가 vs 교육적 평가

| 구분 | 평가 항목 |
|---|---|
| **기술적 측면** | 현재 환경에서 프로그램 구동 여부, 오류 여부, 사용 설명서 구비, 화면 색상·음질·디자인 품질 |
| **교육적 측면** | 학습 목표 달성 위한 내용 체계성, 불필요·혼란 내용 여부, 어휘 수준 적합성, 학습자와 상호작용 적절성, 학습자 관심 유발 여부 |

#### 효과적인 교수용 프로그램의 특징

- 학습 목표에 맞는 주제, 컴퓨터 특성에 맞게 구성
- 학습자 흥미 수준에 적합하여 동기 제공
- 좌절감·실패 유도 금지, **도전의 기회** 제공
- 능동적 참여 가능, 상호작용 장면 제공
- 피드백과 강화가 장애 학생에게 적절
- **개별적** 상호작용 이루어져야 함
- 기능 수준이 낮을수록 청각·시각·촉각 통제가 더 중요
- 지시문 선택·통제는 사용자 **능력 한도 내**에서 제공
- 화면 유형은 학습자의 기능적 수준·학습 요구 반영
- 오류 확인 가능, 잘못된 입력으로 프로그램이 쉽게 종료되면 안 됨

> ⚠️ **기출 포인트**: 교육용 소프트웨어 평가에서 **초학문적 팀** 구성과 **외부·내부 평가의 역할 구분**이 출제된다. 외부 평가는 '거시적·기술 중심', 내부 평가는 '개별 학생·수업 중심'으로 대비할 것.

## 5. ICT 활용 교육

ICT 활용 교육은 학습자의 자율성과 자기 주도적 학습을 촉진하고 창의력 및 문제 해결 능력 향상에 기여하지만, 특수교육 현장의 인프라 부족과 프로그램의 양적·질적 한계라는 단점을 가지고 있다.

**ICT**: 정보 기술(IT)과 통신 기술(CT)의 합성어. 하드웨어 및 운영 소프트웨어 기술과 이를 이용한 정보 수집·생산·가공·보존·전달·활용 모든 방법을 의미한다.

### 5.1. ICT 활용 교육의 5가지 특성

| 특성 | 내용 |
|---|---|
| **학습자 자율성·유연한 학습** | 지식 전달 위주·교실 중심에서 벗어나 학습자의 자율과 특성을 존중하며 다양하고 유연한 학습 활동 수행 |
| **자기 주도적 학습 환경** | 정보 검색·의견 교환을 통해 학습 목표·방법 수립·평가 등 일련의 과정에서 학습자에게 주도적 역할 부여 |
| **창의력·문제 해결 능력 향상** | 정보 검색·수집·분석·종합 등 새로운 정보 창출 과정에 직접 참여하여 창의력과 문제 해결 능력 신장 |
| **다양한 교수-학습 활동 유발** | 문제 해결 학습, 프로젝트 학습, 상황 학습, 협동 학습 등 다양한 수업 활동 지원 |
| **교육의 장 확대** | 시·공간의 제약 극복(예: 웹 기반 가상 교육), 사고의 폭 확대, 고차적 사고 능력 신장 |

### 5.2. ICT 활용 교육의 장단점

| 구분 | 내용 |
|---|---|
| **장점** | 다양한 교수-학습 방법 적용 가능, 필요한 정보 접근·활용, 학습 동기 부여, 주의 집중이 어려운 장애 학생의 문제 극복 가능성 향상 |
| **단점** | ① 특수교육 현장의 교육 정보화 인프라가 장애 학생 교육에 적절하지 못함 ② ICT 활용 교육 프로그램의 양적·질적 부족 ③ 교육과정과 ICT 활용 교육 프로그램 간 연결 고리 희박 |

> ⚠️ **기출 포인트**: ICT 활용 교육의 **5가지 특성**이 서술형으로 출제된다. '자기 주도적 학습 환경'과 '학습자 자율성'의 차이, 그리고 단점 3가지(인프라·프로그램 양질·교육과정 연결 부족)를 명확히 구분할 것.
