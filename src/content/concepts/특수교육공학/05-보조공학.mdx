## 핵심 요약

보조공학은 장애인의 기능적 능력을 유지·향상시키기 위한 **장치(device)** 와 **서비스(service)** 의 총칭이다. 사정 시에는 HAAT·SETT 등 체계적 모델을 활용하여 인간-활동-환경의 조화를 먼저 탐색한다.

---

## 1. 보조공학의 이해

### 1.1 정의

| 구분 | 내용 |
|------|------|
| **보조공학 장치** | 장애인의 기능적 능력을 늘리거나 유지·향상시키기 위해 사용되는 물건, 장비, 제품의 일부분 또는 생산 시스템. 기성화된 것, 개조된 것, 전용 맞춤 모두 포함 |
| **보조공학 서비스** | 장애인이 보조공학 장치를 **선택, 습득, 사용**할 수 있도록 직접 도와주는 것 |

**목적**: 장애의 영향을 줄여 개인의 능력을 확대하고, 과제 수행의 대안적 방법을 제공하여 장애를 보상하거나 무시되도록 한다.

> ⚠️ **기출 포인트**: 보조공학 장치와 서비스를 혼동하지 말 것. "선택·습득·사용을 직접 도와주는 것"은 **서비스**다.

---

### 1.2 기술력 수준에 따른 분류

저급 → 고급으로 갈수록 비용·융통성·내구성·훈련 요구·세밀성·이동 가능성·유지 요구가 강화된다.

| 수준 | 예시 | 특징 |
|------|------|------|
| **고급(High) 테크놀로지** | 컴퓨터, 상호작용 멀티미디어 시스템 | 효과적·융통성 높음. 조작 복잡, 고비용, 훈련 필요 |
| **중급 테크놀로지** | 비디오 장치, 휠체어 | 덜 복잡한 전기·기계 장치 |
| **저급(Low) 테크놀로지** | 연필 잡기 보조 도구, 그림 의사소통판 | 일반 도구에 약간의 기능 수정 추가 |
| **무(No) 테크놀로지** | 교실 내 책상 간격 넓히기 | 장치·기기 없음. 교수 절차·관련 서비스(물리치료, 작업치료)로 해결 |

> ⚠️ **기출 포인트**: 특수교육공학에서는 **하이테크보다 로우테크를 먼저 고려**하는 것이 바람직하다.

---

### 1.3 보편적 설계(UD) vs 보조공학(AT)

| 구분 | 보편적 설계(UD) / UDL | 보조공학(AT) |
|------|----------------------|-------------|
| **적용 대상** | 폭넓은 범위의 모든 학생 | **개별** 학생 |
| **제공 시점** | 제품·서비스 개발 중 | 제품 완성 후 / 서비스 전달 중 |
| **접근성** | 계속적(지속) 접근성 | 소모적 접근성 |
| **실행 주체** | 일반 교육 교사 | **특수 교육 교사** |

## 2. 보조공학의 사정

### 2.1 사정 및 중재 원칙

1. 보조공학 중도 포기 방지를 위해 인간·활동·보조공학·주변 상황을 체계적으로 고려하는 **생태학적 사정** 실시
2. 목적은 손상 치료·재활이 아니라 **기능적 활동 수행 가능**에 있음
3. 사정은 **지속적이고 신중**하게
4. **협력과 소비자 중심** 방법 필요
5. 데이터 수집·해석 방법에 대한 이해 필요

---

### 2.2 사정 모델 비교

| 모델 | 핵심 초점 | 구성 요소 |
|------|-----------|-----------|
| **HAAT 모델** | 개인이 원하는 활동과 환경 탐색 → 성취에 초점 | 인간 / 활동 / 보조공학 / 맥락(사회적·물리적·문화적·제도적) |
| **MPT 모델** (인간과 공학의 조화) | 선택·구입 전 **조화의 정도** 확인 | 환경 / 개인 특징·선호도 / 공학의 기능·특성 |
| **SETT 구조 모델** | 학생·환경·과제·도구 4영역 + 팀 협력 | Student / Environment / Task / Tool |
| **보조공학 숙고 과정 모델** | 적절한 보조공학 선택의 직접적 과정 | 검토 → 개발 → 조사 → 평가 → 확인 (5단계) |

---

#### HAAT 모델 구성 요소 상세

| 구성 | 세부 내용 |
|------|-----------|
| **인간** | 신체적·인지적·정서적 숙련 요소 |
| **활동** | 여가 / 자기 관리 / 생산적 활동 |
| **보조공학** | 인간 공학 인터페이스 / 처리 장치 / 활동 출력 장치 / 환경 교류 장치 |
| **맥락** | 사회적·물리적·문화적·제도적 요소 |

> ⚠️ **기출 포인트**: HAAT 모델은 **임상적 관점이나 손상에 초점 맞추는 시각을 지양**하고, 사용자-보조공학-환경 간 **부조화**로 문제를 이해할 것을 강조한다.

---

#### SETT 구조 모델 영역별 역할

| 영역 | 내용 |
|------|------|
| **S**tudent (학생) | 참여자들이 학생이 해야 할 일을 함께 결정 |
| **E**nvironment (환경) | 물리적 환경에 존재하는 것들을 찾아 목록 작성 |
| **T**ask (과제) | 학생이 수행해야 할 모든 과제 조사 |
| **T**ool (도구) | 학생·환경·과제를 파악한 참여자들이 결정에 초점 |

> ⚠️ **기출 포인트**: SETT는 교육자·관련 인력·가족·학생 본인의 **팀 참여**를 전제한다.

---

#### 보조공학 숙고 과정 모델 5단계

| 단계 | 내용 |
|------|------|
| 1. 검토 | 학생의 **능력** 검토 |
| 2. 개발 | 능력과 교육적 발전에 맞는 **연간 목표·목적·기준** 개발 |
| 3. 조사 | 목표 수행에 필요한 **모든 과제** 조사 |
| 4. 평가 | 확인된 과제의 **난이도** 평가 |
| 5. 확인 | 학생에게 맞는 **지원과 서비스** 확인 |

## 3. 보조공학 기기 및 컴퓨터 접근

### 3.1 보조공학 기기 예시

| 기기 | 설명 |
|------|------|
| **헤드 포인터** | 머리에 장착하여 키보드 키 입력. 끝부분 고무팁으로 미끄럼 방지 |
| **마우스 스틱** | 입에 물고 키보드 키 입력 |
| **트랙볼** | 볼 마우스를 뒤집어 놓은 형태. 움직이지 않는 틀 위의 볼을 조작 → 운동 능력 낮은 학생이 제한된 공간에서도 사용 가능 |
| **화면 키보드** | 컴퓨터 화면상의 키보드에 마우스/대체 마우스로 입력. 자판 크기·배열 변형 가능 |
| **키가드** | 표준 키보드 위에 장착. 운동 신경 장애 사용자가 **다른 키를 건드리지 않고** 원하는 키만 정확히 입력하도록 지원 |
| **음성 인식 시스템** | 키보드 대신 음성으로 입력. 시스템 훈련을 통해 인식 정확성 향상 |
| **음성 합성 장치** | 텍스트를 음성으로 출력 |
| **의사소통판** | 그림·기호 기반 AAC 도구 |
| **스탠드 확대경** | 저시력 학생의 읽기 자료 확대 |

---

### 3.2 컴퓨터 접근 유의점 — 내게 필요한 옵션

| 기능 | 대상 | 내용 |
|------|------|------|
| **고정키(StickyKeys)** | 지체 장애 학생 | 두 개 이상의 키를 **동시에 누르는 데 어려움**이 있는 경우 활용 |
| **필터키(FilterKeys)** | 지체 장애 학생 | **반복된 키 입력을 자동 무시**. 키를 오래 눌러 같은 문자가 계속 입력되는 문제 해결 |
| **스크린 리더** | 시각 장애 학생 | 컴퓨터 **화면 읽기** 프로그램. 대체 입력 프로그램이 **아님** |

> ⚠️ **기출 포인트**: **스크린 리더는 대체 입력 프로그램이 아니라 화면 읽기 프로그램**이다. 혼동 주의.

> ⚠️ **기출 포인트**: 고정키는 "동시 누름 어려움", 필터키는 "반복 입력 문제"에 대응한다. 각각의 적용 대상과 목적을 구분해서 암기할 것.
